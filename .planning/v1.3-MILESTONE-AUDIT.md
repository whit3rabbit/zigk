## Tech Debt

### Phase 34: Timer Notification Modes
- SIGEV_THREAD_ID timer expiry path (scheduler.zig:1140-1159) does not call `sched.unblock` on the target thread if it is blocked. The signal is delivered via `pending_signals` bitmask and `siginfo_queue`, but the target will not wake until its next scheduling turn. This differs from `deliverSignalToThreadWithInfo` which explicitly unblocks. Not a correctness bug -- the signal is still delivered -- but an optimization gap for blocked SIGEV_THREAD_ID targets.

### Documentation
- REQUIREMENTS.md traceability table still shows all 16 requirements as `[ ] Pending`. The implementation is complete and verified across all phases. The checkboxes should be updated to `[x]` with "Complete" status.

## Bonus Fixes Delivered

Several pre-existing bugs were fixed as part of milestone execution:

- **destroyProcess use-after-free** (Phase 31): Fork failure left dangling child pointer in parent's children list, causing #GP on traversal. Fixed by calling `parent.removeChild(proc)` before free.
- **rt_sigreturn rax clobber** (Phase 28): dispatch_syscall's `setReturnSigned` overrode the restored frame from rt_sigreturn. Fixed by skipping setReturnSigned for SYS_RT_SIGRETURN.
- **recvfromIp /10 divisor** (Phase 33): Stale divisor from 100Hz era caused 10x longer UDP receive timeouts at 1000Hz. Fixed by removing divisor.
- **exitWithStatus zombie marking** (Phase 30): Single-thread processes killed by signal were not marked Zombie, causing wait4 to hang. Fixed by checking refcount and setting Zombie state.
