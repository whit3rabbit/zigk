**CLN-02: Fix slab_bench.zig for Zig 0.16.x**

4. In `tests/unit/slab_bench.zig`, replace the `std.time.Timer` usage with `std.c.clock_gettime`. The `std.time.Timer` type was removed in Zig 0.16.x. The std.time module now only exports time unit constants.

   Replace the current timing pattern:
   ```zig
   var timer = try std.time.Timer.start();
   // ... work ...
   const alloc_ns = timer.lap();
   // ... work ...
   const free_ns = timer.lap();
   ```

   With a `clock_gettime(CLOCK_MONOTONIC)` based approach. Since this is a host-side test (not freestanding), `std.c.clock_gettime` is available. Define a helper function:
   ```zig
   fn nanoTimestamp() u64 {
       var ts: std.c.timespec = undefined;
       _ = std.c.clock_gettime(std.c.CLOCK.MONOTONIC, &ts);
       return @as(u64, @bitCast(@as(i64, ts.sec))) * std.time.ns_per_s + @as(u64, @bitCast(@as(i64, ts.nsec)));
   }
   ```
   **Note:** Use `@bitCast` (not `@intCast`) for the isize-to-u64 conversion. `@intCast` panics on negative values, which is theoretically possible if the timespec fields are signed. `@bitCast` reinterprets the bits safely. In practice monotonic clock values are non-negative, but defensive coding avoids a runtime panic.

   Then use it:
   ```zig
   const t0 = nanoTimestamp();
   // ... alloc loop ...
   const t1 = nanoTimestamp();
   // ... free loop ...
   const t2 = nanoTimestamp();
   const alloc_ns = t1 - t0;
   const free_ns = t2 - t1;
   ```

   Remove the `try` from the timer initialization (nanoTimestamp cannot fail). The test function signature remains `!void` because `testing.expect` still returns error.
