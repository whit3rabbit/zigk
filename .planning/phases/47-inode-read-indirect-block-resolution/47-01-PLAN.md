**Part A: ext2 image population (build.zig)**

Extend the ext2 image creation script in build.zig to populate test files after mke2fs. The current script only formats the image. Add a `debugfs` step (from e2fsprogs, same package as mke2fs) that writes three test files:

1. `hello.txt` -- A small file (13 bytes: "Hello, ext2!\n") using only direct blocks. Verifies INODE-02.
2. `medium.bin` -- A ~100KB file using singly indirect blocks (needs >48KB = >12 direct blocks at 4KB). Fill with a repeating pattern: bytes 0x00 through 0xFF repeating. Verifies INODE-03.
3. `large.bin` -- A ~5MB file using doubly indirect blocks (needs > 12 + 1024 = 1036 blocks = ~4.1MB at 4KB). Fill with the same repeating pattern. Verifies INODE-04.

The population script should:
- Run after the mke2fs step, before the stamp file is written
- Use `debugfs -w ext2.img` with `-R` commands to create files, OR use `dd` to create temp files and `e2cp` to copy them in. Since `debugfs` is part of e2fsprogs and is guaranteed available, prefer `debugfs` with a script approach:
  ```
  # Create temp files with known content
  dd if=/dev/zero bs=1 count=13 | tr '\0' ... > /tmp/hello.txt  # or printf
  dd if=/dev/urandom bs=1024 count=100 of=/tmp/medium.bin  # but we need deterministic!
  ```
  Actually, use `printf` for hello.txt and a deterministic pattern generator for the binary files. The simplest approach: use a shell loop writing 256-byte blocks of sequential bytes.

  Better approach: use `debugfs -w ext2.img` with write commands:
  ```sh
  printf 'Hello, ext2!\n' > /tmp/zk_hello.txt
  # Generate 100KB deterministic pattern (repeating 0..255)
  python3 -c "import sys; sys.stdout.buffer.write(bytes(range(256))*400)" > /tmp/zk_medium.bin
  # Generate 5MB deterministic pattern
  python3 -c "import sys; sys.stdout.buffer.write(bytes(range(256))*20480)" > /tmp/zk_large.bin
  # Copy into ext2 image
  debugfs -w ext2.img -R "write /tmp/zk_hello.txt hello.txt"
  debugfs -w ext2.img -R "write /tmp/zk_medium.bin medium.bin"
  debugfs -w ext2.img -R "write /tmp/zk_large.bin large.bin"
  rm -f /tmp/zk_hello.txt /tmp/zk_medium.bin /tmp/zk_large.bin
  ```

  IMPORTANT: `debugfs` uses the same Homebrew/PATH resolution as mke2fs. Use the `$DEBUGFS` variable found alongside `$MKE2FS` in the same e2fsprogs install. The path is `$(dirname $MKE2FS)/debugfs`.

  If `python3` is not available, fall back to a `dd` + `awk` approach, or just use `dd if=/dev/urandom`. But deterministic content is critical for byte-for-byte test verification. On macOS and CI (Ubuntu), `python3` is always available. If not, the script should warn and skip population (tests will skip).

  The pattern for medium.bin and large.bin: repeating bytes 0x00, 0x01, ..., 0xFF. At offset N, the byte value is `N % 256`. This is trivially verifiable in userspace tests.

  File sizes:
  - hello.txt: 13 bytes (fits in 1 direct block)
  - medium.bin: 102400 bytes (100KB = 25 blocks, needs single-indirect at block 12+)
  - large.bin: 5242880 bytes (5MB = 1280 blocks, needs double-indirect at block 1036+)

**Part B: Create src/fs/ext2/inode.zig**

Create `src/fs/ext2/inode.zig` with the following functions and types:

1. **Ext2Error** error set: `IOError, InvalidInode, InvalidSuperblock, OutOfMemory, FileTooLarge, NotFound, AccessDenied`

2. **readInode(fs: *Ext2Fs, inum: u32) Ext2Error!types.Inode**
   - Validate inum != 0 (return InvalidInode)
   - Compute group_idx = (inum - 1) / ipg, offset_in_group = (inum - 1) % ipg
   - Validate group_idx < fs.group_count
   - Get bg_inode_table from fs.block_groups[group_idx]
   - Compute byte offset: table_start_byte + offset_in_group * fs.inode_size (use std.math.mul for overflow safety)
   - Convert to LBA: byte_offset / SECTOR_SIZE
   - Read 1-2 sectors (2 if inode spans sector boundary) into a stack buffer of 2*SECTOR_SIZE bytes with align(4)
   - SECURITY: Zero-init the buffer before read (@memset or [_]u8{0})
   - Cast to types.Inode via @ptrCast/@alignCast
   - Log inode details with console.debug
   - Return inode by value

3. **resolveBlock(fs: *Ext2Fs, inode: *const types.Inode, logical_block: u32) Ext2Error!u32**
   - Compute ptrs_per_block = fs.block_size / 4
   - Direct blocks (logical_block < 12): return inode.i_block[logical_block]
   - Single-indirect (logical_block < 12 + ptrs_per_block):
     - Get indirect block num from inode.i_block[types.INDIRECT_BLOCK]
     - If 0, return 0 (sparse)
     - Heap-allocate block buffer with alignedAlloc(u8, .@"4", fs.block_size) -- NOT stack! (MEMORY.md: 4KB stack overflow)
     - @memset buffer to 0 (DMA hygiene)
     - Read indirect block via dev.readSectors
     - Cast to u32 slice via std.mem.bytesAsSlice(u32, buf)
     - Return ptrs[lb_after_direct]
   - Double-indirect (logical_block < 12 + ptrs_per_block + ptrs_per_block^2):
     - Get double-indirect block num from inode.i_block[types.DOUBLE_INDIRECT_BLOCK]
     - If 0, return 0 (sparse)
     - Heap-allocate outer block buffer, read outer table
     - Compute outer_idx and inner_idx
     - Heap-allocate inner block buffer, read inner table
     - Return inner_ptrs[inner_idx]
   - Beyond double-indirect: return error.FileTooLarge (triple-indirect is ADV-01, deferred)

4. **Ext2File struct** (private_data for FileDescriptor):
   ```zig
   const Ext2File = struct {
       fs: *Ext2Fs,
       inode_num: u32,
       inode: types.Inode,
       size: u64,
   };
   ```

5. **ext2_file_ops** (fd.FileOps vtable):
   - read = ext2FileRead
   - write = null (read-only Phase 47)
   - close = ext2FileClose
   - seek = ext2FileSeek
   - stat = ext2FileStat
   - All others null

6. **ext2FileRead(file_desc: *fd.FileDescriptor, buf: []u8) isize**
   - Get Ext2File from private_data
   - Check position >= size -> return 0 (EOF)
   - Calculate remaining, to_read
   - Loop block by block:
     - logical_block = pos / block_size
     - byte_in_block = pos % block_size
     - resolveBlock to get phys_block
     - If phys_block == 0: memset zeros (sparse)
     - Else: heap-alloc block buf, readSectors, memcpy chunk
     - Update read_count and pos
   - Update file_desc.position
   - Return read count as isize

7. **ext2FileClose(file_desc: *fd.FileDescriptor) isize**
   - Get Ext2File, destroy with heap.allocator()
   - Return 0

8. **ext2FileSeek(file_desc: *fd.FileDescriptor, offset: i64, whence: u32) isize**
   - SEEK_SET (0), SEEK_CUR (1), SEEK_END (2)
   - Compute new position, clamp to file size
   - Update file_desc.position
   - Return new position

9. **ext2FileStat(file_desc: *fd.FileDescriptor, stat_buf: *anyopaque) isize**
   - Fill a Stat struct (or use uapi.stat.Stat) with inode metadata
   - Mode from inode.i_mode (u16 widened to u32), size from inode.i_size, uid/gid, timestamps, nlink, ino
   - Return 0

10. **lookupInRootDir(fs: *Ext2Fs, name: []const u8) Ext2Error!u32**
    - Read root directory inode (inode 2) via readInode
    - Iterate through data blocks of the root directory inode
    - For each block: resolve physical block, read block data, walk DirEntry records by rec_len
    - Compare name_len + name bytes against the search name
    - Return inode number on match, NotFound otherwise
    - This is a single-level lookup (root dir only). Full path traversal is Phase 48.

11. **openInode(fs: *Ext2Fs, inum: u32, flags: u32) !*fd.FileDescriptor**
    - Read inode via readInode
    - Allocate Ext2File on heap, fill with fs/inum/inode/size
    - Call fd.createFd(&ext2_file_ops, flags, file_ctx)
    - Use errdefer to clean up Ext2File on failure

**Critical implementation notes (from RESEARCH and MEMORY.md):**
- All block buffer allocations MUST be heap (not stack) to avoid kernel stack overflow on aarch64
- Use alignedAlloc(u8, .@"4", ...) for buffers that will be cast to u32 or struct pointers
- Use std.math.mul/add for ALL offset arithmetic (CLAUDE.md rule 5)
- Zero-init all buffers before DMA reads (CLAUDE.md rule 3)
- inode_size (stride in table) != @sizeOf(types.Inode) (struct size = 128). Use fs.inode_size for table stride.
- SECTOR_SIZE is from block_device module, imported as @import("block_device").SECTOR_SIZE
- Block 0 in i_block means sparse (not allocated) -- return zeros, do NOT read disk block 0
- inode numbers are 1-based; (inum-1) in all offset calculations
