<task type="auto">
  <name>Task 1: Create page cache module with core data structures and API</name>
  <files>src/kernel/fs/page_cache.zig</files>
  <action>
Create `src/kernel/fs/page_cache.zig` implementing a global file page cache.

**Data structures:**

1. `CachedPage` struct:
   - `phys_addr: u64` -- physical address of the 4KB page
   - `file_id: u64` -- identifies the file (use FileDescriptor.file_identifier)
   - `page_offset: u64` -- page-aligned offset within the file (byte offset / PAGE_SIZE)
   - `ref_count: std.atomic.Value(u32)` -- reference count, starts at 1 when inserted
   - `dirty: bool` -- true if page has been modified and needs write-back
   - `valid: bool` -- true if page contains valid data (vs allocated but not yet populated)
   - `next: ?*CachedPage` -- for hash chain linking

2. `PageCache` struct -- global singleton:
   - Hash table of CachedPage entries indexed by (file_id, page_offset)
   - Use a fixed-size hash table: `HASH_BUCKETS = 256` entries, each bucket is a linked list head (`?*CachedPage`)
   - `lock: sync.Spinlock` -- protects the hash table
   - `total_pages: usize` -- count of cached pages for statistics
   - `MAX_CACHED_PAGES: usize = 1024` -- eviction threshold (4MB of cached data)

**Hash function:** `hash(file_id, page_offset) = ((file_id *% 0x9E3779B97F4A7C15) ^ (page_offset *% 0x517CC1B727220A95)) % HASH_BUCKETS`

**Public API:**

- `pub fn init() void` -- initialize the global page cache (clear hash table)

- `pub fn getPages(file_id: u64, offset: u64, len: usize, read_fn: ?ReadFn, fd: ?*FileDescriptor) ![]PageRef`
  - Returns an array of `PageRef` structs covering the requested byte range
  - `PageRef = struct { page: *CachedPage, offset_in_page: usize, len: usize }`
  - For each page in the range: lookup in hash table, if found increment ref_count and return
  - On cache miss: allocate a physical page via `pmm.allocZeroedPages(1)`, populate via `read_fn` if provided, insert into hash table
  - Read-ahead: when a miss occurs, also prefetch the NEXT page (1-page read-ahead window) if within file bounds
  - The caller array should be allocated from heap (caller frees via `releasePages`)
  - Return error.ENOMEM if PMM or heap allocation fails

- `pub fn releasePages(refs: []PageRef) void`
  - Decrements ref_count on each page
  - Frees the PageRef array from heap
  - Does NOT evict pages (they stay cached with ref_count=0 for future hits)

- `pub fn markDirty(page: *CachedPage) void`
  - Sets `dirty = true` on the page

- `pub fn writeback(file_id: u64, write_fn: WriteFn, fd: *FileDescriptor) void`
  - Iterates all pages for file_id, writes dirty pages back via write_fn, clears dirty flag
  - Called on fsync or file close

- `pub fn invalidate(file_id: u64) void`
  - Removes all pages for file_id from the cache
  - Frees physical pages for entries with ref_count == 0
  - Entries with ref_count > 0 are marked invalid (will be freed when released)

- `pub fn getPageData(page: *CachedPage) [*]u8`
  - Returns kernel virtual address for the page's physical memory
  - Uses `hal.paging.physToVirt(page.phys_addr)` to get HHDM virtual address

**Type definitions:**
- `ReadFn = *const fn (fd: *FileDescriptor, buf: []u8) isize` -- same as FileOps.read
- `WriteFn = *const fn (fd: *FileDescriptor, buf: []const u8) isize` -- same as FileOps.write
- `PageRef = struct { page: *CachedPage, offset_in_page: usize, len: usize }` -- a reference to a range within a cached page

**Eviction (simple LRU-approximation):**
When `total_pages >= MAX_CACHED_PAGES` and a new page is needed, scan buckets for pages with `ref_count == 0` and `dirty == false`. Free the first one found (FIFO within bucket). If all unreferenced pages are dirty, write them back first, then free. If no unreferenced pages exist, return error.ENOMEM.

**Thread safety:** All hash table operations (lookup, insert, remove) must hold `lock`. Physical page data access after getPages does NOT require the lock (ref_count prevents eviction).

**Imports:** Use `@import("pmm")`, `@import("hal")`, `@import("heap")`, `@import("sync")`, `@import("fd")` (for FileDescriptor type). Follow the same import pattern as pipe.zig.

**Important:** Use checked arithmetic (`std.math.add`, `std.math.mul`) for all offset/size calculations per CLAUDE.md security rules. Zero-initialize allocated pages via `pmm.allocZeroedPages` to prevent information leaks.

1. **fd.zig -- add page_cache field to FileDescriptor:**
   - Add `page_cache_file_id: u64 = 0` field to FileDescriptor struct (reuse `file_identifier` value when non-zero)
   - This ID is set at VFS open time (already done via `file_identifier`)
   - No separate field needed -- zero-copy syscalls will use `file_identifier` directly as the page cache key
   - Actually, on reflection, `file_identifier` is already set by VFS.open and is unique per (mount_idx, file). It is the right key.
   - Add to the `unref` function (where refcount hits 0): call `page_cache.writeback(self.file_identifier, ...)` if the file was writable, then call `page_cache.invalidate(self.file_identifier)` to flush cached pages for this file. Guard with `if (self.file_identifier != 0)`.
   - Import page_cache: `const page_cache = @import("page_cache");`
   - The writeback call needs a write_fn. Use `self.ops.write` if non-null, otherwise skip writeback.
   - For the writeback call, the fd needs to be seeked to the correct position. The writeback function in page_cache.zig handles this internally by calling seek + write on each dirty page.

2. **root.zig -- re-export page_cache:**
   - Add `pub const page_cache = @import("page_cache.zig");` to `src/kernel/fs/root.zig`

- The unref/close path must NOT hold the page_cache lock for extended periods. Do writeback first (which acquires/releases lock per page), then invalidate.
