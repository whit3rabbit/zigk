<objective>
Refactor all zero-copy I/O syscalls (splice, sendfile, tee, copy_file_range) to use the page cache from Plan 01 instead of allocating 64KB kernel buffers.

Purpose: Eliminate kernel buffer copies for file-to-file and file-to-pipe transfers. Data flows through page cache references: file data is read into cached pages once, then those pages are referenced directly by the destination (pipe buffer write or file write), avoiding the intermediate heap allocation and memcpy.

Output: Updated splice.zig and read_write.zig using page cache, plus integration tests confirming correct behavior.
</objective>

**Refactor `spliceFileToPipe` in splice.zig:**

Replace the 64KB heap buffer approach with page cache:

```
fn spliceFileToPipe(file_fd, off_in_ptr, pipe_fd, len) -> usize:
  1. Determine read_offset (from off_in_ptr or file_fd.position)
  2. Get file_id from file_fd.file_identifier
  3. If file_id == 0, fall back to the existing 64KB buffer approach (non-VFS files)
  4. Loop while total_sent < len:
     a. chunk_size = min(remaining, PAGE_SIZE * 4)  // process up to 4 pages at a time
     b. Call page_cache.getPages(file_id, read_offset, chunk_size, file_fd.ops.read, file_fd)
     c. For each PageRef in the returned slice:
        - Get page data pointer via page_cache.getPageData(ref.page)
        - Write the relevant portion (ref.offset_in_page..ref.offset_in_page+ref.len) to pipe via pipe_mod.writeToPipeBuffer
        - Track bytes written
     d. Call page_cache.releasePages(refs)
     e. Update read_offset and total_sent
     f. Handle short writes (pipe full) and EOF
  5. Write updated offset back to userspace if using offset_ptr
```

**Refactor `splicePipeToFile` in splice.zig:**

For pipe-to-file, data comes from the pipe buffer (not the page cache), so the pattern is:
1. Read from pipe into a temporary buffer (still needed -- pipe data is in the pipe's circular buffer)
2. Write to file -- here we COULD cache, but the primary win is on the read side

Keep the existing approach for pipe-to-file but use a stack buffer instead of heap allocation when possible (pipe buffer is only 4KB, so stack is fine):

```
fn splicePipeToFile(pipe_fd, file_fd, off_out_ptr, len) -> usize:
  1. Use a stack buffer: var kbuf: [4096]u8 = undefined;
  2. Read from pipe buffer into kbuf
  3. Write to file via file_fd.ops.write
  4. If the file has a page_cache file_id, invalidate the written range
     (the page cache for this file region is stale after a direct write)
```

Actually, for pipe-to-file, the write goes through the normal FileOps.write path. The page cache invalidation for the written region is important to prevent stale reads. Call `page_cache.invalidate(file_id)` after writing (simple approach -- invalidate entire file since the write could be anywhere).

**Refactor `sys_tee` in splice.zig:**

Tee copies data between two pipes. Currently it peeks from source pipe into a 64KB heap buffer, then writes to dest pipe. Since pipes use a 4KB circular buffer, use a stack buffer:

```
fn sys_tee(fd_in, fd_out, len, flags) -> usize:
  1. var kbuf: [4096]u8 = undefined;  // Stack buffer, pipe is max 4KB
  2. peek_len = min(len, 4096)
  3. bytes_peeked = pipe_mod.peekPipeBuffer(in_handle, kbuf[0..peek_len])
  4. bytes_written = pipe_mod.writeToPipeBuffer(out_handle, kbuf[0..bytes_peeked])
  5. Return bytes_written
```

This eliminates the 64KB heap allocation. Tee between pipes does not benefit from the page cache directly (pipe data is not file data), but eliminating the heap allocation is still an improvement.

**Refactor `sys_copy_file_range` in splice.zig:**

Replace the 64KB heap buffer with page cache for the source file:

```
fn sys_copy_file_range(fd_in, off_in_ptr, fd_out, off_out_ptr, len, flags) -> usize:
  1. Get in_file_id from in_fd.file_identifier
  2. If in_file_id == 0, fall back to existing buffer approach
  3. Loop while total_copied < len:
     a. chunk_size = min(remaining, PAGE_SIZE * 4)
     b. refs = page_cache.getPages(in_file_id, read_offset, chunk_size, in_fd.ops.read, in_fd)
     c. For each PageRef:
        - Get page data via page_cache.getPageData
        - Seek out_fd to write_offset
        - Write the page data to out_fd via out_fd.ops.write
        - Track bytes written
     d. page_cache.releasePages(refs)
     e. If out_fd has a file_identifier, invalidate its cache (written data makes cached pages stale)
     f. Update offsets and total_copied
  4. Write updated offsets back to userspace
```

**Refactor `sys_sendfile` in read_write.zig:**
