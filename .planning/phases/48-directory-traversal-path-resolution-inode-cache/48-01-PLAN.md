3. In `init()`, after `self.* = .{ ... }`, explicitly zero-initialize the cache:

**Part C: Implement core path resolution and inode cache (inode.zig)**

Add the following functions to `src/fs/ext2/inode.zig`:

1. **INODE_CACHE_SIZE constant and InodeCacheEntry struct:**
   ```zig
   pub const INODE_CACHE_SIZE: usize = 16;
   pub const InodeCacheEntry = struct {
       inum: u32,       // 0 = slot is empty
       inode: types.Inode,
       lru_gen: u64,    // higher = more recently used
   };
   ```

2. **getCachedInode(fs: *Ext2Fs, inum: u32) Ext2Error!types.Inode** (INODE-05):
   - If inum == 0, return error.InvalidInode
   - Linear scan of fs.inode_cache (16 entries):
     - On hit: bump lru_gen, return inode
     - Track oldest slot (lowest lru_gen) during scan
   - On miss: call readInode(fs, inum), evict oldest slot, return inode
   - See 48-RESEARCH.md Pattern 6 for exact implementation

3. **lookupInDir(fs: *Ext2Fs, dir_inode: *const types.Inode, name: []const u8) Ext2Error!u32** (DIR-01):
   - Generalized from lookupInRootDir. Takes a pre-read directory inode instead of always reading inode 2.
   - Validate dir_inode.isDir(), name.len in range
   - Compute dir_blocks from dir_inode.i_size
   - For each logical block: resolveBlock (supports ALL indirection levels, not just direct), read block, walk DirEntry by rec_len
   - Compare entry name against search name
   - Return inode number on match, error.NotFound otherwise
   - See 48-RESEARCH.md Pattern 2 for exact implementation

4. **resolvePath(fs: *Ext2Fs, path: []const u8) Ext2Error!u32** (DIR-01):
   - Start from ROOT_INODE (inode 2)
   - Strip leading '/'
   - If empty after strip, return ROOT_INODE (root itself)
   - Use std.mem.tokenizeScalar(u8, path, '/') to split components
   - For each component:
     - getCachedInode for current_inum
     - Verify inode.isDir()
     - lookupInDir to get next inode number
   - Component count limit: 255 (safety against circular structures)
   - Return final inode number
   - See 48-RESEARCH.md Pattern 1 for exact implementation

   NOTE: resolvePath does NOT follow symlinks during traversal. If a path component is a symlink, lookupInDir returns the symlink's inode number, and the next iteration's isDir() check will fail (symlinks are not directories). This is the correct behavior for Phase 48 -- symlink-following during traversal is a future enhancement.

5. **Ext2DirFd struct and ext2_dir_ops vtable** (DIR-02):
   ```zig
   const Ext2DirFd = struct {
       fs: *Ext2Fs,
       dir_inode_num: u32,
       dir_inode: types.Inode,
   };

   pub const ext2_dir_ops = fd.FileOps{
       .read = null,
       .write = null,
       .close = ext2DirClose,
       .seek = null,
       .stat = ext2DirStat,
       .ioctl = null,
       .mmap = null,
       .poll = null,
       .truncate = null,
       .getdents = ext2GetdentsFromFd,
       .chown = null,
   };
   ```

6. **ext2GetdentsFromFd(file_desc, dirp, count) isize** (DIR-02):
   - Get Ext2DirFd from file_desc.private_data
   - Use file_desc.position as absolute byte offset in directory data
   - For each directory block:
     - Compute logical_block and byte_in_block from abs_offset
     - resolveBlock, readSectors into heap-allocated block buffer
     - Walk DirEntry records by rec_len (CRITICAL: use rec_len as stride, NOT computed from name_len)
     - For each valid entry (inode != 0, name_len > 0):
       - Compute Dirent64 output size (header + name + null + 8-byte alignment)
       - If it fits in remaining user buffer: write Dirent64 header, name, null terminator
       - Map ext2 file_type to DT_* constants (FT_REG_FILE->DT_REG, FT_DIR->DT_DIR, FT_SYMLINK->DT_LNK, etc.)
       - Update bytes_written
     - Advance abs_offset to next block
   - Update file_desc.position
   - Return bytes_written as isize
   - See 48-RESEARCH.md Pattern 3 for exact implementation

   CRITICAL ANTI-PATTERN: The last entry in each directory block has rec_len padded to the end of the block. Walking by `align(8 + name_len, 4)` instead of `entry.rec_len` silently skips the last entry. Always use `offset += entry.rec_len`.

   NOTE: getdents writes to userspace via volatile pointer writes (same pattern as sfsGetdents). The dirp address has already been validated by sys_getdents64 via isValidUserAccess before calling the FileOps getdents callback.

7. **ext2DirClose(file_desc) isize**:
   - Get Ext2DirFd from private_data, destroy with heap.allocator()
   - Return 0

8. **ext2DirStat(file_desc, stat_buf) isize**:
   - Get Ext2DirFd from private_data
   - Fill Stat struct from dir_inode (same pattern as ext2FileStat but with S_IFDIR mode)
   - Return 0

9. **openDirInode(fs: *Ext2Fs, inum: u32, flags: u32) Ext2Error!*fd.FileDescriptor**:
   - Read inode via getCachedInode
   - Verify isDir()
   - Allocate Ext2DirFd on heap
   - Call fd.createFd(&ext2_dir_ops, flags, dir_ctx)
   - Use errdefer for cleanup
   - This replaces the Phase 47 openRootDir stub

**Part D: Wire VFS callbacks (mount.zig)**

1. **Update ext2Open** to use resolvePath instead of lookupInRootDir:
   - Strip leading '/'
   - If rel_path is empty, call openDirInode(fs, ROOT_INODE, flags) instead of openRootDir
   - For non-empty paths: call resolvePath(fs, rel_path), then read the resolved inode
   - If the resolved inode is a directory, call openDirInode
   - If a regular file, call openInode (existing)
   - If a symlink or other type, return error.NotSupported (symlinks not opened directly in Phase 48)

2. **Update ext2StatPath** to use resolvePath instead of lookupInRootDir:
   - For root (empty path): use getCachedInode(fs, ROOT_INODE)
   - For non-empty paths: use resolvePath(fs, rel_path), then getCachedInode

3. **Implement ext2Readlink** (DIR-03):
   - Signature: `fn ext2Readlink(ctx: ?*anyopaque, path: []const u8, buf: []u8) vfs.Error!usize`
   - Strip leading '/' from path
   - resolvePath to get inode number
   - getCachedInode to get inode
   - Verify isSymlink(), else return error.NotSupported
   - Fast symlink check: i_size <= 60 AND i_blocks == 0
   - If not fast symlink: return error.NotSupported (slow symlinks ADV-02, deferred)
   - Cast i_block[0..15] as *const [60]u8, copy i_size bytes to buf
   - Return bytes copied
   - See 48-RESEARCH.md Pattern 4 for exact implementation

4. **Implement ext2Statfs** (DIR-05):
   - Signature: `fn ext2Statfs(ctx: ?*anyopaque) vfs.Error!uapi.stat.Statfs`
   - Read from fs.superblock (in-memory, no disk read)
   - f_type = 0xEF53 (EXT2_SUPER_MAGIC)
   - f_bsize = fs.block_size
   - f_blocks = s_blocks_count
   - f_bfree = s_free_blocks_count
   - f_bavail = s_free_blocks_count - s_r_blocks_count (guard underflow)
   - f_files = s_inodes_count
   - f_ffree = s_free_inodes_count
   - f_namelen = 255
   - f_frsize = fs.block_size
   - f_flags = 1 (ST_RDONLY)
   - See 48-RESEARCH.md Pattern 5 for exact implementation

5. **Update VFS FileSystem struct** in init():
   - Set `statfs = ext2Statfs` (was null)
   - Set `readlink = ext2Readlink` (was null)
   - The `getdents = null` in the VFS FileSystem is NOT used for ext2 -- ext2 getdents is dispatched through the directory FD's FileOps.getdents callback, which is checked first in sys_getdents64.

6. **Remove or deprecate openRootDir and lookupInRootDir direct usage:**
   - openRootDir is replaced by openDirInode
   - lookupInRootDir remains in inode.zig but is no longer called from mount.zig (replaced by resolvePath which uses lookupInDir)
   - Keep lookupInRootDir as a convenience alias: `pub const lookupInRootDir = ...` or just leave it as dead code. The compiler will optimize it away if unused.

**Critical implementation notes:**
- All block buffer allocations MUST be heap-allocated (not stack). 4KB buffers on stack = overflow on aarch64.
- Use alignedAlloc(u8, .@"4", ...) for buffers cast to u32 or struct pointers.
- Use std.math.mul/add for ALL offset arithmetic (CLAUDE.md rule 5).
- Zero-init all buffers before DMA reads (CLAUDE.md rule 3).
- The inode cache struct increases Ext2Fs size by ~2240 bytes (16 * 140). This is heap-allocated so no stack concern.
- ext2GetdentsFromFd writes to userspace via volatile pointer (same as sfsGetdents pattern). The user address is validated by the caller (sys_getdents64).
- Inode number 0 is the empty-slot sentinel in the cache. This is safe because ext2 inode numbering is 1-based and getCachedInode rejects inum=0.
