Create `src/user/test_runner/tests/syscall/fs_extras.zig` with integration tests following the pattern in `at_ops.zig`. Import `syscall` and `std`.

Define local constants:
```zig
const AT_FDCWD: i32 = -100;
const AT_SYMLINK_NOFOLLOW: i32 = 0x100;
```

**Tests to implement (each is a `pub fn` returning `!void`):**

**FS-01: readlinkat (success + error)**

1. `testReadlinkatBasic` -- Success path:
   - Create a symlink on /mnt: `syscall.symlink("/shell.elf", "/mnt/test_rl_link")` (if symlink is supported on SFS). If SFS does not support symlink, test with an InitRD symlink if one exists, OR test that readlinkat on a non-symlink file returns the expected error (EINVAL).
   - Workaround if SFS has no symlink support: call `syscall.readlinkat(AT_FDCWD, "/proc/self/exe", ...)` or similar known symlink. If no symlinks exist at all in the filesystem, test readlinkat on a regular file and verify it returns an error (EINVAL -- not a symlink).
   - Best approach: Test readlinkat error path -- call on a regular file (e.g., `/shell.elf`) and verify it returns EINVAL. This tests the syscall plumbing without requiring symlink filesystem support.

2. `testReadlinkatInvalidPath` -- Error path:
   - Call `readlinkat(AT_FDCWD, "/nonexistent_readlinkat_path", buf, buf.len)` and verify ENOENT.

**FS-02: linkat (success + error)**

3. `testLinkatBasic` -- Success path:
   - Create a test file on /mnt: `const fd = try syscall.open("/mnt/test_linkat_src", O_WRONLY | O_CREAT | O_TRUNC, 0o644);` (keep fd open, no close -- SFS deadlock workaround)
   - Call `syscall.linkat(AT_FDCWD, "/mnt/test_linkat_src", AT_FDCWD, "/mnt/test_linkat_dst", 0)`
   - Verify the link exists: `syscall.access("/mnt/test_linkat_dst", 0)` should succeed
   - Cleanup: `syscall.unlink("/mnt/test_linkat_dst") catch {};` and `syscall.unlink("/mnt/test_linkat_src") catch {};`

4. `testLinkatCrossDevice` -- Error path:
   - Try to link a file from / (initrd) to /mnt (SFS): `linkat(AT_FDCWD, "/shell.elf", AT_FDCWD, "/mnt/cross_link", 0)` should return EXDEV (cross-device link) or EROFS (read-only source).
   - Accept either error as valid.

**FS-03: symlinkat (success + error)**

5. `testSymlinkatBasic` -- Success path:
   - Call `syscall.symlinkat("/shell.elf", AT_FDCWD, "/mnt/test_symlinkat_link")`
   - If it succeeds, verify the symlink target via readlink: `syscall.readlink("/mnt/test_symlinkat_link", buf)` should return "/shell.elf"
   - Cleanup: `syscall.unlink("/mnt/test_symlinkat_link") catch {};`
   - If symlinkat returns an error (SFS may not support symlinks), skip the test by returning success (document: "SFS may not support symlinks")

6. `testSymlinkatEmptyTarget` -- Error path:
   - Call `symlinkat("", AT_FDCWD, "/mnt/test_empty_sym")` -- should return ENOENT (empty target)

**FS-04: utimensat (success + error)**

7. `testUtimensatNull` -- Success path (NULL times = set to now):
   - Create a file: `const fd = try syscall.open("/mnt/test_utime", O_WRONLY | O_CREAT | O_TRUNC, 0o644);`
   - Call `syscall.utimensat(AT_FDCWD, "/mnt/test_utime", null, 0)`
   - Verify it returns success (0)
   - Cleanup: `syscall.unlink("/mnt/test_utime") catch {};`
   - Note: We may not be able to verify actual timestamp values if SFS does not persist them, but the syscall should not error.

8. `testUtimensatSpecificTime` -- Success path (specific times):
   - Create a file on /mnt
   - Set specific timestamps: `times[0] = .{ .tv_sec = 1000000, .tv_nsec = 500 };` (atime), `times[1] = .{ .tv_sec = 2000000, .tv_nsec = 999 };` (mtime)
   - Call utimensat with the times array
   - Verify it succeeds (or returns EROFS if filesystem is read-only -- accept both for SFS)
   - Cleanup

9. `testUtimensatSymlinkNofollow` -- Error path:
   - Call `utimensat(AT_FDCWD, "/shell.elf", null, AT_SYMLINK_NOFOLLOW)` -- should return ENOSYS

10. `testUtimensatInvalidNsec` -- Error path:
    - Set `times[0].tv_nsec = 1_000_000_000` (out of range)
    - Call utimensat -- should return EINVAL

**FS-05: futimesat (success + error)**

11. `testFutimesatBasic` -- Success path:
    - Create a file on /mnt
    - Call `syscall.futimesat(AT_FDCWD, "/mnt/test_futimesat", null)` (NULL = set to now)
    - Verify success
    - Cleanup

12. `testFutimesatSpecificTime` -- Success path:
    - Create a file on /mnt
    - Set times: `times[0] = .{ .tv_sec = 1000000, .tv_usec = 500000 };` (atime), `times[1] = .{ .tv_sec = 2000000, .tv_usec = 999999 };` (mtime)
    - Call futimesat with times
    - Verify success (or EROFS -- accept both)
    - Cleanup

**Important patterns:**
- Use `O_WRONLY | O_CREAT | O_TRUNC` for file creation (import or define these constants)
- Do NOT explicitly close SFS files -- SFS close deadlock workaround
- Each test cleans up after itself with `catch {}` on cleanup calls
- Test functions return `!void` and return `error.TestFailed` on assertion failures
- Access O_* constants from `syscall.O_WRONLY`, `syscall.O_CREAT`, `syscall.O_TRUNC` (check what's exported)
