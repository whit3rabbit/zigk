<objective>
Fix the double-copy EFAULT bug in existing *at syscalls (linkat, symlinkat, readlinkat) and add infrastructure for timestamp manipulation syscalls (utimensat, futimesat).

Purpose: The existing *at syscalls pass kernel pointers to base syscalls that expect userspace pointers, causing EFAULT on relative paths. This is the same bug already fixed in fstatat/mkdirat/unlinkat/renameat/fchmodat. Additionally, utimensat/futimesat need syscall number constants and VFS-level timestamp infrastructure before implementation.
Output: Working *at syscalls for link/symlink/readlink, VFS timestamp infrastructure ready for Plan 02.
</objective>

<task type="auto">
  <name>Task 1: Extract kernel-space helpers and fix *at double-copy bugs</name>
  <files>src/kernel/sys/syscall/fs/fs_handlers.zig</files>
  <action>
Extract kernel-space path helpers for link, symlink, and readlink, following the exact pattern established by `unlinkKernel`, `mkdirKernel`, `rmdirKernel`, `renameKernel`, `chmodKernel`, and `chownKernel` already in fs_handlers.zig.

**1. Create `linkKernel(raw_old: []const u8, raw_new: []const u8) SyscallError!usize`:**
- Move the body of `sys_link` into this helper (canonicalize both paths, check permissions via Vfs.statPath + checkAccess, call Vfs.link)
- The helper takes kernel-space path slices (already copied from userspace)
- `sys_link` becomes: copy both paths from userspace, call `linkKernel(raw_old, raw_new)`

**2. Create `symlinkKernel(target: []const u8, raw_linkpath: []const u8) SyscallError!usize`:**
- Move body of `sys_symlink` into this helper
- Note: `target` is stored as-is (data, not resolved). Only `raw_linkpath` gets canonicalized.
- `sys_symlink` becomes: copy both from userspace, call `symlinkKernel(target, raw_linkpath)`

**3. Create `readlinkKernel(raw_path: []const u8, buf_ptr: usize, bufsiz: usize) SyscallError!usize`:**
- Move body of `sys_readlink` into this helper (canonicalize path, validate user buffer, call Vfs.readlink, copy to user via UserPtr)
- `sys_readlink` becomes: copy path from userspace, call `readlinkKernel(raw_path, buf_ptr, bufsiz)`

**4. Fix `sys_linkat`:**
- Remove the call to `sys_link(@intFromPtr(...), @intFromPtr(...))` which causes double-copy EFAULT
- Instead call `linkKernel(resolved_old, resolved_new)` directly
- For absolute paths: call `linkKernel(oldpath, newpath)` after canonicalization

**5. Fix `sys_symlinkat`:**
- Remove call to `sys_symlink(@intFromPtr(...), @intFromPtr(...))`
- Call `symlinkKernel(target, resolved_link)` directly
- For absolute linkpath: call `symlinkKernel(target, linkpath)` directly (target is never resolved against dirfd -- it is data)

**6. Fix `sys_readlinkat`:**
- Remove call to `sys_readlink(@intFromPtr(...), ...)`
- Call `readlinkKernel(resolved, buf_ptr, bufsiz)` directly
- For absolute paths: call `readlinkKernel(path, buf_ptr, bufsiz)` directly

Follow the exact allocator pattern (heap.allocator alloc/free with defer) used by the existing *Kernel helpers. See `renameKernel` or `chownKernel` for the two-path pattern.

**4. Add Vfs.setTimestamps static method:**

In `src/fs/vfs.zig`, add a public method to the `Vfs` struct (near chmod/chown):
```zig
/// Set file timestamps with nanosecond precision
/// atime_sec/mtime_sec of -1 means "leave unchanged"
pub fn setTimestamps(path: []const u8, atime_sec: i64, atime_nsec: i64, mtime_sec: i64, mtime_nsec: i64) Error!void {
    if (path.len == 0) return error.InvalidPath;
    if (path[0] != '/') return error.InvalidPath;

    const held = lock.acquire();
    defer held.release();

    const mount = findMountPoint(path) orelse return error.NotFound;

    if (mount.fs.set_timestamps) |set_ts_fn| {
        return set_ts_fn(mount.fs.context, path, atime_sec, atime_nsec, mtime_sec, mtime_nsec);
    }
    return error.NotSupported;
}
```

Note: InitRD (read-only) and DevFS (virtual) will have `set_timestamps = null`, returning NotSupported. SFS can be hooked up later if needed. For now, the MVP accepts that timestamp setting only works on filesystems that implement the callback. The utimensat syscall will translate NotSupported to EROFS.
