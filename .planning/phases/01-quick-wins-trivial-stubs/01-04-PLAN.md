    - Identify which files to add wrappers to (match the kernel-side module grouping)

    The wrappers should follow the existing pattern. Each wrapper:
    1. Takes typed arguments (not raw usize)
    2. Calls the raw syscall via primitive.syscallN()
    3. Checks for error (negative return) and converts to Zig error
    4. Returns the result

    **Scheduling wrappers** (add to appropriate file, likely scheduling.zig or create if needed):
    - sched_get_priority_max(policy: u32) -> i32 or error
    - sched_get_priority_min(policy: u32) -> i32 or error
    - sched_getscheduler(pid: u32) -> u32 or error
    - sched_getparam(pid: u32, param: *SchedParam) -> void or error
    - sched_setscheduler(pid: u32, policy: u32, param: *const SchedParam) -> void or error
    - sched_setparam(pid: u32, param: *const SchedParam) -> void or error
    - sched_rr_get_interval(pid: u32, interval: *Timespec) -> void or error
    - ppoll(fds: [*]PollFd, nfds: usize, timeout: ?*const Timespec, sigmask: ?*const u64) -> i32 or error

    **Resource limit wrappers** (add to resource.zig or appropriate file):
    - prlimit64(pid: u32, resource: u32, new_limit: ?*const Rlimit, old_limit: ?*Rlimit) -> void or error
    - getrusage(who: i32, usage: *Rusage) -> void or error

    **Signal wrappers** (add to signal.zig):
    - rt_sigpending(set: *u64) -> void or error
    (rt_sigsuspend is tricky since it always returns EINTR; wrap accordingly)

    **Memory wrappers** (add to io.zig or memory.zig):

    - munlockall() -> void or error
    - mincore(addr: usize, length: usize, vec: [*]u8) -> void or error

    Also ensure the wrapper module's root.zig (or equivalent) re-exports these new functions so tests can `@import("syscall")` and access them.

    Add integration tests for all 14 new syscalls. Distribute tests across existing test files by category:

    **In misc.zig (scheduling + resource limits + signal queries):**

    Add test functions following the existing pattern (pub fn testXxx() !void):

    1. testSchedGetPriorityMax: Call sched_get_priority_max(SCHED_FIFO), verify result == 99.
    2. testSchedGetPriorityMin: Call sched_get_priority_min(SCHED_FIFO), verify result == 1.
    3. testSchedGetPriorityInvalid: Call sched_get_priority_max(999), verify it returns EINVAL.
    4. testSchedGetScheduler: Call sched_getscheduler(0), verify result == 0 (SCHED_OTHER).
    5. testSchedGetParam: Call sched_getparam(0, &param), verify param.sched_priority == 0.
    6. testSchedSetScheduler: Call sched_setscheduler(0, SCHED_RR, &{.sched_priority=50}), verify returns 0. Then call sched_getscheduler(0), verify == SCHED_RR. Then restore SCHED_OTHER.
    7. testSchedRrGetInterval: Call sched_rr_get_interval(0, &ts), verify ts.tv_nsec == 100_000_000.
    8. testPrlimit64GetNofile: Call prlimit64(0, RLIMIT_NOFILE, null, &old), verify old.rlim_cur == 1024 (DEFAULT_NOFILE_SOFT).
    9. testGetrusageSelf: Call getrusage(RUSAGE_SELF, &usage), verify returns 0 (we accept zeroed stats).
    10. testGetrusageInvalid: Call getrusage with invalid `who` (e.g., 99), verify returns EINVAL.
    11. testRtSigpending: Call rt_sigpending(&set), verify returns 0 and set value is a valid u64.

    **In memory.zig (memory management stubs):**

    12. testMadviseBasic: mmap an anonymous page, call madvise(addr, PAGE_SIZE, MADV_NORMAL), verify returns 0, then munmap.
    13. testMadviseInvalidAlign: Call madvise(1, PAGE_SIZE, MADV_NORMAL) with unaligned addr, verify returns EINVAL.
    14. testMlockMunlock: mmap a page, call mlock(addr, PAGE_SIZE), verify 0. Call munlock(addr, PAGE_SIZE), verify 0.
    15. testMlockallMunlockall: Call mlockall(MCL_CURRENT), verify 0. Call munlockall(), verify 0.
    16. testMincoreBasic: mmap a page, call mincore(addr, PAGE_SIZE, &vec), verify vec[0] == 1 (resident).
    17. testMincoreInvalidAlign: Call mincore(1, PAGE_SIZE, &vec) with unaligned addr, verify returns EINVAL.

    - Handle errors by returning `error.TestFailed`
    - Be self-contained (no dependencies between tests)
    - Clean up any resources (munmap pages, restore scheduler policy)
