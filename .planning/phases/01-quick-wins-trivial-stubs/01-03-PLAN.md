Output: 4 new syscall handlers in the process and signals modules.

    **sys_prlimit64(pid: usize, resource: usize, new_limit_ptr: usize, old_limit_ptr: usize) SyscallError!usize:**

    This is the modern get/set resource limits syscall. It supersedes getrlimit/setrlimit.

    1. Resolve target process:
       ```zig
       const target_pid: u32 = @truncate(pid);
       const proc = if (target_pid == 0)
           base.getCurrentProcess()
       else
           process_mod.findProcessByPid(target_pid) orelse return error.ESRCH;
       ```

    2. If old_limit_ptr != 0, get current limit (reuse the existing getrlimit switch logic):
       - Build an Rlimit struct using the same switch(resource) pattern already in sys_getrlimit.
       - Write to userspace via UserPtr.from(old_limit_ptr).writeValue(old_limit).

    3. If new_limit_ptr != 0, set new limit:
       - Read Rlimit from userspace via UserPtr.from(new_limit_ptr).readValue(Rlimit).
       - Validate: if rlim_cur > rlim_max and rlim_max != RLIM_INFINITY, return EINVAL.
       - For RLIMIT_AS: store proc.rlimit_as = new_limit.rlim_cur. Non-root check: if new hard limit > current hard and proc.euid != 0, return EPERM.
       - For other resources: accept but don't enforce (same as sys_setrlimit).

    4. Return 0.

    NOTE: The Rlimit type and RLIMIT_* constants are already defined in process.zig. Reuse them.

    **sys_getrusage(who: usize, usage_ptr: usize) SyscallError!usize:**

    1. Validate usage_ptr != 0 (return EFAULT).

    2. Define the Rusage struct (Linux ABI compatible) locally in the file:
       ```zig
       const Timeval = extern struct {
           tv_sec: i64,
           tv_usec: i64,
       };

       const Rusage = extern struct {
           ru_utime: Timeval,    // user CPU time
           ru_stime: Timeval,    // system CPU time
           ru_maxrss: i64,       // max RSS in KB
           ru_ixrss: i64,
           ru_idrss: i64,
           ru_isrss: i64,
           ru_minflt: i64,
           ru_majflt: i64,
           ru_nswap: i64,
           ru_inblock: i64,
           ru_oublock: i64,
           ru_msgsnd: i64,
           ru_msgrcv: i64,
           ru_nsignals: i64,
           ru_nvcsw: i64,
           ru_nivcsw: i64,
       };
       ```

    3. Validate `who` parameter:
       - RUSAGE_SELF (0): Return stats for current process
       - RUSAGE_CHILDREN (@bitCast(@as(isize, -1))): Return stats for waited-for children
       - RUSAGE_THREAD (1): Return stats for current thread
       - Any other value: return EINVAL

    4. For MVP, return a zeroed Rusage struct for all valid `who` values. The kernel does not currently track most resource usage fields. Use `std.mem.zeroes(Rusage)`.

    5. Write to userspace: `UserPtr.from(usage_ptr).writeValue(usage) catch return error.EFAULT;`

    6. Return 0.

    NOTE: Import `std` at top of file if not already present (it likely is for std.math operations in existing code).

  <files>src/kernel/sys/syscall/process/signals.zig</files>
  <action>
    Add 2 new syscall handlers to the EXISTING signals.zig file. Place them after the existing sys_sigaltstack function (around line 200) and before the "Signal Sending Syscalls" section.

    **sys_rt_sigpending(set_ptr: usize, sigsetsize: usize) SyscallError!usize:**

    1. Validate sigsetsize == @sizeOf(uapi.signal.SigSet). If not, return EINVAL.
       Check what SigSet is -- it should be u64 (8 bytes). If it's a different type, adjust accordingly.

    2. Get current thread: `const thread = sched.getCurrentThread() orelse return error.ESRCH;`

    3. Compute pending signals that are blocked: `const pending = thread.pending_signals & thread.sigmask;`

    4. Write to userspace: `UserPtr.from(set_ptr).writeValue(pending) catch return error.EFAULT;`

    5. Return 0.

    **sys_rt_sigsuspend(mask_ptr: usize, sigsetsize: usize) SyscallError!usize:**

    1. Validate sigsetsize == @sizeOf(uapi.signal.SigSet). If not, return EINVAL.

    2. Read new signal mask from userspace:
       `const new_mask = UserPtr.from(mask_ptr).readValue(uapi.signal.SigSet) catch return error.EFAULT;`

    3. Get current thread: `const thread = sched.getCurrentThread() orelse return error.ESRCH;`

    4. Save old mask: `const old_mask = thread.sigmask;`

    5. Set new mask (temporarily): `thread.sigmask = new_mask;`

    6. Ensure SIGKILL and SIGSTOP cannot be blocked (same pattern as rt_sigprocmask):
       ```zig
       uapi.signal.sigdelset(&thread.sigmask, uapi.signal.SIGKILL);
       uapi.signal.sigdelset(&thread.sigmask, uapi.signal.SIGSTOP);
       ```

    7. Suspend until a signal arrives: `sched.block();`
       The thread will be woken when a signal is delivered to it.

    8. Restore old mask: `thread.sigmask = old_mask;`

    9. Always return EINTR (sigsuspend always returns -1 with errno=EINTR per POSIX):
       `return error.EINTR;`

    NOTE: The imports for `uapi`, `sched`, `UserPtr`, `SyscallError` are already at the top of signals.zig. Verify that `uapi.signal.SigSet` and `uapi.signal.sigdelset` are accessible.
  </action>

    Grep for `sys_rt_sigpending` and `sys_rt_sigsuspend` in signals.zig to confirm functions exist.
  </verify>
  <done>

Both validate sigsetsize and handle EFAULT on invalid pointers.
  </done>
