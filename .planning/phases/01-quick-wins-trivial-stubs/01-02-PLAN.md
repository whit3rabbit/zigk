    Add 7 new syscall handler functions to the EXISTING scheduling.zig file. These go after the existing scheduling section (after sys_sched_yield) but before the clock/timer section. Add a new section header comment: "// Scheduling Policy Queries/Sets".

    First, add needed imports at the top of the file (if not already present):
    - `const process_mod = @import("process");` (for findProcessByPid)
    Verify `base` is already imported (it is). The Process type is available via `base.getCurrentProcess()`.

    Add a helper constant block for scheduling policies:
    ```zig
    // POSIX scheduling policies
    const SCHED_OTHER: usize = 0;
    const SCHED_FIFO: usize = 1;
    const SCHED_RR: usize = 2;
    const SCHED_BATCH: usize = 3;
    const SCHED_IDLE: usize = 5;
    ```

    Add the SchedParam struct:
    ```zig
    const SchedParam = extern struct {
        sched_priority: i32,
    };
    ```

    **sys_sched_get_priority_max(policy: usize) SyscallError!usize:**
    - Switch on policy: SCHED_FIFO/SCHED_RR return 99, SCHED_OTHER/SCHED_BATCH/SCHED_IDLE return 0, else return EINVAL.

    **sys_sched_get_priority_min(policy: usize) SyscallError!usize:**
    - Switch on policy: SCHED_FIFO/SCHED_RR return 1, SCHED_OTHER/SCHED_BATCH/SCHED_IDLE return 0, else return EINVAL.

    **sys_sched_getscheduler(pid: usize) SyscallError!usize:**
    - `const target_pid: u32 = @truncate(pid);`
    - If target_pid == 0, use getCurrentProcess(), else use process_mod.findProcessByPid(target_pid) orelse return error.ESRCH.
    - Return `proc.sched_policy` (the new field from Plan 01 Task 1).

    **sys_sched_getparam(pid: usize, param_ptr: usize) SyscallError!usize:**
    - Validate param_ptr != 0 (return EFAULT).
    - Resolve process same as getscheduler.
    - Create SchedParam with .sched_priority = proc.sched_priority.
    - Write to userspace via UserPtr.from(param_ptr).writeValue(param).
    - Return 0.

    **sys_sched_setscheduler(pid: usize, policy: usize, param_ptr: usize) SyscallError!usize:**
    - Validate param_ptr != 0 (return EFAULT).
    - Resolve process same as above.
    - Validate policy is one of SCHED_OTHER/FIFO/RR/BATCH/IDLE, else EINVAL.
    - Read SchedParam from userspace via UserPtr.from(param_ptr).readValue(SchedParam).
    - Validate priority: for FIFO/RR, priority must be 1-99; for others, must be 0. Return EINVAL on mismatch.
    - Store: `proc.sched_policy = @truncate(policy);` and `proc.sched_priority = param.sched_priority;`
    - Return 0 (MVP: does not actually change scheduler behavior).

    **sys_sched_setparam(pid: usize, param_ptr: usize) SyscallError!usize:**
    - Validate param_ptr != 0.
    - Resolve process.
    - Read SchedParam from userspace.
    - Validate priority against current proc.sched_policy (same rules as setscheduler).
    - Store: `proc.sched_priority = param.sched_priority;`
    - Return 0.

    **sys_sched_rr_get_interval(pid: usize, interval_ptr: usize) SyscallError!usize:**
    - Validate interval_ptr != 0 (return EFAULT).
    - Resolve process (pid=0 means current).
    - Write Timespec{ .tv_sec = 0, .tv_nsec = 100_000_000 } (100ms, Linux default RR quantum) to userspace.
    - Return 0.

  <action>
    Add sys_ppoll to scheduling.zig. ppoll is an extended version of poll that takes a timespec (instead of int milliseconds) and a signal mask. For MVP, delegate to the existing poll infrastructure, ignoring the signal mask.

    **sys_ppoll(fds_ptr: usize, nfds: usize, timeout_ptr: usize, sigmask_ptr: usize, sigsetsize: usize) SyscallError!usize:**

    1. If sigmask_ptr != 0 and sigsetsize != @sizeOf(u64), return error.EINVAL. (SigSet is u64 in this kernel.)

    2. Convert timespec to poll timeout:
       - If timeout_ptr == 0: timeout_ms = -1 (infinite, but cast as isize for sys_poll)
       - If timeout_ptr != 0: Read Timespec from userspace. Validate tv_sec >= 0, tv_nsec >= 0, tv_nsec < 1_000_000_000. Convert: `total_ms = @as(u64, @intCast(ts.tv_sec)) * 1000 + @as(u64, @intCast(ts.tv_nsec)) / 1_000_000`. Cap at maxInt(i32) for safety.

    3. The existing sys_poll is in the net/poll.zig module. However, ppoll should NOT call sys_poll directly (it goes through userspace argument conventions). Instead, ppoll should replicate the core poll logic or call an internal helper.

       Check how the existing sys_poll is structured. The simplest approach: if there is a shared `doPoll` helper in poll.zig, call it. If not, ppoll can perform the same fd polling loop that sys_poll does. Look at net/poll.zig for the internal implementation. If sys_poll takes (fds_ptr, nfds, timeout_ms, socket_file_ops), then ppoll can call it with the computed timeout_ms cast. BUT sys_poll's timeout parameter might be typed as isize -- check.

       The simplest correct approach: have ppoll compute timeout_ms and then call `@import("net").sys_poll(fds_ptr, nfds, @bitCast(@as(isize, timeout_ms_i)), ...)` -- but this is fragile. Better: create a small internal `pollInternal` helper if one doesn't exist, or just call the existing poll's entry point with appropriate argument casting.

       Actually, looking at the research code example, the approach is:
       ```zig
       // For MVP, ignore sigmask and delegate to poll
       const net_poll = @import("net"); // or wherever sys_poll lives
       ```
       But this creates a cross-module dependency. The cleaner approach: just invoke the existing poll mechanism directly. Read the poll.zig code first to understand the internal API.

       FALLBACK APPROACH (if internal API is unclear): Implement ppoll as a simple no-op that returns 0 for timeout=0 (immediate check), or blocks via sleep for timeout > 0 then returns 0. This is technically wrong but satisfies the "programs don't crash" goal. Comment it as needing Phase 3 (I/O Multiplexing) for real implementation.

       PREFERRED APPROACH: Import the existing poll handler and delegate. The net module is already imported by table.zig. Add `const net_mod = @import("net");` or access poll through the existing fd poll infrastructure.

    4. For the signal mask: For MVP, ignore it entirely. Add a comment: "// MVP: sigmask ignored. Full implementation requires atomic mask swap around poll."
