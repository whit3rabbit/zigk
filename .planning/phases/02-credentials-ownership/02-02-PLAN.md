  <name>Task 1: Implement setreuid, setregid, setfsuid, setfsgid</name>
  <files>src/kernel/sys/syscall/process/process.zig</files>
  <action>
Add these 4 syscall handlers to process.zig, after the existing sys_setresgid/sys_getresgid functions (after line 448).

Follow the EXACT pattern of sys_setresuid/sys_setresgid for permission checking, cred_lock usage, and error handling.

1. **sys_setreuid(ruid: usize, euid: usize) SyscallError!usize**
   POSIX semantics (match Linux exactly):
   - const UNCHANGED: u32 = 0xFFFFFFFF (-1 truncated to u32)
   - Acquire cred_lock
   - If privileged (euid == 0 or hasSetUidCapability):
     - Can set ruid and euid to any value
   - If non-privileged:
     - ruid can only be set to current real or effective UID
     - euid can only be set to current real, effective, or saved UID
   - If ruid != UNCHANGED: set proc.uid = new_ruid
   - If euid != UNCHANGED: set proc.euid = new_euid; set proc.fsuid = new_euid (auto-sync per user decision)
   - POSIX rule: if ruid was set (or euid was set to value != previous real UID), then saved-set-user-ID is set to the new effective UID
     Specifically: `if (new_ruid != UNCHANGED or (new_euid != UNCHANGED and new_euid != old_euid)) proc.suid = proc.euid;`
     Capture old_euid before applying changes.
   - Return 0 on success, EPERM on failure

2. **sys_setregid(rgid: usize, egid: usize) SyscallError!usize**
   Same pattern as setreuid but for GIDs:
   - If privileged: can set any values
   - If non-privileged: rgid can be real or effective GID; egid can be real, effective, or saved GID
   - Auto-sync: if egid changes, set proc.fsgid = new_egid
   - If rgid was set or egid changed: proc.sgid = proc.egid
   - Return 0 on success

3. **sys_setfsuid(fsuid: usize) SyscallError!usize**
   Linux semantics (NOT standard error return):
   - const new_fsuid: u32 = @truncate(fsuid)
   - Acquire cred_lock
   - Save old_fsuid = proc.fsuid
   - Permission check: if non-privileged (euid != 0), new_fsuid must match one of: uid, euid, suid, or current fsuid. If not, return old_fsuid unchanged (NOT an error -- Linux semantics)
   - If privileged or permitted: proc.fsuid = new_fsuid
   - Return old_fsuid (ALWAYS -- this is the Linux return convention, returns previous value not 0/-errno)

4. **sys_setfsgid(fsgid: usize) SyscallError!usize**
   Same as setfsuid but for GID:
   - Save old_fsgid = proc.fsgid
   - Permission check: non-privileged must match gid, egid, sgid, or current fsgid
   - If permitted: proc.fsgid = new_fsgid
   - Return old_fsgid (ALWAYS)

IMPORTANT: setfsuid/setfsgid ALWAYS return the previous fsuid/fsgid, even on "failure" (when the value didn't change). They never return a negative errno. This is by Linux design.

IMPORTANT: For the capability checks in setreuid/setregid, use the same pattern as setresuid/setresgid:
- `proc.euid == 0 or proc.hasSetUidCapability(new_uid)` for UID
- `proc.euid == 0 or proc.hasSetGidCapability(new_gid)` for GID

  <name>Task 2: Implement getgroups and setgroups</name>

  <action>
Add getgroups and setgroups syscall handlers to process.zig, after the functions added in Task 1.

1. **sys_getgroups(size: usize, list_ptr: usize) SyscallError!usize**
   Linux semantics:
   - If size == 0: return proc.supplementary_groups_count (just query count, don't write)
   - If size < proc.supplementary_groups_count: return error.EINVAL
   - Validate user buffer: `if (!user_mem.isValidUserAccess(list_ptr, size * @sizeOf(u32), .Write)) return error.EFAULT;`
     (import user_mem at top of file if not already imported -- check existing imports)
   - Copy supplementary_groups[0..count] to userspace using UserPtr:
     ```zig
     const count = proc.supplementary_groups_count;
     const data_bytes = @as(*const [@sizeOf(u32) * 16]u8, @ptrCast(&proc.supplementary_groups));
     const uptr = UserPtr.from(list_ptr);
     _ = uptr.copyFromKernel(data_bytes[0..count * @sizeOf(u32)]) catch return error.EFAULT;
     ```
   - Return count
   - No lock needed for getgroups (supplementary_groups are modified only under cred_lock, and reading a stale count is acceptable for this read-only syscall)

2. **sys_setgroups(size: usize, list_ptr: usize) SyscallError!usize**
   Linux semantics:
   - Permission check: must be root (euid == 0) or have hasSetGidCapability per user decision
     Use: `if (proc.euid != 0 and !proc.hasSetGidCapability(0)) return error.EPERM;`
     Note: hasSetGidCapability takes a target GID; passing 0 is fine since we're checking for general setgid capability, same pattern as setgroups in Linux
   - If size > 16 (the supplementary_groups array size): return error.EINVAL
     (NGROUPS_MAX is 16 in this kernel)
   - If size > 0: validate user buffer and copy from userspace:
     ```zig
     if (!user_mem.isValidUserAccess(list_ptr, size * @sizeOf(u32), .Read)) return error.EFAULT;
     const uptr = UserPtr.from(list_ptr);
     var buf: [16 * @sizeOf(u32)]u8 = undefined;
     _ = uptr.copyToKernel(buf[0..size * @sizeOf(u32)]) catch return error.EFAULT;
     ```
   - Acquire cred_lock
   - Copy groups: `@memcpy(@as([*]u8, @ptrCast(&proc.supplementary_groups))[0..size * @sizeOf(u32)], buf[0..size * @sizeOf(u32)]);`
   - Set proc.supplementary_groups_count = @intCast(size)
   - If size < 16: zero remaining entries to prevent info leak
   - Release cred_lock
   - Return 0

Check that user_mem is imported at top of process.zig. The file already imports base.zig which re-exports UserPtr. Check if user_mem (for isValidUserAccess) is available through base. If sys_getresuid already does user pointer validation, follow its pattern.
