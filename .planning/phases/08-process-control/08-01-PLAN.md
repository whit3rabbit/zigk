  <name>Task 2: Kernel syscall implementations (prctl + affinity)</name>
  <files>

1. Create `src/kernel/sys/syscall/process/control.zig` with three syscall handlers:

   **sys_prctl(option, arg2, arg3, arg4, arg5) -> SyscallError!usize:**
   - Import: base.zig, uapi, sched, user_mem (same pattern as scheduling.zig)
   - Get current thread via `sched.getCurrentThread() orelse return error.ESRCH`
   - Switch on `option`:
     - `uapi.prctl.PR_SET_NAME`:
       - Validate arg2 != 0, return EFAULT if null
       - Use `user_mem.copyStringFromUser()` with a 16-byte kernel buffer to copy name from userspace
       - Truncate to 15 chars + null (Linux semantics: 16 bytes including null terminator)
       - Copy into `thread.name[0..copy_len]` and set `thread.name[copy_len] = 0`
       - Zero remaining bytes: `@memset(thread.name[copy_len + 1..], 0)` for clean reads
       - Return 0
     - `uapi.prctl.PR_GET_NAME`:
       - Validate arg2 != 0, return EFAULT if null
       - Find string length in thread.name (scan for null, max 16)
       - Use `UserPtr.from(arg2).copyFromKernel(thread.name[0..16])` to copy 16 bytes to userspace
       - Return 0
     - `else`: return error.EINVAL
   - Ignore arg3, arg4, arg5 (unused for SET_NAME/GET_NAME)

   **sys_sched_setaffinity(pid, cpusetsize, mask_ptr) -> SyscallError!usize:**
   - Validate cpusetsize >= 8 (minimum for 64-bit mask), return EINVAL otherwise
   - Resolve target process: pid 0 = current (via base.getCurrentProcess()), nonzero = findProcessByPid, return ESRCH if not found
   - Read first u64 from user mask via UserPtr.from(mask_ptr).readValue(u64), return EFAULT on failure
   - Validate CPU 0 is set: `if ((mask & 1) == 0) return error.EINVAL`
   - Single-CPU kernel: no state to store, just validate and succeed
   - Return 0

   **sys_sched_getaffinity(pid, cpusetsize, mask_ptr) -> SyscallError!usize:**
   - Validate cpusetsize >= 8, return EINVAL otherwise
   - Resolve target process same as setaffinity
   - Build 128-byte zero buffer, set byte 0 = 1 (CPU 0 bit)
   - Copy min(cpusetsize, 128) bytes to userspace via UserPtr.from(mask_ptr).copyFromKernel()
   - Return the copy size (Linux returns kernel's cpuset size)

2. Export the new functions from `src/kernel/sys/syscall/process/scheduling.zig` by adding at the bottom of the file:
   ```zig
   // Process control (prctl, CPU affinity) - separate module for organization
   const control = @import("control.zig");
   pub const sys_prctl = control.sys_prctl;
   pub const sys_sched_setaffinity = control.sys_sched_setaffinity;
   pub const sys_sched_getaffinity = control.sys_sched_getaffinity;
   ```
   The dispatch table discovers syscalls via the `scheduling` module (imported as such in table.zig), so exports must be visible through it.

Note: The dispatch table in table.zig uses comptime reflection on the module hierarchy. The `scheduling` module is imported in table.zig line 21 as `const scheduling = @import("scheduling");`. Adding pub const re-exports from scheduling.zig makes the new functions discoverable by the dispatch table without modifying table.zig.
