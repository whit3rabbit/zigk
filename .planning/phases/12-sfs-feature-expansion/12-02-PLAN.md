**In `src/fs/sfs/ops.zig`:**

1. Implement `sfsSymlink(ctx: ?*anyopaque, target: []const u8, linkpath: []const u8) vfs.Error!void`:
   - Extract linkname from linkpath (strip leading '/')
   - Validate linkname with `isValidFilename`, check length < 32
   - Reject nested paths (no '/' in linkname)
   - Target can be any string up to 511 bytes (stored in one data block). Validate target.len > 0 and target.len <= 511.
   - Follow the established TOCTOU-safe pattern from sfsOpen O_CREAT:
     a. Allocate a data block via `sfs_alloc.allocateBlock(self)` -- errdefer freeBlock on failure
     b. Write target string to the allocated block: zero a 512-byte buffer, copy target into it, write to disk via `sfs_io.writeSector(self, start_block, &buf)`. The first `target.len` bytes contain the target path, remaining bytes are zero (implicitly null-terminated).
     c. Read directory UNLOCKED to find free slot and check name does not exist
     d. Under alloc_lock: re-read block, validate slot still free, validate name not taken
     e. Create new DirEntry with:
        - name = linkname
        - start_block = allocated block
        - size = @intCast(target.len) (size stores the target string length)
        - flags = 1
        - mode = S_IFLNK | 0o777 (symlinks are always rwxrwxrwx per POSIX convention). Use `@import("fs_meta").S_IFLNK` for the constant (0o120000).
        - uid = 0, gid = 0
        - mtime = 0, atime = 0
        - nlink = 1
     f. Increment file_count
     g. Release lock
     h. Write directory block and superblock outside lock
     i. On failure: rollback file_count under lock

2. Implement `sfsReadlink(ctx: ?*anyopaque, path: []const u8, buf: []u8) vfs.Error!usize`:
   - Extract name from path (strip leading '/')
   - Validate name with `isValidFilename`
   - Read directory (UNLOCKED scan is fine for read-only operation):
     a. Allocate and read directory blocks via `sfs_io.readDirectoryAsync`
     b. Find entry by name
     c. If not found, return `vfs.Error.NotFound`
     d. If found but NOT a symlink (check `!entry.isSymlink()`), return `vfs.Error.NotSupported` (maps to EINVAL at syscall layer, which is correct for "not a symlink")
     e. Read the data block at `entry.start_block`: `sfs_io.readSector(self, entry.start_block, &sector_buf)`
     f. The target length is `entry.size`. Copy `min(entry.size, buf.len)` bytes from sector_buf to buf.
     g. Return the number of bytes copied.

   Security considerations:
   - Validate entry.start_block is within valid range (>= data_start, < total_blocks)
   - Validate entry.size <= 511 (max target length that fits in one block)
   - Zero-initialize sector_buf before read to prevent information leak

3. Update `sfsUnlink` to handle symlinks: When unlinking a symlink entry, its data block (containing the target path) needs to be freed. The existing unlink logic already handles this because it frees blocks based on `entry.start_block` and `blocks_used`. For symlinks, `blocks_used` will be 1 (since size <= 511, (size + 511) / 512 = 1). No changes needed here -- just verify the logic works.

4. Update `sfsOpen` to handle symlink entries: When opening a symlink via sfsOpen, the current code would open it as a regular file. This is actually fine for SFS because the VFS/syscall layer handles symlink resolution (or reports ELOOP). For now, opening a symlink returns the raw symlink data. Add a comment noting this behavior.

5. Update `sfsStatPath` to return correct mode for symlinks: The existing code already returns `e.mode` which will include `S_IFLNK`. Verify this is correct. Also update `sfsGetdents` to return `DT_LNK` (10) for symlink entries. Currently it only checks `isDirectory` for DT_DIR, otherwise DT_REG. Add: `if (entry.isSymlink()) DT_LNK else if (entry.isDirectory()) DT_DIR else DT_REG`.

6. Wire the new functions into the FileSystem struct:
   ```zig
   .symlink = sfs_ops.sfsSymlink,
   .readlink = sfs_ops.sfsReadlink,
   ```
