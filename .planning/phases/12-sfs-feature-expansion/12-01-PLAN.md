**In `src/fs/sfs/types.zig`:**

1. Add `nlink` and `atime` fields to `DirEntry` by repurposing padding bytes. The current layout is:
   - name[32] + start_block(4) + size(4) + flags(4) + mode(4) + uid(4) + gid(4) + mtime(4) = 60 bytes
   - _pad: [128 - 60]u8 = 68 bytes of padding

   New layout (still 128 bytes total):
   - name[32] + start_block(4) + size(4) + flags(4) + mode(4) + uid(4) + gid(4) + mtime(4) + atime(4) + nlink(4) + _pad[128 - 68]u8 = 128 bytes

   Add these fields BEFORE `_pad`:
   ```
   atime: u32,    // Access time (Unix timestamp)
   nlink: u32,    // Number of hard links (0 or 1 = single, >1 = hard-linked)
   _pad: [128 - 68]u8,
   ```

2. Add `isSymlink()` method to DirEntry:
   ```zig
   pub fn isSymlink(self: *const @This()) bool {
       return (self.mode & 0o170000) == 0o120000;
   }
   ```

**In `src/fs/sfs/ops.zig`:**

3. Update `sfsStat` to return actual mtime and atime from the DirEntry. Currently it returns 0 for all timestamps. In the `refreshMetadataFromDisk` function, add mtime and atime to the `RefreshedMetadata` struct (also update `SfsFile.RefreshedMetadata` in types.zig to include mtime and atime fields). Then in `sfsStat`, populate `stat.mtime` and `stat.atime` from the metadata.

4. Update `sfsOpen` O_CREAT path: When creating a new DirEntry, set `nlink = 1` and `atime = 0` in the new entry initialization.

5. Update `sfsMkdir`: When creating a new directory DirEntry, set `nlink = 1` and `atime = 0`.

6. Update `sfsUnlink`: Before clearing the directory entry, check `entry.nlink`. If nlink > 1, decrement nlink instead of deleting the entry. Only delete (clear flags, free blocks) when nlink <= 1 or nlink == 0 (for backward compat with entries created before this change, treat nlink=0 as nlink=1). The check should be:
   ```
   const effective_nlink = if (entry.nlink == 0) @as(u32, 1) else entry.nlink;
   if (effective_nlink > 1) {
       // Decrement nlink, keep entry alive
       entry.nlink -= 1;
       // Write back and return (do NOT clear entry or free blocks)
   } else {
       // Existing deletion logic
   }
   ```

7. Update `sfsStatPath` to include nlink in the returned FileMeta. Note: FileMeta in `src/fs/meta.zig` may not have a nlink field. If not, this is fine -- nlink is returned via sfsStat (fstat path) which populates the Stat struct directly. In sfsStat, set `stat.nlink` to `metadata.nlink` (or 1 if 0 for backward compat).

<task type="auto">
  <name>Task 2: Implement sfsLink and sfsSetTimestamps, wire into FileSystem</name>
  <files>src/fs/sfs/ops.zig, src/fs/sfs/root.zig</files>

**In `src/fs/sfs/ops.zig`:**

1. Implement `sfsLink(ctx: ?*anyopaque, old_path: []const u8, new_path: []const u8) vfs.Error!void`:
   - Extract old_name and new_name (strip leading '/')
   - Validate both filenames with `isValidFilename`
   - Reject nested paths (no '/' in name) -- SFS is flat
   - Follow the established TOCTOU-safe pattern from sfsRename:
     a. Read directory UNLOCKED to find old_entry and check new_name does not exist
     b. Verify old entry is a regular file (not directory -- POSIX: hard links to directories not allowed, return EPERM equivalent, which is `AccessDenied`)
     c. Check old entry is NOT a symlink (hard links to symlinks are technically allowed in POSIX but simplify by rejecting for SFS MVP)
     d. Find a free slot for the new entry
     e. Under alloc_lock: re-read both blocks, validate old entry still exists with same name, validate free slot still free, validate new_name not taken
     f. Create new DirEntry: copy old entry's start_block, size, mode, uid, gid, mtime, atime. Set name=new_name, flags=1, nlink=old_nlink+1. Also update old entry's nlink to match.
     g. Increment file_count in superblock
     h. Release lock
     i. Write directory blocks outside lock (may need to write 2 blocks if old and new are in different blocks)
     j. Write superblock outside lock
     k. On failure: rollback file_count and nlink under lock

   Important: Both the old entry AND the new entry must have the same nlink value. When you create the link, if old_entry.nlink was 0 (legacy), treat as 1, then set both entries' nlink to 2. If old_entry.nlink was already N, set both to N+1.

2. Implement `sfsSetTimestamps(ctx: ?*anyopaque, path: []const u8, atime_sec: i64, atime_nsec: i64, mtime_sec: i64, mtime_nsec: i64) vfs.Error!void`:
   - Extract name (strip leading '/')
   - Validate filename
   - Follow the established TOCTOU-safe pattern from sfsChmod:
     a. Read directory UNLOCKED to find entry
     b. Under alloc_lock: re-read block, validate entry still exists with same name
     c. If atime_sec != -1 (not UTIME_OMIT): set entry.atime = @truncate(@as(u64, @bitCast(atime_sec)))
     d. If mtime_sec != -1 (not UTIME_OMIT): set entry.mtime = @truncate(@as(u64, @bitCast(mtime_sec)))
     e. Release lock
     f. Write block outside lock
   - Note: SFS DirEntry stores timestamps as u32 Unix seconds. Nanosecond precision is not stored (lossy, acceptable for SFS).
