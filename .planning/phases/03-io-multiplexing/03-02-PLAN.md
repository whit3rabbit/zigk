<task type="auto">
  <name>Task 2: Rewrite sys_epoll_wait with real poll dispatch and blocking</name>
  <files>src/kernel/sys/syscall/process/scheduling.zig</files>
  <action>
Replace the body of sys_epoll_wait (starting around line 979) with a real implementation. Keep the function signature identical.

The new implementation must:

1. **Validate inputs** (keep existing validation for maxevents and events_ptr)

2. **Allocate kernel result buffer** (keep existing pattern)

3. **Poll loop with timeout:**
```
Parse timeout: cast to isize. -1 = infinite, 0 = immediate, >0 = milliseconds.
Record start TSC via hal.timing.rdtsc().

Loop:
  a. Acquire instance.lock, snapshot entries to local copy, release lock.
  b. For each active entry in snapshot:
     - Look up fd in FdTable via base.getGlobalFdTable().get(fd_u32)
     - If fd not found: revents = EPOLLNVAL (do NOT use POLLNVAL -- epoll uses u32 constants)
     - If fd found and fd.ops.poll exists: call fd.ops.poll(fd, entry.events) to get revents
     - If fd found but no poll method: assume always ready -- set EPOLLIN if fd.isReadable(), EPOLLOUT if fd.isWritable()
     - Always OR in EPOLLERR | EPOLLHUP from revents even if not in requested events (Linux behavior)
     - Mask revents: revents = revents & (entry.events | EPOLLERR | EPOLLHUP | EPOLLNVAL)

     Edge-triggered check (if EPOLLET set in entry.events):
     - new_events = revents & ~entry.last_revents  (only newly set bits)
     - If new_events == 0: skip this entry (no state transition)
     - Else: use new_events as revents for reporting

     Update last_revents in the REAL instance entry (not snapshot):
     - Acquire instance.lock briefly, find entry by fd, set entry.last_revents = revents_before_masking
     - Release lock

     EPOLLONESHOT check:
     - If EPOLLONESHOT set in entry.events and revents != 0:
       - Acquire instance.lock, find entry, set entry.events = 0 (disabled until EPOLL_CTL_MOD)
       - Release lock

     If revents != 0 (after edge-triggered filtering):
     - Add to result_buf: EpollEvent.init(revents, entry.data)
     - Increment ready_count
     - Break if ready_count >= maxevents

  c. If ready_count > 0 or timeout_i == 0: break out of loop
  d. If timeout_i > 0: check if elapsed time exceeds timeout (use hal.timing.hasTimedOut with timeout * 1000 microseconds). If expired, break.
  e. If timeout_i == -1 or not expired: yield via sched.yield() and retry.
     (Using sched.yield for MVP blocking. Full wait queue integration deferred per Claude's Discretion -- scheduler sleep with timeout retry is simpler and matches the existing select implementation pattern.)
```

4. **Copy results to userspace** (keep existing pattern with UserPtr.from(events_ptr).copyFromKernel)

5. **Return ready_count**

IMPORTANT: Remove the hardcoded stdin/stdout/stderr check (lines 1019-1028). The new code uses FileOps.poll universally for all fds.

IMPORTANT: The snapshot-then-update pattern for last_revents requires acquiring the instance lock twice in the inner loop (once for snapshot, once for update). This is acceptable because the lock is a spinlock held for very short durations. Alternative: work directly on instance.entries under lock for the entire scan. Choose whichever is cleaner -- the key correctness requirement is that last_revents is updated atomically relative to other epoll_wait calls on the same instance.

Simpler approach: Instead of snapshotting, iterate instance.entries directly under lock for the poll check, but release lock before calling poll (to avoid holding lock during poll which may acquire other locks). Pattern:
- Lock, copy entry fields (fd, events, data, last_revents, active) to locals, unlock
- Call poll
- Lock, update last_revents and handle EPOLLONESHOT, unlock
