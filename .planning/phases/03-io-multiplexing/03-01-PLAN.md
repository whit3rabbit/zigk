  <name>Task 1: Implement pipePoll for pipe file descriptors</name>
  <files>src/kernel/fs/pipe.zig</files>
  <action>
Add a `pipePoll` function and wire it into `pipe_ops` (replace `.poll = null` with `.poll = pipePoll`).

The function signature must match FileOps.poll: `fn(fd: *fd_mod.FileDescriptor, requested_events: u32) u32`

Implementation:
1. Extract PipeHandle from fd.private_data (same pattern as pipeRead/pipeWrite)
2. Acquire pipe.lock
3. Build revents bitmask based on pipe state:

For READ end (handle.end == .Read):
- Set POLLIN | POLLRDNORM if pipe.data_len > 0 (data available to read)
- Set POLLHUP if pipe.writers == 0 (all write ends closed -- EOF condition)
- Per Linux behavior: if writers == 0 AND data_len > 0, set BOTH POLLIN and POLLHUP (unread data + EOF)

For WRITE end (handle.end == .Write):
- Set POLLOUT | POLLWRNORM if (PIPE_BUF_SIZE - pipe.data_len) > 0 (space available)
- Set POLLERR if pipe.readers == 0 (broken pipe -- no reader)

4. Release pipe.lock
5. Return revents -- do NOT mask with requested_events. POLLERR and POLLHUP are always reported regardless of what was requested (Linux behavior). The caller (epoll_wait, select, poll) is responsible for masking.

Import uapi for the POLL* constants: `const uapi = @import("uapi");` (already imported in pipe.zig).
Use the epoll constants since FileOps.poll uses u32: `uapi.epoll.EPOLLIN`, `uapi.epoll.EPOLLOUT`, etc.
The low 16 bits of EPOLL* match POLL* values (verified by comptime check in poll.zig).
  </action>

`zig build -Darch=x86_64` compiles without errors. Grep pipe.zig for `pipePoll` and `.poll = pipePoll`.
  </verify>
  <done>
pipe_ops.poll is wired to pipePoll. Read end reports POLLIN when data available, POLLHUP when writers gone. Write end reports POLLOUT when space available, POLLERR when readers gone.

The function signature must match FileOps.poll: `fn(fd: *fd_mod.FileDescriptor, requested_events: u32) u32`

This function is a thin wrapper that delegates to the existing `checkPollEvents` in `src/net/transport/socket/poll.zig`, which already implements the correct TCP/UDP readiness logic (POLLIN when recv buffer has data or incoming connection, POLLOUT when send buffer has space, POLLHUP on peer close, POLLERR on socket error).

Implementation:
1. Extract SocketFdData from fd.private_data using getSocketData(fd)
2. If getSocketData returns null, return 0 (invalid socket fd)
3. Call `socket_mod.checkPollEvents(data.socket_idx, @truncate(requested_events))` where socket_mod is the socket transport module
4. Convert the u16 result to u32: `return @as(u32, result)`

The key insight: `checkPollEvents` takes a socket index (usize) and events mask (u16), and returns u16 revents. The FileOps.poll signature takes u32 events and returns u32 revents. So the wrapper converts between these widths.
