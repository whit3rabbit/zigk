<objective>
Add wait queue blocking to SysV IPC semaphores and message queues, and implement SEM_UNDO tracking.

Purpose: The current semop returns EAGAIN and msgsnd/msgrcv return EAGAIN/ENOMSG even without IPC_NOWAIT, which violates POSIX semantics. Blocking callers should sleep until the resource becomes available. SEM_UNDO is accepted but not tracked, which can leak semaphore adjustments when processes crash.

Output: Modified sem.zig and msg.zig with wait queue blocking, per-process SEM_UNDO list in Process struct, and cleanup on process exit.
</objective>

**sem.zig changes:**

1. Add `sched` import: `const sched = @import("sched");`

2. Add a `wait_queue: sched.WaitQueue` field to `SemSet` struct. Initialize to `.{}` in the default initializer.

3. Rewrite the blocking path in `semop()` (lines 169-193). The current code holds `sem_lock` for the entire operation and returns EAGAIN for the blocking case. The new approach:

   a. When a decrement operation would block (sem_op < 0 and semval < decr) and IPC_NOWAIT is NOT set:
      - Release sem_lock via `sched.waitOn(&set.wait_queue, held)` which atomically releases the lock and blocks the thread.
      - After wakeup, the function must re-acquire sem_lock and re-validate the semaphore set (it may have been removed via IPC_RMID while we slept).
      - Re-run the entire operation check from the beginning (re-validate all operations atomically).
      - This requires restructuring semop as a retry loop.

   b. When a wait-for-zero operation would block (sem_op == 0 and semval != 0) and IPC_NOWAIT is NOT set:
      - Same approach: `sched.waitOn(&set.wait_queue, held)`, then retry.

   c. The IPC_NOWAIT path remains unchanged: return error.EAGAIN immediately.

   The restructured semop should look like:
   ```
   pub fn semop(id: u32, sops: []const SemBuf, proc: *const process.Process) !void {
       while (true) {
           const held = sem_lock.acquire();
           // Validate sem set still exists (re-check after wakeup)
           const idx = ipc_perm.idToIndex(id);
           const seq = ipc_perm.idToSeq(id);
           if (idx >= SEMMNI) { held.release(); return error.EINVAL; }
           const set = &sem_sets[idx];
           if (!set.in_use or set.perm.seq != seq) { held.release(); return error.EIDRM; }
           // Re-validate permissions and operations...
           // If would block:
           //   if IPC_NOWAIT: { held.release(); return error.EAGAIN; }
           //   else: sched.waitOn(&set.wait_queue, held); continue; // retry
           // If all operations can proceed: apply them, wake waiters, release lock, return
       }
   }
   ```

   IMPORTANT: Add `error.EIDRM` to the error set. When a semaphore set is removed via IPC_RMID while threads are blocked, the woken threads should get EIDRM. Also add EIDRM mapping in the syscall layer `mapIpcError` in `src/kernel/sys/syscall/ipc/sem.zig`.

4. In `semctl` IPC_RMID handler (lines 277-293): After freeing resources and clearing the slot, wake ALL threads waiting on the wait queue:
   `set.wait_queue.wakeUp(std.math.maxInt(usize));`
   The woken threads will re-check set.in_use and get EIDRM.

5. In `semctl` SETVAL handler (lines 296-310): After setting the value, wake one thread:
   `set.wait_queue.wakeUp(1);`
   A thread waiting for the semaphore value to change should be woken to re-check.

6. After applying semop operations that INCREMENT a semaphore value, wake threads:
   After the apply loop, call `set.wait_queue.wakeUp(set.wait_queue.count);` to wake all waiters (they need to re-check if their operations can now proceed).

2. Add two wait queues to `MsgQueue` struct:

3. Rewrite `msgsnd` blocking path (lines 185-193). When the queue is full and IPC_NOWAIT is NOT set:
   - Release msg_lock, free the already-allocated data buffer, and block: `sched.waitOn(&q.send_wait_queue, held)`.
   - After wakeup, retry from the beginning (re-validate queue, re-copy data from userspace, re-check capacity).
   - IMPORTANT: Data was already copied and allocated before the capacity check. Must free it before blocking, then re-allocate after wakeup.
   - The restructured msgsnd needs a retry loop similar to semop.

   Actually, simpler approach: Move the capacity check BEFORE the data copy. This avoids allocating and freeing on each retry:
   ```
   pub fn msgsnd(id: u32, msgp: usize, msgsz: usize, msgflg: i32, proc: *const process.Process) !void {
       if (msgsz > MSGMAX) return error.EINVAL;
       while (true) {
           const held = msg_lock.acquire();
           // Validate queue...
           // Check capacity FIRST
           if (q.qbytes + msgsz > q.qbytes_max) {
               if ((msgflg & IPC_NOWAIT) != 0) { held.release(); return error.EAGAIN; }
               sched.waitOn(&q.send_wait_queue, held); continue;
           }
           // NOW copy data from userspace (lock released during copy is OK since we checked capacity)
           held.release();
           // Copy message... allocate... then re-acquire lock and double-check
           const held2 = msg_lock.acquire();
           // Re-validate and re-check capacity (TOCTOU)
           // If still OK, append message
           // Wake recv waiters
           held2.release();
           return;
       }
   }
   ```

   Wait -- we cannot do userspace copies while holding a spinlock (potential page fault). But we also cannot release the lock and re-acquire without TOCTOU. The simpler approach: copy userspace data OUTSIDE the lock (before the loop), then inside the lock just do the capacity check and enqueue. This is safe because the data buffer content does not depend on queue state.

   Restructured msgsnd:
   ```
   pub fn msgsnd(...) !void {
       // Step 1: Copy from userspace (outside any lock)
       const header = UserPtr.from(msgp).readValue(MsgBufHeader) catch return error.EFAULT;
       if (header.mtype <= 0) return error.EINVAL;
       const data = heap.allocator().alloc(u8, msgsz) catch return error.ENOMEM;
       errdefer heap.allocator().free(data);
       _ = UserPtr.from(msgp + @sizeOf(MsgBufHeader)).copyToKernel(data) catch {
           heap.allocator().free(data);
           return error.EFAULT;
       };

       // Step 2: Acquire lock, check capacity, enqueue or block
       while (true) {
           const held = msg_lock.acquire();
           // Validate queue still exists...
           if (q.qbytes + msgsz > q.qbytes_max) {
               if ((msgflg & IPC_NOWAIT) != 0) { held.release(); heap.allocator().free(data); return error.EAGAIN; }
               sched.waitOn(&q.send_wait_queue, held); continue; // retry with lock re-acquired
           }
           // Allocate KernelMsg and enqueue
           const msg = heap.allocator().create(KernelMsg) catch { held.release(); heap.allocator().free(data); return error.ENOMEM; };
           msg.mtype = header.mtype; msg.data = data; msg.next = null;
           // Append to queue...
           // Wake recv waiters
           q.recv_wait_queue.wakeUp(1);
           held.release();
           return;
       }
   }
   ```

LOCK ORDERING NOTE: The current IPC lock ordering is `sem_lock` / `msg_lock` at level 2 (same as SFS alloc_lock). `sched.waitOn` releases the lock atomically before blocking, maintaining correct ordering. `WaitQueue.wakeUp` internally acquires scheduler.lock, which is at level 4. Since sem_lock/msg_lock are at level 2, calling wakeUp while holding them is safe (2 -> 4).

Implement SEM_UNDO flag tracking so that semaphore adjustments are automatically reversed when a process exits.

**1. Define SEM_UNDO tracking structure in sem.zig:**

Add a per-process undo entry struct:
```zig
const SemUndoEntry = struct {
    semid: u32,       // Semaphore set ID
    sem_num: u16,     // Semaphore number within the set
    adjustment: i32,  // Accumulated adjustment (negate to undo)
};
```

Add a per-process undo list (simple fixed-size array for MVP):
```zig
pub const MAX_SEM_UNDO = 32; // Max tracked undo entries per process

pub const SemUndoList = struct {
    entries: [MAX_SEM_UNDO]SemUndoEntry = [_]SemUndoEntry{.{ .semid = 0, .sem_num = 0, .adjustment = 0 }} ** MAX_SEM_UNDO,
    count: u32 = 0,

    /// Record an undo adjustment for a semaphore operation
    pub fn record(self: *SemUndoList, semid: u32, sem_num: u16, adj: i32) void {
        // Search for existing entry
        for (self.entries[0..self.count]) |*e| {
            if (e.semid == semid and e.sem_num == sem_num) {
                e.adjustment += adj;
                // If adjustment is now 0, could remove entry (optimization)
                return;
            }
        }
        // Add new entry
        if (self.count < MAX_SEM_UNDO) {
            self.entries[self.count] = .{ .semid = semid, .sem_num = sem_num, .adjustment = adj };
            self.count += 1;
        }
        // If full, silently drop (POSIX allows this)
    }
};
```

Add `pub fn applySemUndo(proc: *process.Process) void` to sem.zig:
```zig
pub fn applySemUndo(proc: *process.Process) void {
    const held = sem_lock.acquire();
    defer held.release();

    for (proc.sem_undo_entries[0..proc.sem_undo_count]) |entry| {
        if (entry.adjustment == 0) continue;

        const idx = ipc_perm.idToIndex(entry.semid);
        const seq = ipc_perm.idToSeq(entry.semid);
        if (idx >= SEMMNI) continue;

        const set = &sem_sets[idx];
        if (!set.in_use or set.perm.seq != seq) continue;

        const sems = set.sems orelse continue;
        if (entry.sem_num >= set.nsems) continue;

        const sem = &sems[entry.sem_num];
        // Apply undo: add adjustment to current value
        if (entry.adjustment > 0) {
            sem.semval +|= @intCast(entry.adjustment); // saturating add
        } else {
            const sub = @as(u32, @intCast(-entry.adjustment));
            if (sem.semval >= sub) {
                sem.semval -= sub;
            } else {
                sem.semval = 0; // Clamp to 0
            }
        }
        // Wake waiters since semaphore value changed
        set.wait_queue.wakeUp(set.wait_queue.count);
    }

    proc.sem_undo_count = 0;
}
```
