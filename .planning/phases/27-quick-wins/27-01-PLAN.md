  <files>src/kernel/sys/syscall/io/dir.zig, src/kernel/sys/syscall/io/root.zig, src/user/lib/syscall/io.zig, src/user/lib/syscall/root.zig, src/user/test_runner/tests/syscall/uid_gid.zig</files>
  <action>
Implement sys_fchdir (Linux syscall 81 on x86_64, 50 on aarch64) which changes the working directory using an open file descriptor instead of a path string.

**Kernel implementation** (src/kernel/sys/syscall/io/dir.zig):
Add `pub fn sys_fchdir(fd_num: usize) SyscallError!usize` after sys_chdir:
1. Cast fd_num to u32 safely (return EBADF if out of range)
2. Get the FD table via `base.getGlobalFdTable()` (same pattern as sys_getdents64)
3. Look up the FD: `table.get(fd_u32) orelse return error.EBADF`
4. Verify the FD is a directory: check if `fd.ops == &fd_mod.dir_ops`. If not, return ENOTDIR.
5. Determine the directory path from the FD. Directory FDs use `private_data` pointing to a `DirTag`. For a proper implementation:
   - Currently directory FDs only exist for "/" (initrd_root) and "/dev" (devfs_root) via VFS openFile
   - Read the DirTag from private_data: cast `fd.private_data` to `*const fd_mod.DirTag`
   - Map DirTag to path: `.initrd_root` -> "/", `.devfs_root` -> "/dev"
   - For SFS directories under /mnt, the VFS openFile would need to store the path -- but SFS directories opened via openat also use dir_ops. For SFS, check if the private_data is NOT a DirTag pointer (it could be a VFS mount-specific path or index). Given current VFS architecture, start with DirTag-based paths and add SFS directory support if needed.
   - Simpler approach: Store the path in the FD. Since directory FDs are rare and the path was validated at open time, this is safe. However, changing FD struct is heavy. Instead, walk the VFS to resolve the path from the FD's context.
   - SIMPLEST approach for MVP: If fd.ops == dir_ops, check private_data DirTag. For initrd_root set cwd to "/". For devfs_root set cwd to "/dev". For other directory FDs (SFS), we need the path -- but SFS directories create FDs differently. Check how SFS and VFS create directory FDs (search for dir_ops assignments in fs/ code). If SFS dirs use a different mechanism, handle that case.
6. Acquire proc.cwd_lock, copy the resolved path into proc.cwd, update proc.cwd_len, release lock.
7. Return 0.

**Export** (src/kernel/sys/syscall/io/root.zig):
Add: `pub const sys_fchdir = dir.sys_fchdir;`

**Userspace wrapper** (src/user/lib/syscall/io.zig):
Add `pub fn fchdir(fd: i32) SyscallError!void` that calls `primitive.syscall1(syscalls.SYS_FCHDIR, @bitCast(@as(isize, fd)))` and checks for error.

**Export wrapper** (src/user/lib/syscall/root.zig):
Add `pub const fchdir = io.fchdir;` in the appropriate section.

**Update test** (src/user/test_runner/tests/syscall/uid_gid.zig):

1. Save current cwd via syscall.getcwd()
2. Open "/" using syscall.open("/", O_RDONLY | O_DIRECTORY, 0) -- note: O_DIRECTORY = 0x10000 (check existing constants)
3. If open fails (current VFS may not support opening dirs as FDs), skip the test gracefully
4. Call syscall.fchdir(fd)
5. Verify getcwd() now returns "/"
6. Close the fd
7. Restore original cwd via syscall.chdir(saved_cwd)

Also update the test name in main.zig from "uid/gid: fchdir not implemented" to "uid/gid: fchdir basic" (the test registration name).

Note: The SYS_FCHDIR constant is already defined in uapi/syscalls (81 for x86_64, 50 for aarch64).
