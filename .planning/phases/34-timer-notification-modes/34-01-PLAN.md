<objective>
Add SIGEV_THREAD and SIGEV_THREAD_ID notification modes to the kernel-side POSIX timer infrastructure, plus the sys_gettid syscall needed for SIGEV_THREAD_ID target identification.

Purpose: Currently timer_create only accepts SIGEV_SIGNAL and SIGEV_NONE, returning EINVAL for SIGEV_THREAD (2) and SIGEV_THREAD_ID (4). This plan extends the kernel to support both modes, enabling directed signal delivery to specific threads and timer-driven notification with sigval data.

Note on SIGEV_THREAD semantics: In Linux, SIGEV_THREAD is implemented by glibc, which spawns a helper thread to catch the signal and invoke the user's callback. At the kernel level, SIGEV_THREAD is identical to SIGEV_SIGNAL -- it delivers a signal with SI_TIMER code and sigev_value. Our implementation matches this kernel-level behavior. The ROADMAP success criteria ("spawns notification thread") describes glibc behavior, not kernel behavior, and will be updated during phase execution to reflect this.

Output: Kernel accepts all four SIGEV_* notification modes. Timer expiration delivers signals to the correct target (process main thread, specific thread, or with SI_TIMER metadata). sys_gettid returns the current thread's TID.
</objective>
