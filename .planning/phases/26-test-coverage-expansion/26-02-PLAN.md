  **Step 1: Add select/epoll edge cases to io_mux.zig**

  Add these test functions at the end of `src/user/test_runner/tests/syscall/io_mux.zig`:

  - `testSelectNfdsZero`: Call select with nfds=0, all fd_sets null, timeout=0. Should return 0 immediately (no fds to watch). This tests the "empty sets" edge case.

  - `testSelectNullAllSets`: Create a pipe, call select with nfds=pipe[0]+1, but pass null for ALL three fd_set pointers, timeout=0. Should return 0 (nothing to watch).

  - `testEpollCtlDel`: Create epoll, add a pipe read-end, then call EPOLL_CTL_DEL to remove it. Then epoll_wait with timeout=0 should return 0 events (the fd was removed).

  - `testEpollCtlMod`: Create epoll, add a pipe read-end monitoring EPOLLIN. Write data. Then EPOLL_CTL_MOD to change to EPOLLOUT only. Now epoll_wait should return the fd with EPOLLOUT (pipe write-end is always writable). Actually -- MOD changes events on the same fd. Since we added read-end, after MOD to EPOLLOUT, epoll_wait should return 0 (read-end of pipe is not writable). This tests that MOD actually changes the monitored events.

  Careful: epoll_ctl MOD signature is same as ADD (pass epfd, EPOLL_CTL_MOD, fd, &new_event). Verify the constant EPOLL_CTL_MOD = 3 matches what's exported.

  - `testSelectMultipleFdsReady`: Create two pipes, write to both, set up readfds with both read-ends. Call select. Should return 2 (both ready). Verify both fdIsSet.

  **Step 2: Add memory advisory extras to memory.zig**

  Add to `src/user/test_runner/tests/syscall/memory.zig`:

  - `testMadviseDontneed`: mmap anonymous page, write data, call madvise with MADV_DONTNEED (4), then read the page. On Linux, DONTNEED means the page contents may be discarded. In zk it is likely a stub that succeeds without effect. Verify the madvise call succeeds. Read the data back -- it may or may not be zero depending on implementation. Just verify no crash.

  - `testMincoreUnmappedAddr`: Call mincore with an obviously unmapped address (e.g., 0x1000). Should return ENOMEM or EFAULT.

  **Step 3: Add resource limit edge cases to resource_limits.zig**

  Add to `src/user/test_runner/tests/syscall/resource_limits.zig`:

  - `testGetrlimitInvalidResource`: Call getrlimit with resource=999 (invalid). Should return EINVAL.

  - `testSetrlimitRaiseSoftToHard`: Get current RLIMIT_NOFILE, if soft < hard, set soft = hard. Verify it succeeds. Then restore original.

  - `testGetrlimitStack`: Call getrlimit(RLIMIT_STACK). Verify soft > 0 and soft <= hard. This exercises a resource type not yet tested in isolation.
  </action>
