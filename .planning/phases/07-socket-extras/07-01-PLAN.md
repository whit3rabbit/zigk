Purpose: This is the blocker for all Phase 7 work. Without this fix, no socket syscall can execute without crashing the kernel. The root cause is that `initSyscallOnly()` is only called inside `initNetwork()` AFTER RSDP/PCI checks that may return early, leaving the IrqLock uninitialized.

Output: Kernel that can execute socket syscalls without panic. Existing 8 socket tests pass or skip gracefully.
</objective>

<task type="auto">
  <name>Task 1: Fix IrqLock initialization ordering in initNetwork</name>
  <files>src/kernel/core/init_hw.zig</files>
  <action>
Move the `net.transport.initSyscallOnly(heap.allocator())` call to the VERY BEGINNING of `initNetwork()`, before the RSDP check on line 386. Currently it is on line 438, which is unreachable when:
- `rsdp_address == 0` (returns on line 388)
- `pci.initFromAcpi()` fails (returns on line 393)

The fix is simple -- move lines 435-439 (the socket subsystem init block) to immediately after the console log on line 383. The socket subsystem does NOT depend on RSDP, PCI, or any NIC driver. It only needs a heap allocator, which is available at this point.

After the move, the function should look like:
```zig
pub fn initNetwork() void {
    console.print("\n");
    console.info("Initializing network subsystem...", .{});

    // Initialize Socket Subsystem FIRST (before PCI/NIC).
    // Socket syscalls need lock initialization even without a network stack.
    // This MUST happen before any early return, otherwise IrqLock panics.
    net.transport.initSyscallOnly(heap.allocator());
    console.debug("[NETSTACK] Socket subsystem initialized (syscall-only mode)", .{});

    // 1. Get RSDP for PCI ECAM
    if (rsdp_address == 0) {
        console.warn("RSDP not found, network disabled.", .{});
        return;
    }
    // ... rest of function unchanged ...
```

Remove the old lines 435-439 where initSyscallOnly was previously called (now moved up).

Do NOT change any other logic in initNetwork. The PCI, NIC, and NETSTACK MIGRATION code stays exactly where it is.

4. Verify in test output: "Socket subsystem initialized (syscall-only mode)" appears BEFORE any "RSDP not found" or "PCI init failed" messages
  </verify>
