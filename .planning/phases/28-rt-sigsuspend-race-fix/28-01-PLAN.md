<objective>
Fix the POSIX rt_sigsuspend race condition (SIG-01) where pending signals are not delivered when the temporary mask unblocks them.

Purpose: rt_sigsuspend must atomically swap the signal mask and suspend until a signal arrives. The current implementation restores the old mask before checkSignalsOnSyscallExit runs, so signals unblocked only by the temporary mask are never delivered -- violating POSIX semantics.

Output: Race-free rt_sigsuspend with deferred mask restoration pattern, passing test on both x86_64 and aarch64.
</objective>

The race: sys_rt_sigsuspend restores the old signal mask (line 276) BEFORE dispatch_syscall calls checkSignalsOnSyscallExit (table.zig:190). Signals unblocked only by the temporary mask get re-blocked before delivery.

Fix using the deferred mask restoration pattern (same approach as Linux kernel):

1. In `src/kernel/proc/thread.zig`, add two fields to the Thread struct after `sigmask` (around line 111):
   ```zig
   /// Saved signal mask for rt_sigsuspend deferred restoration.
   /// When has_saved_sigmask is true, checkSignalsOnSyscallExit restores
   /// this mask after signal delivery (not before).
   saved_sigmask: uapi.signal.SigSet = 0,
   has_saved_sigmask: bool = false,
   ```

2. In `src/kernel/sys/syscall/process/signals.zig`, rewrite sys_rt_sigsuspend (lines 231-280):
   - Keep: sigsetsize validation, reading new_mask from userspace, getting current_thread, saving old_mask
   - Keep: Setting new mask and clearing SIGKILL/SIGSTOP from it
   - Change: Instead of restoring old mask before return, save it for deferred restoration:
     ```zig
     // Save old mask for deferred restoration by checkSignalsOnSyscallExit.
     // The temp mask stays active so signal delivery can see unblocked signals.
     current_thread.saved_sigmask = old_mask;
     current_thread.has_saved_sigmask = true;
     ```
   - Keep: The pending signal check and conditional block() (lines 261-267) -- this optimization is still correct
   - Remove: The line `current_thread.sigmask = old_mask;` (line 276) -- this is the bug
   - Keep: Return error.EINTR

3. In `src/kernel/proc/signal.zig`, update checkSignalsOnSyscallExit (starts at line 316):
   - After the existing signal delivery logic (the entire function body), add mask restoration at the end, AFTER all signal delivery is complete:
     ```zig
     // Restore saved signal mask from rt_sigsuspend.
     // This runs AFTER signal delivery so the temporary mask is active
     // during signal handler setup (handler sees unblocked signals).
     if (current_thread.has_saved_sigmask) {
         current_thread.sigmask = current_thread.saved_sigmask;
         current_thread.has_saved_sigmask = false;
     }
     ```
   - IMPORTANT: This restoration must happen at the very end of checkSignalsOnSyscallExit, after any signal handler has been set up via setupSignalFrameForSyscall. The signal handler's rt_sigreturn will restore the mask from ucontext.sigmask (which contains the old mask saved in setupSignalFrameForSyscall's ucontext.sigmask field at line 447 -- this already copies current_thread.sigmask which at that point is still the temp mask). Wait -- that means ucontext.sigmask will have the temp mask, not the old mask. We need to fix this too.
   - CORRECTION: In setupSignalFrameForSyscall (line 411), the ucontext.sigmask is set to current_thread.sigmask. When rt_sigsuspend is active, this is the TEMP mask. But rt_sigreturn restores this into current_thread.sigmask (signals.zig line 367). We need the ucontext to save the ORIGINAL mask so rt_sigreturn restores correctly.
   - Fix in setupSignalFrameForSyscall: When building ucontext, check has_saved_sigmask:
     ```zig
     .sigmask = if (current_thread.has_saved_sigmask) current_thread.saved_sigmask else current_thread.sigmask,
     ```
   - And the .oldmask field in mcontext (line 447) should also use the saved mask:
     ```zig
     .oldmask = if (current_thread.has_saved_sigmask) current_thread.saved_sigmask else current_thread.sigmask,
     ```
   - After setting up the signal frame, clear has_saved_sigmask (the handler's rt_sigreturn will handle restoration via ucontext.sigmask):
     ```zig
     // If rt_sigsuspend saved a mask, clear the flag -- rt_sigreturn
     // will restore the original mask from ucontext.sigmask.
     if (current_thread.has_saved_sigmask) {
         current_thread.has_saved_sigmask = false;
     }
     ```
   - Also update checkSignals (interrupt path, line 95) with the same saved mask logic for the interrupt-based signal delivery path. In setupSignalFrame (line 138), apply the same ucontext.sigmask and .oldmask fix. The interrupt path is less likely to hit rt_sigsuspend (since rt_sigsuspend returns via syscall exit, not interrupt), but for correctness add the check.

Summary of the flow after fix:
- sys_rt_sigsuspend: sets temp mask, saves old mask in saved_sigmask, blocks or skips, returns EINTR (does NOT restore mask)
- dispatch_syscall: writes EINTR to frame, calls checkSignalsOnSyscallExit
- checkSignalsOnSyscallExit: sees unblocked signal (temp mask active), delivers it via setupSignalFrameForSyscall (which saves old mask into ucontext.sigmask), clears has_saved_sigmask
- Signal handler runs in userspace with temp mask (signal blocked by SA_NODEFER)
- rt_sigreturn: restores original mask from ucontext.sigmask
- If NO signal was pending: checkSignalsOnSyscallExit does nothing for signal delivery, but restores saved_sigmask at the end (the code added in step 3)

```zig
// Test 17: rt_sigsuspend unblocks signal and delivers it (returns EINTR)
pub fn testRtSigsuspendBasic() !void {
    const SIGUSR1 = 10;
    const SIG_BLOCK = 0;
    const SIG_UNBLOCK = 1;

    // Reset flag
    sigusr1_called = false;

    // Install SIGUSR1 handler
    var act = syscall.SigAction{
        .handler = @intFromPtr(&handleSigusr1),
        .flags = 0,
        .restorer = 0,
        .mask = 0,
    };
    try syscall.sigaction(SIGUSR1, &act, null);

    // Block SIGUSR1
    var block_set: syscall.SigSet = 0;
    syscall.uapi.signal.sigaddset(&block_set, SIGUSR1);
    try syscall.sigprocmask(SIG_BLOCK, &block_set, null);

    // Send SIGUSR1 to self (it becomes pending because it's blocked)
    try syscall.kill(syscall.getpid(), SIGUSR1);

    // Verify signal is pending but NOT delivered yet
    if (sigusr1_called) {
        // Cleanup
        _ = syscall.sigprocmask(SIG_UNBLOCK, &block_set, null) catch {};
        return error.TestFailed;
    }

    // Call rt_sigsuspend with a mask that UNBLOCKS SIGUSR1.
    // This should atomically swap the mask and deliver the pending signal.
    // The empty mask (0) unblocks everything.
    var suspend_mask: syscall.SigSet = 0; // Unblock all signals

    // rt_sigsuspend always returns EINTR (per POSIX) after signal delivery
    const result = syscall.rt_sigsuspend(&suspend_mask);
    if (result) |_| {
        // Should not succeed -- rt_sigsuspend always returns EINTR
        _ = syscall.sigprocmask(SIG_UNBLOCK, &block_set, null) catch {};
        return error.TestFailed;
    } else |err| {
        // EINTR is the expected return (maps to Interrupted in userspace)
        if (err != error.Interrupted) {
            _ = syscall.sigprocmask(SIG_UNBLOCK, &block_set, null) catch {};
            if (err == error.NotImplemented) return error.SkipTest;
            return error.TestFailed;
        }
    }

    // The signal handler should have been called during rt_sigsuspend
    if (!sigusr1_called) {
        _ = syscall.sigprocmask(SIG_UNBLOCK, &block_set, null) catch {};
        return error.TestFailed;
    }

    // Cleanup: unblock SIGUSR1
    _ = syscall.sigprocmask(SIG_UNBLOCK, &block_set, null) catch {};
}
```

Key design choices:
- Block SIGUSR1 first, send it (becomes pending), then call rt_sigsuspend with an empty mask that unblocks it
- The old test was skipped because this exact scenario deadlocked (signal never delivered)
- Check that sigusr1_called is true after rt_sigsuspend returns
- Check that rt_sigsuspend returns EINTR (the POSIX-mandated return)
- Note: EINTR maps to error.Interrupted in the userspace errno mapping (check this -- it might be a different error name)
