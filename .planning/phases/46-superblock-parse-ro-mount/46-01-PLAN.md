  <name>Task 1: Create VirtIO-SCSI BlockDevice adapter</name>
  <files>src/drivers/virtio/scsi/block_adapter.zig, src/drivers/virtio/scsi/root.zig</files>
  <action>
Create `src/drivers/virtio/scsi/block_adapter.zig` that bridges `BlockDevice` vtable reads to `VirtioScsiController.readBlocks` calls.

**ScsiLunCtx struct:**
- `controller: *root.VirtioScsiController` -- pointer to the global controller
- `lun_idx: u8` -- which LUN this adapter represents
- `total_sectors: u64` -- capacity in 512-byte sectors (converted from LUN total_blocks * block_size / 512)

**scsiReadSectors function:**
- Signature: `fn scsiReadSectors(ctx: *anyopaque, lba: u64, count: u32, buf: []u8) BlockDeviceError!void`
- Cast ctx to `*ScsiLunCtx`
- The VirtIO-SCSI `readBlocks` uses the LUN's native block_size (typically 512), while BlockDevice uses SECTOR_SIZE (512). If they match, pass through directly. If the LUN block_size is larger (e.g., 4096), convert: `native_lba = lba * SECTOR_SIZE / lun_block_size`, `native_count = (count * SECTOR_SIZE + lun_block_size - 1) / lun_block_size`. Use `std.math.mul` for overflow safety on all arithmetic.
- Call `controller.readBlocks(lun_idx, native_lba, native_count, buf)` -- on error, return `error.IOError`
- Discard the return value (bytes transferred) since readSectors is void on success

**scsiWriteSectors function:**
- Same pattern as read but calls `controller.writeBlocks` (check if writeBlocks exists; if not, return `error.IOError` unconditionally since Phase 46 is read-only and write is not needed yet)

**pub fn asBlockDevice:**
- Signature: `pub fn asBlockDevice(controller: *root.VirtioScsiController, lun_idx: u8) !BlockDevice`
- Call `controller.getLun(lun_idx)` -- return `error.IOError` if null or not active
- Heap-allocate a `ScsiLunCtx` with `heap.allocator().create(ScsiLunCtx)`, `errdefer` destroy on failure
- Compute `total_sectors = lun_info.total_blocks * (lun_info.block_size / SECTOR_SIZE)` using `std.math.mul` for overflow safety
- Return `BlockDevice{ .ctx = ctx, .readSectorsFn = scsiReadSectors, .writeSectorsFn = scsiWriteSectors, .sector_count = total_sectors, .sector_size = lun_info.block_size }`

**Imports:**
- `const root = @import("root.zig");`
- `const fs = @import("fs");` (for `fs.block_device.BlockDevice`, `fs.block_device.BlockDeviceError`, `fs.block_device.SECTOR_SIZE`)
- `const heap = @import("heap");`
- `const std = @import("std");`

Add `pub const block_adapter = @import("block_adapter.zig");` to `src/drivers/virtio/scsi/root.zig` alongside the existing sub-module exports (after the `pub const irq` line near line 33).

**Important: Do NOT add writeBlocks if it does not exist.** Check the controller API. If `writeBlocks` is missing, the write adapter should just return `error.IOError`. Phase 46 does not need writes.

<task type="auto">
  <name>Task 2: Create ext2 mount module with superblock parse, BGDT read, and VFS adapter</name>
  <files>src/fs/ext2/mount.zig, src/fs/root.zig</files>
  <action>
Create `src/fs/ext2/mount.zig` implementing the ext2 filesystem state, superblock parsing, BGDT read, and VFS FileSystem adapter.

**Ext2Fs struct:**
```
pub const Ext2Fs = struct {
    dev: BlockDevice,
    superblock: types.Superblock,
    block_groups: []types.GroupDescriptor,
    block_size: u32,
    sectors_per_block: u32,
    group_count: u32,
    inode_size: u16,
};
```
The `block_groups` slice is heap-allocated (freed in unmount).

**parseSuperblock(dev: BlockDevice) !types.Superblock:**
1. Declare `var sb_buf: [1024]u8 align(4) = @splat(0);` -- zero-init, 4-byte aligned for safe @ptrCast
2. Call `dev.readSectors(types.SUPERBLOCK_LBA, 2, &sb_buf)` -- 2 sectors = 1024 bytes. Map BlockDeviceError to a local error (wrap: `catch return error.IOError`)
3. Copy via: `const sb: types.Superblock = @as(*const types.Superblock, @ptrCast(@alignCast(&sb_buf))).*;`
4. Validate magic: `if (sb.s_magic != types.EXT2_MAGIC)` -- log with `console.err` and return `error.InvalidSuperblock`
5. Validate s_blocks_per_group != 0 and s_inodes_per_group != 0 (prevent division by zero in groupCount) -- return `error.InvalidSuperblock` if either is zero
6. Check INCOMPAT: `const unsupported = sb.s_feature_incompat & ~types.SUPPORTED_INCOMPAT; if (unsupported != 0)` -- log hex value, return `error.NotSupported`
7. Log success: `console.info("ext2: superblock OK, magic=0xEF53, block_size={d}, blocks={d}, groups={d}", .{sb.blockSize(), sb.s_blocks_count, sb.groupCount()});`
8. Return sb

**readBgdt(dev: BlockDevice, sb: types.Superblock, allocator: std.mem.Allocator) ![]types.GroupDescriptor:**
1. Compute `group_count = sb.groupCount()` -- if 0, return `error.InvalidSuperblock`
2. Compute `block_size = sb.blockSize()`, `sectors_per_block = block_size / SECTOR_SIZE` (using `std.math.divExact` or just `/` since block_size is always a power of 2 >= 1024)
3. BGDT block = `std.math.add(u32, sb.s_first_data_block, 1) catch return error.InvalidSuperblock`
4. BGDT LBA = `std.math.mul(u64, @as(u64, bgdt_block), @as(u64, sectors_per_block)) catch return error.InvalidSuperblock`
5. BGDT bytes = `std.math.mul(usize, group_count, @sizeOf(types.GroupDescriptor)) catch return error.InvalidSuperblock`
6. BGDT sectors = `(bgdt_bytes + SECTOR_SIZE - 1) / SECTOR_SIZE`
7. Allocate aligned raw buffer: `const raw_buf = try allocator.alignedAlloc(u8, 4, bgdt_sectors * SECTOR_SIZE); errdefer allocator.free(raw_buf);`
8. Read: `dev.readSectors(bgdt_lba, @intCast(bgdt_sectors), raw_buf) catch return error.IOError;`
9. Allocate `groups = try allocator.alloc(types.GroupDescriptor, group_count); errdefer allocator.free(groups);`
10. Use `std.mem.bytesAsSlice(types.GroupDescriptor, raw_buf[0..bgdt_bytes])` to get typed view, `@memcpy(groups, gd_slice)`, then `allocator.free(raw_buf)`
11. Log: `console.info("ext2: BGDT OK, {d} groups, first inode_table=block {d}", .{group_count, groups[0].bg_inode_table});`
12. Return groups

**VFS adapter callbacks (all stubs for Phase 46 read-only):**

- `ext2Open(ctx: ?*anyopaque, path: []const u8, flags: u32) vfs.Error!*fd.FileDescriptor` -- Phase 46 stub: if write flags set, return `error.AccessDenied`; otherwise return `error.NotFound` (no inode resolution yet). Suppress unused params with `_ = .{ctx, path};`
- `ext2StatPath(ctx: ?*anyopaque, path: []const u8) ?FileMeta` -- return null (no inode lookup until Phase 47). Suppress unused params.
- `ext2Unmount(ctx: ?*anyopaque) void` -- cast to `*Ext2Fs`, free `self.block_groups`, then `heap.allocator().destroy(self)`.

**pub fn init(dev: BlockDevice) !vfs.FileSystem:**
1. Parse superblock on stack (no heap yet): `const sb = try parseSuperblock(dev);`
2. Read BGDT: `const groups = try readBgdt(dev, sb, heap.allocator());`
3. Determine inode_size: if `sb.s_rev_level >= types.EXT2_DYNAMIC_REV` then `sb.s_inode_size` else `types.EXT2_GOOD_OLD_INODE_SIZE`
4. Allocate Ext2Fs: `const self = try heap.allocator().create(Ext2Fs); errdefer heap.allocator().destroy(self);`
5. Fill fields: `.dev = dev, .superblock = sb, .block_groups = groups, .block_size = sb.blockSize(), .sectors_per_block = sb.blockSize() / SECTOR_SIZE, .group_count = sb.groupCount(), .inode_size = inode_size`
6. Return VFS FileSystem vtable with:
   - `.context = self`
   - `.open = ext2Open`
   - `.unmount = ext2Unmount`
   - `.unlink = null, .stat_path = ext2StatPath`
   - All optional write fields explicitly null: `.chmod = null, .chown = null, .mkdir = null, .rmdir = null, .rename = null, .rename2 = null, .truncate = null, .link = null, .symlink = null, .readlink = null, .set_timestamps = null, .getdents = null, .statfs = null`
