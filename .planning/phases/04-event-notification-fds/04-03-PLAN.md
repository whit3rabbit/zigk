Create src/kernel/sys/syscall/io/signalfd.zig following the eventfd pattern:

**SignalFdState struct:**
- sigmask: u64 -- bitmask of signals to accept (bit N-1 = signal N)
- lock: sync.Spinlock -- protects sigmask updates
- blocked_readers: ?*sched.Thread -- thread waiting for signals
- reader_woken: std.atomic.Value(bool) -- SMP wakeup flag

**Helper: filterMask(mask: u64) u64:**
- Strip SIGKILL and SIGSTOP bits: mask & ~(@as(u64, 1) << (SIGKILL - 1)) & ~(@as(u64, 1) << (SIGSTOP - 1))
- Import SIGKILL (9) and SIGSTOP (19) from uapi.process.signal (or hardcode if import path is complex -- check how signal constants are accessed from the syscall layer)

**signalfdRead (FileOps.read):**
- Return EINVAL if buf.len < @sizeOf(SignalFdSigInfo) (128 bytes)
- Get current thread via sched.getCurrentThread(). Return EINVAL if null.
- Acquire state.lock
- Check pending = thread.pending_signals & state.sigmask
- If pending == 0:
  - If O_NONBLOCK: return EAGAIN
  - Else: use yield-loop blocking (similar to timerfd read pattern -- release lock, yield, re-acquire, re-check). The signal delivery path will set pending_signals, and our yield loop will detect it on the next iteration.
- Find first set bit: sig_bit = @ctz(pending), signum = sig_bit + 1
- CRITICAL: Clear the pending bit to consume the signal. thread.pending_signals &= ~(@as(u64, 1) << @intCast(sig_bit)). This prevents the signal handler from also receiving it.
- Build SignalFdSigInfo: zero-initialize with std.mem.zeroes, set ssi_signo = @intCast(signum). For MVP, leave other fields as zero (ssi_code, ssi_pid, ssi_uid, etc. require signal queue metadata that doesn't exist yet per research doc).
- Release lock
- Copy SignalFdSigInfo to buf[0..@sizeOf(SignalFdSigInfo)] using @memcpy + std.mem.asBytes
- Return @sizeOf(SignalFdSigInfo) (128)

**Support for reading multiple signals:**
If buf.len >= 2 * @sizeOf(SignalFdSigInfo), could read multiple signals in one call. For MVP, just read one signal per read() call. This matches the common usage pattern.

**signalfdPoll (FileOps.poll):**
- Get current thread
- If requested EPOLLIN: check thread.pending_signals & state.sigmask != 0, set EPOLLIN
- No lock needed for poll (reading pending_signals is a single u64 load, sigmask is stable between updates)
- Return revents

**signalfdClose (FileOps.close):**
- Destroy SignalFdState
- Return 0

**signalfd_file_ops:** read, close, poll populated. write = null (signalfd is read-only). Others null.

**sys_signalfd4(fd_num_raw: usize, mask_ptr: usize, mask_size: usize, flags: usize) SyscallError!usize:**

NOTE: The Linux signalfd4 ABI is signalfd4(int fd, const sigset_t *mask, size_t sizemask, int flags). The fd parameter is signed: -1 means create new, >= 0 means update existing. The mask_size parameter specifies the size of the signal mask (typically 8 bytes for 64-bit systems).

- Cast fd_num_raw to isize: const fd_num: isize = @bitCast(fd_num_raw)
- Validate flags: only SFD_CLOEXEC | SFD_NONBLOCK allowed. Return EINVAL for invalid flags.
- Read mask from userspace: UserPtr.from(mask_ptr).readValue(u64). Return EFAULT on failure.
- Apply filterMask to strip SIGKILL/SIGSTOP.

If fd_num == -1 (create new):
- Allocate SignalFdState, init with filtered mask
- Allocate FileDescriptor, set ops = &signalfd_file_ops, flags = O_RDONLY | (O_NONBLOCK if SFD_NONBLOCK), private_data = state, cloexec = (flags & SFD_CLOEXEC) != 0
- Install via allocAndInstall, return fd_num

If fd_num >= 0 (update existing):
- Get FD from table. Return EBADF if not found.
- Verify fd.ops == &signalfd_file_ops (it's actually a signalfd). Return EINVAL if not.
- Get state from private_data, acquire lock, update sigmask with filtered mask, release lock.
- Return @intCast(fd_num) (the same fd number)

**sys_signalfd(fd_num_raw: usize, mask_ptr: usize, mask_size: usize) SyscallError!usize:**
- Delegate to sys_signalfd4(fd_num_raw, mask_ptr, mask_size, 0)

NOTE on syscall argument count: signalfd4 takes 4 args (fd, mask, sizemask, flags). signalfd takes 3 args (fd, mask, sizemask). Check how the dispatch table handles different argument counts. The sys_* functions take usize args positionally.

**Export in io/root.zig:**
