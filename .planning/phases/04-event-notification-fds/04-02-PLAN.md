**Design decision: Polling-based expiration instead of TimerWheel integration.**

The TimerWheel uses IoRequest which has complex lifecycle management. For MVP, use a simpler approach: timerfd stores the absolute expiration time. On read(), compare current clock time against stored expiration. If expired, calculate how many intervals have passed and return the count. This avoids timer wheel callback complexity while providing correct behavior.

**TimerFdState struct:**
- clockid: i32 -- CLOCK_REALTIME or CLOCK_MONOTONIC
- lock: sync.Spinlock
- armed: bool -- is timer currently set?
- next_expiry_ns: u64 -- absolute nanosecond time of next expiration
- interval_ns: u64 -- repetition interval in ns (0 = one-shot)
- expiry_count: u64 -- accumulated expirations not yet read (updated on read/poll)
- blocked_readers: ?*sched.Thread
- reader_woken: std.atomic.Value(bool)

**Helper: getClockNanoseconds(clockid: i32) u64:**
- Look at how clock_gettime is implemented in the kernel. Find the function that returns current time for a given clock ID.
- For CLOCK_MONOTONIC: use the kernel's monotonic clock source (likely hal.time or similar)
- For CLOCK_REALTIME: use wall clock time
- Convert to nanoseconds (sec * 1_000_000_000 + nsec)

**Helper: timespecToNanoseconds(ts: uapi.io.timerfd.TimeSpec) u64:**
- Return @as(u64, @intCast(ts.tv_sec)) * 1_000_000_000 + @as(u64, @intCast(ts.tv_nsec))

**Helper: nanosecondsToTimespec(ns: u64) uapi.io.timerfd.TimeSpec:**
- Return .{ .tv_sec = @intCast(ns / 1_000_000_000), .tv_nsec = @intCast(ns % 1_000_000_000) }

**Helper: updateExpiryCount(state: *TimerFdState) void:**
Called before reading expiry_count. Checks if timer has expired and updates count:
- If not armed, return
- Get current time for state.clockid
- If current_time >= next_expiry_ns:
  - If interval_ns > 0 (periodic): count how many intervals have elapsed = (current_time - next_expiry_ns) / interval_ns + 1. Advance next_expiry_ns by count * interval_ns.
  - If interval_ns == 0 (one-shot): count = 1, disarm timer (armed = false)
  - Add count to expiry_count

**timerfdRead (FileOps.read):**
- Return EINVAL if buf.len < 8
- Acquire lock
- Call updateExpiryCount(state)
- If expiry_count == 0: if O_NONBLOCK return EAGAIN. Else: compute time until next expiry, release lock, use sched.yield() in a loop checking expiry (similar to epoll_wait blocking pattern from scheduling.zig -- NOT the pipe blocking pattern since there is no writer to wake us). Re-acquire lock after expiry, call updateExpiryCount again.
- Read expiry_count, reset to 0
- Release lock
- Copy u64 count to buf[0..8]
- Return 8

NOTE on blocking: Since we are not using the TimerWheel callback, the blocking read approach should use a yield loop similar to sys_epoll_wait (which calls sched.yield() in a loop with timeout checking). The scheduler tick (10ms) provides natural wakeup points. This is acceptable for MVP since timer granularity is already 1-10ms.

**timerfdPoll (FileOps.poll):**
- Acquire lock (brief, to update count)
- Call updateExpiryCount(state)
- If expiry_count > 0 and EPOLLIN requested: set EPOLLIN
- Release lock
- Return revents

**timerfdClose (FileOps.close):**
- Destroy TimerFdState
- Return 0

**timerfd_file_ops:** read, close, poll populated. write = null (timerfds are not writable). Others null.

**sys_timerfd_create(clockid: usize, flags: usize) SyscallError!usize:**
- Validate clockid: must be CLOCK_REALTIME (0) or CLOCK_MONOTONIC (1). Accept CLOCK_BOOTTIME (7) mapped to CLOCK_MONOTONIC. Return EINVAL for others.
- Validate flags: only TFD_CLOEXEC | TFD_NONBLOCK allowed. Return EINVAL for others.
- Allocate TimerFdState, initialize with clockid, armed=false
- Allocate FileDescriptor, set ops = &timerfd_file_ops, flags = O_RDONLY | (O_NONBLOCK if TFD_NONBLOCK)
- Install via allocAndInstall, return fd_num

**sys_timerfd_settime(fd_num: usize, flags: usize, new_value_ptr: usize, old_value_ptr: usize) SyscallError!usize:**
- Get FD from table, verify it has timerfd_file_ops (return EINVAL if not a timerfd)
- Read ITimerSpec from userspace at new_value_ptr using UserPtr.readValue
- Acquire state lock
- If old_value_ptr != 0: compute current state as ITimerSpec (time remaining until next expiry as it_value, current interval as it_interval), write to userspace via UserPtr.writeValue
- Call updateExpiryCount first to get accurate state for old_value
- If new it_value is zero (tv_sec == 0 and tv_nsec == 0): disarm timer (armed = false), reset expiry_count to 0
- Else: calculate next_expiry_ns. If TFD_TIMER_ABSTIME flag set, use it_value directly as absolute time. Otherwise, add it_value to current clock time. Set interval_ns from it_interval. Set armed = true, reset expiry_count to 0.
- Wake blocked_readers if any (timer settings changed)
- Release lock
- Return 0

**sys_timerfd_gettime(fd_num: usize, curr_value_ptr: usize) SyscallError!usize:**
- Get FD, verify timerfd
- Acquire lock
- Call updateExpiryCount
- Compute ITimerSpec: it_value = time remaining (next_expiry_ns - now_ns, clamped to 0 if past), it_interval = interval_ns as timespec. If not armed, both are zero.
- Write to userspace
- Release lock
- Return 0

**Verifying the FD is a timerfd:** Compare fd.ops == &timerfd_file_ops. This is how the kernel distinguishes FD types.
