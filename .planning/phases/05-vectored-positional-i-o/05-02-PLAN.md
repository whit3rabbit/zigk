Add RWF_* flag constants at module scope in read_write.zig (after the Iovec definition created in plan 01):
```zig
// RWF flags for preadv2/pwritev2
const RWF_HIPRI: u32 = 0x00000001;
const RWF_DSYNC: u32 = 0x00000002;
const RWF_SYNC: u32 = 0x00000004;
const RWF_NOWAIT: u32 = 0x00000008;
const RWF_APPEND: u32 = 0x00000010;
const RWF_SUPPORTED: u32 = RWF_HIPRI | RWF_DSYNC | RWF_SYNC | RWF_NOWAIT | RWF_APPEND;
```

**sys_preadv2(fd_num: usize, bvec_ptr: usize, count: usize, offset_lo: usize, offset_hi: usize, flags: usize) SyscallError!usize:**
Note: Linux ABI passes offset as two 32-bit halves on some architectures. For simplicity on 64-bit, combine: `const offset: i64 = @bitCast((@as(u64, offset_hi) << 32) | @as(u64, offset_lo));`
- Validate flags: if (flags & ~RWF_SUPPORTED) != 0 return EOPNOTSUPP
- If RWF_HIPRI is set, return EOPNOTSUPP (stub -- no polling I/O infrastructure)
- If offset == -1: delegate to sys_readv logic (use current file position). This is the "use current position" mode.
- If offset >= 0: delegate to sys_preadv logic with the given offset
- RWF_NOWAIT: for MVP, return EAGAIN always (our I/O is synchronous). This is correct behavior -- if caller cannot guarantee non-blocking, EAGAIN is valid.
- RWF_DSYNC/RWF_SYNC: accept but ignore (we have no write-back cache to flush)
- RWF_APPEND: not valid for reads, return EINVAL if set

Actually, the simpler approach: preadv2 with flags=0 and offset >= 0 is identical to preadv. With offset=-1, identical to readv. So refactor: extract the core vectored-read logic into a helper `do_readv_locked` that takes the fd and kvecs, then:
- sys_readv calls it directly
- sys_preadv wraps it with seek/restore
- sys_preadv2 checks flags, handles offset=-1 vs positive, delegates appropriately

**sys_pwritev2(fd_num: usize, bvec_ptr: usize, count: usize, offset_lo: usize, offset_hi: usize, flags: usize) SyscallError!usize:**
- Same flag validation as preadv2
- RWF_APPEND: if set, seek to end before writing (O_APPEND behavior for this call only)
- RWF_HIPRI: return EOPNOTSUPP
- RWF_NOWAIT: return EAGAIN
- RWF_DSYNC/RWF_SYNC: accept but ignore
- offset == -1: delegate to writev logic
- offset >= 0: delegate to pwritev logic

Similarly extract `do_writev_locked` helper.

**sys_sendfile(out_fd_num: usize, in_fd_num: usize, offset_ptr: usize, count: usize) SyscallError!usize:**
- Get both fds from table. Check out_fd isWritable(), in_fd isReadable() and has ops.seek (return EINVAL if not seekable -- non-regular files cannot be sendfile sources)
- Check out_fd flags: if (out_fd.flags & O_APPEND) != 0 return EINVAL
- If offset_ptr != 0: read the offset from userspace (UserPtr), use it as read position for in_fd, update it after transfer. Do NOT modify in_fd.position.
- If offset_ptr == 0: use in_fd.position as read position, update in_fd.position after transfer.
- Transfer loop using kernel buffer:
  - Allocate 64KB kernel buffer from heap
  - Loop while total_sent < count:
    - chunk = min(count - total_sent, 64 * 1024)
    - Acquire in_fd.lock, save position if needed, seek to read_offset, read chunk via do_read_locked, restore position if offset_ptr != 0, release
    - Acquire out_fd.lock, write chunk via do_write_locked, release
    - If read returned 0 (EOF), break
    - Accumulate total_sent, advance read_offset
    - If short write, break
  - Free kernel buffer
- If offset_ptr != 0, write updated offset back to userspace
- Return total_sent
