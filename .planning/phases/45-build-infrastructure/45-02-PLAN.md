  <name>Task 1: Create BlockDevice abstraction</name>
  <files>src/fs/block_device.zig</files>
  <action>
Create `src/fs/block_device.zig` with a driver-portable block device interface.

The interface must:
1. Define `BlockDeviceError = error{ IOError, InvalidLba, DeviceNotReady, BufferTooSmall }`.
2. Define `pub const SECTOR_SIZE: usize = 512`.
3. Define `pub const BlockDevice = struct { ... }` with these fields:
   - `ctx: *anyopaque` -- opaque driver context (e.g., AHCI port pointer, VirtIO-SCSI controller+LUN)
   - `readSectorsFn: *const fn (ctx: *anyopaque, lba: u64, count: u32, buf: []u8) BlockDeviceError!void` -- raw driver read
   - `writeSectorsFn: *const fn (ctx: *anyopaque, lba: u64, count: u32, buf: []const u8) BlockDeviceError!void` -- raw driver write
   - `sector_count: u64` -- total device capacity in 512-byte sectors
   - `sector_size: u32` -- physical sector size (usually 512, but VirtIO-SCSI can report 4096)

4. Add convenience methods on BlockDevice:
   - `pub fn readSectors(self: BlockDevice, lba: u64, count: u32, buf: []u8) BlockDeviceError!void` that validates `buf.len >= @as(usize, count) * SECTOR_SIZE` and `lba + count <= self.sector_count` using `std.math.add` for overflow checking, then delegates to `self.readSectorsFn`.
   - `pub fn writeSectors(self: BlockDevice, lba: u64, count: u32, buf: []const u8) BlockDeviceError!void` -- same validation, delegates to `self.writeSectorsFn`.

```zig
const std = @import("std");

pub const BlockDeviceError = error{
    IOError,
    InvalidLba,
    DeviceNotReady,
    BufferTooSmall,
};

pub const SECTOR_SIZE: usize = 512;

pub const BlockDevice = struct {
    ctx: *anyopaque,
    readSectorsFn: *const fn (ctx: *anyopaque, lba: u64, count: u32, buf: []u8) BlockDeviceError!void,
    writeSectorsFn: *const fn (ctx: *anyopaque, lba: u64, count: u32, buf: []const u8) BlockDeviceError!void,
    sector_count: u64,
    sector_size: u32,

    pub fn readSectors(self: BlockDevice, lba: u64, count: u32, buf: []u8) BlockDeviceError!void {
        const needed = std.math.mul(usize, count, SECTOR_SIZE) catch return error.InvalidLba;
        if (buf.len < needed) return error.BufferTooSmall;
        const end_lba = std.math.add(u64, lba, count) catch return error.InvalidLba;
        if (end_lba > self.sector_count) return error.InvalidLba;
        return self.readSectorsFn(self.ctx, lba, count, buf);
    }

    pub fn writeSectors(self: BlockDevice, lba: u64, count: u32, buf: []const u8) BlockDeviceError!void {
        const needed = std.math.mul(usize, count, SECTOR_SIZE) catch return error.InvalidLba;
        if (buf.len < needed) return error.BufferTooSmall;
        const end_lba = std.math.add(u64, lba, count) catch return error.InvalidLba;
        if (end_lba > self.sector_count) return error.InvalidLba;
        return self.writeSectorsFn(self.ctx, lba, count, buf);
    }
};
```

This is intentionally minimal. No heap allocator, no readBytes helper, no driver adapters yet. Adapters for AHCI and VirtIO-SCSI will be added in Phase 46 when the ext2 filesystem actually needs to use them. Phase 45 only defines the interface contract.

Note: Use `SECTOR_SIZE` (512) for the validation arithmetic regardless of `self.sector_size`. The LBA addressing and count are always in 512-byte units. The `sector_size` field is informational (tells callers the physical sector alignment for performance).
