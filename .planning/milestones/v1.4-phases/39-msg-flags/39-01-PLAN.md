<objective>
Add MSG_PEEK and MSG_DONTWAIT flag support to the recv path for both TCP and UDP sockets.

Purpose: Protocol libraries (e.g., DNS resolvers, HTTP parsers) rely on MSG_PEEK to inspect incoming data without consuming it and MSG_DONTWAIT for per-call non-blocking behavior. Without these flags, any userspace code passing them to recv/recvfrom silently gets default behavior because the kernel ignores the flags parameter.

Output: Flag constants defined, peek functions for TCP and UDP, flags threaded through the recv call chain in both sys_recvfrom and sys_recvmsg, userspace wrapper with flags parameter.
</objective>

pub fn recvfromFlags(fd: i32, buf: []u8, flags: u32, src_addr: ?*SockAddrIn) SyscallError!usize {
    var addrlen: u32 = @sizeOf(SockAddrIn);
    const src_addr_ptr: usize = if (src_addr) |a| @intFromPtr(a) else 0;
    const addrlen_ptr: usize = if (src_addr != null) @intFromPtr(&addrlen) else 0;
    const ret = primitive.syscall6(
        syscalls.SYS_RECVFROM,
        @bitCast(@as(isize, fd)),
        @intFromPtr(buf.ptr),
        buf.len,
        flags,
        src_addr_ptr,
        addrlen_ptr,
    );
    if (primitive.isError(ret)) return primitive.errorFromReturn(ret);
    return ret;
}
