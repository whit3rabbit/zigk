The key insight: `deliverSignalToThreadWithInfo` in signals.zig already calls `sched.unblock(target)` when a signal is delivered to a blocked thread (line 741-743). This means a signalfd reader that is Blocked will be woken immediately when a signal arrives. The only issue is the stale WaitQueue entry.

Changes to `signalfdRead` in signalfd.zig:

1. Replace the blocking section. The current code:
```zig
sched.waitOnWithTimeout(&state.wait_queue, held, 10, null);
```
Replace with `sched.waitOn(&state.wait_queue, held)` for indefinite blocking.

2. After waking (the line after waitOn returns), the thread may have been woken by either:
   - `sched.unblock()` from signal delivery (thread still in wait_queue - stale entry)
   - `state.wait_queue.wakeUp()` from signalfdClose (thread properly removed from queue)

   To handle the stale entry from the first case, add cleanup at the TOP of the loop body, right after re-acquiring `state.lock`:
   ```zig
   // Remove stale wait queue entry in case we were woken by
   // sched.unblock() (signal delivery) rather than wakeUp() (close).
   // removeThread is safe to call when not in queue (returns false).
   _ = state.wait_queue.removeThread(current);
   ```

3. The full revised loop structure should be:
```zig
while (true) {
    if (state.closed.load(.acquire)) return Errno.EBADF.toReturn();

    const held = state.lock.acquire();

    // Clean up any stale wait queue entry from previous iteration
    // (happens when woken by sched.unblock from signal delivery
    // rather than by wait_queue.wakeUp from close)
    _ = state.wait_queue.removeThread(current);

    // Check for pending signals in our mask
    const pending = @atomicLoad(u64, &current.pending_signals, .acquire) & state.sigmask;

    if (pending != 0) {
        // ... existing signal consumption code (unchanged) ...
    }

    // No signals pending - check nonblock
    if ((fd.flags & fd_mod.O_NONBLOCK) != 0) {
        held.release();
        return Errno.EAGAIN.toReturn();
    }

    // Block indefinitely until woken by signal delivery or close.
    // Signal delivery path (deliverSignalToThreadWithInfo) calls
    // sched.unblock(target) which transitions this thread from
    // Blocked to Ready immediately. Close path calls
    // state.wait_queue.wakeUp() which properly pops and readies.
    sched.waitOn(&state.wait_queue, held);
    // held is released by waitOn before blocking
}
```

4. Remove the TODO comment about integrating with signal delivery (it is now done).

Do NOT change signalfdClose, signalfdPoll, sys_signalfd4, or sys_signalfd -- they remain the same.

Where the `user_siginfo` SigInfoT is constructed (for SA_SIGINFO handlers), add SIGSYS-specific field mapping. The Linux siginfo_t layout for SIGSYS uses the _sigsys union fields at specific offsets. For our simplified SigInfoT, we can encode si_syscall in `si_value_int` and si_arch in `si_status` (or use the _pad area). The cleanest approach given our SigInfoT layout:

- `si_value_int` = `@intCast(si_data.syscall_nr)` (the offending syscall number)

A simpler approach: just populate si_value_int with syscall_nr and si_value_ptr with instruction_pointer. This is sufficient for signalfd (which has dedicated ssi_syscall/ssi_arch fields, not available in our struct) and for SA_SIGINFO handlers (which can read si_value).

```zig
/// Test: signalfd read wakes immediately (no 10ms polling delay)
/// Verifies SIG-03: signal delivery triggers instant wakeup of blocked signalfd reader
pub fn testSignalfdDirectWakeup() !void {
    // Block SIGUSR1
    var mask: u64 = @as(u64, 1) << (SIGUSR1 - 1);
    var oldmask: u64 = 0;
    try syscall.sigprocmask(@intCast(SIG_BLOCK), @ptrCast(&mask), @ptrCast(&oldmask));
    defer {
        _ = syscall.sigprocmask(@intCast(SIG_SETMASK), @ptrCast(&oldmask), null) catch {};
    }

    // Create blocking signalfd (no SFD_NONBLOCK)
    const fd = try syscall.signalfd4(-1, &mask, 0);
    defer { syscall.close(fd) catch {}; }

    // Record time before signal+read
    var ts_before: syscall.Timespec = undefined;
    _ = try syscall.clock_gettime(1, &ts_before); // CLOCK_MONOTONIC = 1

    // Send SIGUSR1 to self (signal is now pending)
    const pid = syscall.getpid();
    try syscall.kill(pid, @intCast(SIGUSR1));

    // Read from signalfd (should return immediately since signal is already pending)
    var siginfo: syscall.SignalFdSigInfo = undefined;
    const read_bytes = try syscall.read(fd, @as([*]u8, @ptrCast(&siginfo)), @sizeOf(syscall.SignalFdSigInfo));
    if (read_bytes != @sizeOf(syscall.SignalFdSigInfo)) return error.TestFailed;
    if (siginfo.ssi_signo != SIGUSR1) return error.TestFailed;

    // Record time after read
    var ts_after: syscall.Timespec = undefined;
    _ = try syscall.clock_gettime(1, &ts_after);

    // Verify elapsed time is well under 10ms (should be < 1ms)
    // With 10ms polling, worst case would be ~10ms. Direct wakeup should be < 2ms.
    const elapsed_ns = (@as(i64, ts_after.tv_sec) - @as(i64, ts_before.tv_sec)) * 1_000_000_000 +
        (@as(i64, ts_after.tv_nsec) - @as(i64, ts_before.tv_nsec));
    // Allow up to 5ms (generous margin for scheduler jitter on QEMU)
    if (elapsed_ns > 5_000_000) return error.TestFailed;
}
```

2. In `src/user/test_runner/tests/syscall/seccomp.zig`, add a test `testSeccompSigsysDelivery` that verifies SIGSYS is delivered on SECCOMP_RET_KILL:

- Verify child was killed by SIGSYS (exit status should be 128 + 31 = 159, or WIFSIGNALED with SIGSYS)

```zig
/// Test: seccomp SECCOMP_RET_KILL delivers SIGSYS
pub fn testSeccompSigsysDelivery() !void {
    const pid = try syscall.fork();
    if (pid == 0) {
        // Child: install seccomp filter, trigger blocked syscall
        _ = syscall.prctl(syscall.PR_SET_NO_NEW_PRIVS, 1, 0, 0, 0) catch {
            syscall.exit(1);
        };

        // BPF: block getpid (returns KILL_THREAD)
        const BPF_LD = syscall.BPF_LD;
        const BPF_W = syscall.BPF_W;
        const BPF_ABS = syscall.BPF_ABS;
        const BPF_JMP = syscall.BPF_JMP;
        const BPF_JEQ = syscall.BPF_JEQ;
        const BPF_K = syscall.BPF_K;
        const BPF_RET = syscall.BPF_RET;

        var filter = [_]syscall.SockFilterInsn{
            // Load syscall number (offset 0 in seccomp_data)
            .{ .code = BPF_LD | BPF_W | BPF_ABS, .jt = 0, .jf = 0, .k = 0 },
            // If getpid, jump to kill
            .{ .code = BPF_JMP | BPF_JEQ | BPF_K, .jt = 0, .jf = 1, .k = @intCast(syscall.SYS_GETPID) },
            // Kill thread
            .{ .code = BPF_RET | BPF_K, .jt = 0, .jf = 0, .k = syscall.SECCOMP_RET_KILL },
            // Allow
            .{ .code = BPF_RET | BPF_K, .jt = 0, .jf = 0, .k = syscall.SECCOMP_RET_ALLOW },
        };
        var prog = syscall.SockFprog{ .len = 4, .filter = @intFromPtr(&filter) };
        _ = syscall.seccomp(syscall.SECCOMP_SET_MODE_FILTER, 0, @intFromPtr(&prog)) catch {
            syscall.exit(2);
        };

        // Trigger blocked syscall -- SIGSYS should kill us
        _ = syscall.getpid();

        // If we get here, SIGSYS was not delivered
        syscall.exit(3);
    }

    // Parent: wait for child, verify killed by SIGSYS
    var status: u32 = 0;
    const waited = try syscall.wait4(pid, &status, 0, null);
    if (waited != pid) return error.TestFailed;

    // Child should have been killed by signal (SIGSYS = 31)
    // Exit status for signal death: 128 + signum = 159
    // Or check WIFSIGNALED pattern: low 7 bits of status = signum
    const exit_code: u8 = @truncate(status >> 8);
    const sig: u8 = @truncate(status & 0x7F);
    // Check if killed by signal (sig != 0) and the signal is SIGSYS (31)
    // OR check exit code is 128+31=159 (our kernel uses exitWithStatus(128+signum))
    if (sig == 31 or exit_code == 159) {
        return; // PASS
    }
    return error.TestFailed;
}
```

NOTE: Check the actual exit status encoding in the wait4 result. The kernel's `exitWithStatus(128 + signum)` may encode differently than WIFSIGNALED. Look at how existing process death tests check status (e.g., testSeccompStrictBlocksGetpid or the process tests that verify signal death).
