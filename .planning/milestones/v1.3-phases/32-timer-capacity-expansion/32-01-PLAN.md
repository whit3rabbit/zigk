<objective>
Increase the per-process POSIX timer limit from 8 to 32 and add a fast-path count field to avoid iterating all timer slots on every scheduler tick when no timers are active.

Purpose: Linux exposes POSIX_TIMER_MAX=32 per process by default. Applications that create many timers (e.g., per-connection network keepalives) currently fail with EAGAIN after 8 timers.

Output: Expanded fixed array (32 slots), posix_timer_count field for fast-path skip, updated syscalls, and a new integration test verifying creation of 9+ timers.
</objective>
