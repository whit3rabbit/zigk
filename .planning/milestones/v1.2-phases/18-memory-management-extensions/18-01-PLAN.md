Implement memfd_create, mremap, and msync syscalls for advanced memory management operations.

Purpose: Enable anonymous memory-backed file descriptors (for shared memory and temporary buffers), resizable memory mappings, and memory synchronization. memfd_create is the most complex -- it creates a new file type with full I/O support. mremap enables growing/moving mappings without unmap+remap. msync is a validation-only operation since zk has no buffer cache.

Output: 3 new syscalls working on both x86_64 and aarch64, with userspace wrappers and integration tests.

**memory.zig -- Add sys_memfd_create:**

Create a `MemfdState` struct holding:
- `buffer: ?[*]u8` -- PMM-backed buffer (kernel virtual via physToVirt)
- `phys_addr: u64` -- physical address of backing pages (for mmap)
- `capacity: usize` -- allocated capacity in bytes (page-aligned)
- `size: usize` -- current file size (may be less than capacity)
- `ref_count: std.atomic.Value(u32)` -- lifecycle management (starts at 1)
- `lock: sync.Spinlock` -- protects buffer/size/capacity mutations

MemfdState methods:
- `init(flags: u32) !*MemfdState` -- allocate state on heap, initially 0 capacity/size, no buffer
- `ensureCapacity(new_cap: usize) !void` -- if new_cap > capacity, allocate PMM pages for new_cap (page-aligned), copy old data if any, free old pages. Use `pmm.allocZeroedPages` for new buffer, `hal.paging.physToVirt` for kernel access.
- `ref()/unref()` -- standard ref counting pattern, unref frees state + PMM pages when count hits 0
- `truncate(new_size: usize) !void` -- if new_size > size, ensureCapacity and zero-fill new region. If new_size < size, just set size (don't free pages yet). Update size.

Create `memfd_file_ops: fd_mod.FileOps` with:
- `read`: Read from buffer at fd.position, advance position, return bytes read (0 at EOF). Clamp to size - position.
- `write`: Write to buffer at fd.position, call ensureCapacity if needed, advance position, update size if position > size. Return bytes written.
- `close`: Call state.unref() to drop FD's reference.
- `seek`: Standard SEEK_SET/SEEK_CUR/SEEK_END against state.size. Return new position.
- `mmap`: Return state.phys_addr for the backing memory. If buffer is null (empty memfd), return 0. The offset parameter is ignored (memfd maps from start). Set `*size` to state.capacity (page-aligned).
- `stat`: Fill stat_buf with mode=0o100600 (regular file, owner rw), st_size=state.size, st_blksize=4096.
- `truncate`: Call state.truncate(length).
- `poll`: Return EPOLLIN if size > 0, EPOLLOUT always (writable).

`sys_memfd_create(name_ptr: usize, flags: usize) SyscallError!usize`:
1. Validate flags: only MFD_CLOEXEC (0x0001) and MFD_ALLOW_SEALING (0x0002) are valid. Return EINVAL for unknown flags. We accept but ignore MFD_ALLOW_SEALING (no sealing support).
2. Copy name from user (up to 249 chars + null, Linux limit). Name is informational only -- we don't store it (match minimal kernel pattern).
3. Allocate MemfdState via heap.
4. Allocate FileDescriptor via heap, set ops=&memfd_file_ops, private_data=state, flags=O_RDWR | (O_CLOEXEC if MFD_CLOEXEC).
5. Get FD table from current process, allocate FD number, install FD.
6. Return FD number.

**memory.zig -- Add sys_msync:**

`sys_msync(addr: usize, len: usize, flags: usize) SyscallError!usize`:
1. Validate addr is page-aligned, return EINVAL if not.
2. Validate len > 0, return EINVAL if 0.
3. Validate flags: must have exactly one of MS_ASYNC(1) or MS_SYNC(4), optionally MS_INVALIDATE(2). Return EINVAL if neither or both MS_ASYNC+MS_SYNC.
4. No-op: zk has no buffer cache, data is already on disk. Return 0.

This is the same pattern as the existing msync test (testMsyncNoOp) which already calls the userspace wrapper. The kernel handler just needs to exist and validate args.

**user_vmm.zig -- Add mremap helper:**

Add `pub fn mremap(self: *UserVmm, old_addr: u64, old_size: usize, new_size: usize, flags: u32) isize` to UserVmm:

1. Acquire write lock.
2. Validate old_addr is page-aligned, old_size and new_size are > 0.
3. Page-align old_size and new_size.
4. Find VMA containing old_addr. Return -EFAULT if not found.
5. Verify VMA covers [old_addr, old_addr + aligned_old_size). Return -EFAULT if partial.
6. If new_size <= old_size (shrinking):
   - Unmap pages from old_addr + new_size to old_addr + old_size.
   - Free the unmapped physical pages via PMM.
   - Update VMA end = old_addr + new_size.
   - Return old_addr (no move needed).
7. If new_size > old_size (growing):
   - Check if space after VMA is free (no overlapping VMA in [old_addr + old_size, old_addr + new_size)).
   - If free: allocate new pages, map them, extend VMA end. Return old_addr.
   - If not free and MREMAP_MAYMOVE (1) is set:
     a. Find a new free range of new_size via findFreeRange (call findFreeRangeLocked to avoid double-lock, or release+reacquire, or add an internal unlocked variant).
     b. If no free range, return -ENOMEM.
     c. Allocate new_size pages, map at new address.
     d. Copy data from old mapping to new mapping (page by page via kernel virtual addresses from physToVirt).
     e. Unmap old pages, free old physical pages.
     f. Remove old VMA, create new VMA at new address.
     g. Return new address.
   - If not free and MREMAP_MAYMOVE is NOT set: return -ENOMEM.

IMPORTANT: The findFreeRange method currently acquires the write lock internally. Since mremap already holds the write lock, we need either:
- A `findFreeRangeLocked` variant that does not acquire the lock, OR
- Release the lock before calling findFreeRange and re-acquire after (but this introduces TOCTOU risk).

Best approach: Add `pub fn findFreeRangeLocked(self: *UserVmm, size: usize) ?u64` that contains the actual logic, and have `findFreeRange` call it with lock acquisition. Check if findFreeRange already acquires a lock -- if it doesn't (reader lock only or no lock), then calling it from mremap under write lock may be fine.

Check the actual findFreeRange implementation before implementing. If it only reads VMA list (which mremap's write lock protects), it may work without lock changes.

**io.zig -- Add memfd_create and mremap wrappers:**

After the existing msync function (around line 508), add:

```zig
/// MFD_CLOEXEC flag for memfd_create
pub const MFD_CLOEXEC: u32 = 0x0001;
/// MFD_ALLOW_SEALING flag for memfd_create
pub const MFD_ALLOW_SEALING: u32 = 0x0002;

/// MREMAP flags
pub const MREMAP_MAYMOVE: u32 = 1;

/// Create an anonymous memory-backed file descriptor
pub fn memfd_create(name: [*:0]const u8, flags: u32) SyscallError!i32 {
    const ret = primitive.syscall2(
        syscalls.SYS_MEMFD_CREATE,
        @intFromPtr(name),
        flags,
    );
    if (primitive.isError(ret)) return primitive.errorFromReturn(ret);
    return @intCast(ret);
}

/// Remap a virtual memory region
pub fn mremap(old_addr: [*]u8, old_size: usize, new_size: usize, flags: u32) SyscallError![*]u8 {
    const ret = primitive.syscall4(
        syscalls.SYS_MREMAP,
        @intFromPtr(old_addr),
        old_size,
        new_size,
        flags,
    );
    if (primitive.isError(ret)) return primitive.errorFromReturn(ret);
    return @ptrFromInt(ret);
}
```

7. `testMremapGrow`: mmap 4096 bytes anonymous. Write pattern byte at offset 0. Call mremap(addr, 4096, 8192, MREMAP_MAYMOVE). Verify returned address is valid (non-null, non-error). Verify byte at offset 0 in new mapping still has the pattern. Write at offset 4096 (in the new region). Verify read-back. munmap new address with new size.

8. `testMremapShrink`: mmap 8192 bytes anonymous. Write pattern at offset 0. mremap(addr, 8192, 4096, 0) -- shrink, no MAYMOVE needed. Verify returned address == original address. Verify pattern at offset 0 preserved. munmap with 4096.

9. `testMremapInvalidAddr`: mremap(0x12340000 as ptr, 4096, 8192, MREMAP_MAYMOVE). Verify returns error (EFAULT or EINVAL).

10. `testMsyncValidation`: mmap anonymous page. msync with MS_SYNC -- should succeed (already tested by existing testMsyncNoOp, but this explicitly tests the kernel handler path). msync with invalid flags (MS_SYNC | MS_ASYNC = 5) -- should fail with EINVAL. munmap.
