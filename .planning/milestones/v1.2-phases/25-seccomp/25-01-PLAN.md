    ```
    // Seccomp modes
    SECCOMP_MODE_DISABLED = 0
    SECCOMP_MODE_STRICT = 1
    SECCOMP_MODE_FILTER = 2

    // Seccomp operations (for sys_seccomp)
    SECCOMP_SET_MODE_STRICT = 0
    SECCOMP_SET_MODE_FILTER = 1
    SECCOMP_GET_ACTION_AVAIL = 2

    // BPF filter return values (32-bit, high 16 = action, low 16 = data)
    SECCOMP_RET_KILL_PROCESS = 0x80000000
    SECCOMP_RET_KILL_THREAD  = 0x00000000
    SECCOMP_RET_KILL         = SECCOMP_RET_KILL_THREAD  // alias
    SECCOMP_RET_ERRNO        = 0x00050000  // low 16 bits = errno value
    SECCOMP_RET_ALLOW        = 0x7fff0000
    SECCOMP_RET_ACTION_FULL  = 0xffff0000
    SECCOMP_RET_DATA         = 0x0000ffff

    // seccomp_data struct (passed to BPF filter, 64 bytes)
    pub const SeccompData = extern struct {
        nr: i32,           // syscall number (i32 for Linux compat)
        arch: u32,         // AUDIT_ARCH_* value
        instruction_pointer: u64,
        args: [6]u64,      // syscall arguments
    };
    comptime { assert(@sizeOf(SeccompData) == 64); }

    // AUDIT_ARCH constants
    AUDIT_ARCH_X86_64  = 0xC000003E  // EM_X86_64 | __AUDIT_ARCH_64BIT | __AUDIT_ARCH_LE
    AUDIT_ARCH_AARCH64 = 0xC00000B7  // EM_AARCH64 | __AUDIT_ARCH_64BIT | __AUDIT_ARCH_LE

    // Classic BPF instruction (8 bytes)
    pub const SockFilterInsn = extern struct {
        code: u16,
        jt: u8,    // jump if true
        jf: u8,    // jump if false
        k: u32,    // generic multiuse field
    };
    comptime { assert(@sizeOf(SockFilterInsn) == 8); }

    // BPF program descriptor (passed from userspace)
    pub const SockFprog = extern struct {
        len: u16,          // number of filter instructions
        _pad: u16 = 0,
        _pad2: u32 = 0,
        filter: u64,       // pointer to SockFilterInsn array (as u64 for ABI)
    };

    // Classic BPF opcodes (subset needed for seccomp)
    BPF_LD   = 0x00
    BPF_LDX  = 0x01
    BPF_ST   = 0x02
    BPF_STX  = 0x03
    BPF_ALU  = 0x04
    BPF_JMP  = 0x05
    BPF_RET  = 0x06
    BPF_MISC = 0x07

    // BPF size modifiers
    BPF_W    = 0x00  // 32-bit word
    BPF_H    = 0x08  // 16-bit halfword
    BPF_B    = 0x10  // 8-bit byte

    // BPF source modifiers
    BPF_IMM  = 0x00  // immediate value K
    BPF_ABS  = 0x20  // absolute offset in packet (seccomp_data)
    BPF_IND  = 0x40  // indirect (not used in seccomp)
    BPF_MEM  = 0x60  // scratch memory
    BPF_LEN  = 0x80  // packet length (seccomp_data length = 64)
    BPF_MSH  = 0xa0  // not used in seccomp

    // BPF ALU operations
    BPF_ADD  = 0x00
    BPF_SUB  = 0x10
    BPF_MUL  = 0x20
    BPF_DIV  = 0x30
    BPF_OR   = 0x40
    BPF_AND  = 0x50
    BPF_LSH  = 0x60
    BPF_RSH  = 0x70
    BPF_NEG  = 0x80
    BPF_MOD  = 0x90
    BPF_XOR  = 0xa0

    // BPF jump conditions
    BPF_JA   = 0x00
    BPF_JEQ  = 0x10
    BPF_JGT  = 0x20
    BPF_JGE  = 0x30
    BPF_JSET = 0x40

    // BPF source for ALU/JMP
    BPF_K    = 0x00  // use K value
    BPF_X    = 0x08  // use X register

    // BPF MISC
    BPF_TAX  = 0x00  // A -> X
    BPF_TXA  = 0x80  // X -> A

    // Maximum BPF program length
    BPF_MAXINSNS = 4096
    ```

    **7. Implement sys_seccomp with BPF interpreter** (`src/kernel/sys/syscall/process/process.zig`):
    Add the sys_seccomp function (will be auto-discovered by dispatch table since SYS_SECCOMP exists in uapi). Add it in the same file as capget/capset to avoid adding a new dispatch module (stack growth risk per MEMORY.md).

    The function signature:
    ```zig
    pub fn sys_seccomp(op: usize, flags: usize, args_ptr: usize) SyscallError!usize
    ```

    Implementation:
    - **SECCOMP_SET_MODE_STRICT (op=0)**: flags must be 0, args_ptr must be 0. Set `proc.seccomp_mode = 1`. Once set, only SYS_READ, SYS_WRITE, SYS_EXIT, SYS_EXIT_GROUP, SYS_RT_SIGRETURN are allowed. Cannot be undone.
    - **SECCOMP_SET_MODE_FILTER (op=1)**: flags must be 0 for MVP. args_ptr points to a SockFprog in userspace. Requires `proc.no_new_privs == true` or proc has CAP_SYS_ADMIN. Copy the SockFprog header from userspace, validate len (1..4096), then copy the filter instructions. Append to proc.seccomp_filters array. Set `proc.seccomp_mode = 2` (or keep at 2 if already filtering). If already in STRICT mode, return EACCES (cannot downgrade).
    - **SECCOMP_GET_ACTION_AVAIL (op=2)**: flags must be 0. args_ptr points to a u32 action. Return 0 if action is one of SECCOMP_RET_KILL/ERRNO/ALLOW, else EOPNOTSUPP.

    Also add a BPF interpreter function (private to the module):
    ```zig
    fn runBpfFilter(insns: []const SockFilterInsn, data: *const SeccompData) u32
    ```
    This is a classic BPF interpreter operating on the seccomp_data as the "packet":
    - Registers: A (accumulator, u32), X (index, u32)
    - 16 scratch memory slots (M[0..15], u32 each)
    - Instructions: LD (load from data), LDX, ST, STX, ALU (add/sub/mul/div/and/or/xor/lsh/rsh/neg/mod), JMP (ja/jeq/jgt/jge/jset), RET (return action value)
    - BPF_ABS loads: offset into seccomp_data as byte array, load BPF_W (4 bytes) / BPF_H (2 bytes) / BPF_B (1 byte)
    - On invalid instruction or out-of-bounds access, return SECCOMP_RET_KILL_PROCESS (fail-secure)
    - Maximum 4096 instructions executed (prevent infinite loops -- classic BPF has no backward jumps, but enforce anyway)

    Also add a public function for the dispatch hook:
    ```zig
    pub fn checkSeccomp(proc: *const Process, syscall_num: usize, args: [6]usize) u32
    ```
    This function:
    - If `proc.seccomp_mode == 0`: return SECCOMP_RET_ALLOW
    - If `proc.seccomp_mode == 1` (STRICT): check if syscall_num is in {SYS_READ, SYS_WRITE, SYS_EXIT, SYS_EXIT_GROUP, SYS_RT_SIGRETURN}. If yes, return SECCOMP_RET_ALLOW. Else return SECCOMP_RET_KILL_THREAD.
    - If `proc.seccomp_mode == 2` (FILTER): construct SeccompData from syscall_num + args + arch, then run each filter program in sequence. The most restrictive result wins (lowest action value). Return that result.
    - IMPORTANT: The arch field should use AUDIT_ARCH_X86_64 on x86_64 and AUDIT_ARCH_AARCH64 on aarch64, using `@import("builtin").cpu.arch`.

    ```zig
    // Seccomp filtering -- check before executing any handler
    const seccomp_check = blk_seccomp: {
        const proc = base.getCurrentProcessOrNull();
        if (proc) |p| {
            if (p.seccomp_mode != 0) {
                break :blk_seccomp process.checkSeccomp(p, syscall_num, args);
            }
        }
        break :blk_seccomp uapi.seccomp.SECCOMP_RET_ALLOW;
    };

    const seccomp_action = seccomp_check & uapi.seccomp.SECCOMP_RET_ACTION_FULL;
    if (seccomp_action == uapi.seccomp.SECCOMP_RET_KILL_THREAD or
        seccomp_action == uapi.seccomp.SECCOMP_RET_KILL_PROCESS) {
        // Disallowed syscall -- return ENOSYS (SIGSYS delivery is a future enhancement)
        frame.setReturnSigned(uapi.errno.ENOSYS.toReturn());
        return;
    }
    if (seccomp_action == uapi.seccomp.SECCOMP_RET_ERRNO) {
        const errno_val = seccomp_check & uapi.seccomp.SECCOMP_RET_DATA;
        frame.setReturnSigned(-@as(isize, @intCast(errno_val)));
        return;
    }
    // SECCOMP_RET_ALLOW: fall through to normal dispatch
    ```

    IMPORTANT: Do NOT use `hal.getCurrentThread()` -- that function does not exist. Use `base.getCurrentProcessOrNull()` which internally calls `sched.getCurrentThread()` and casts the opaque process pointer. Using `getCurrentProcessOrNull()` (not `getCurrentProcess()`) avoids panicking during early boot when no process exists. The `process` module (already imported at line 19 of table.zig) provides the `checkSeccomp` function added in action step 7.

    NOTE: The seccomp check runs for ALL syscalls including SYS_SECCOMP itself. In strict mode, seccomp is blocked. Only read/write/exit/sigreturn are allowed. This is correct Linux behavior.

    **3. Create integration tests** (`src/user/test_runner/tests/syscall/seccomp.zig`):
    Write 10 tests covering both seccomp modes and error cases. ALL tests that install seccomp filters MUST run in a forked child process (seccomp is irreversible). The test runner process must never have seccomp installed.

    Pattern for seccomp tests: fork, child installs seccomp and exercises it, parent waits and checks exit status.

    Tests:
    1. `testSeccompStrictAllowsRead` -- Fork child. Child calls seccomp(STRICT). Child calls read() (should succeed). Child exits 0. Parent verifies exit 0.
    2. `testSeccompStrictAllowsWrite` -- Fork child. Child calls seccomp(STRICT). Child calls write() to debug_log (should succeed). Child exits 0. Parent verifies exit 0.
    3. `testSeccompStrictBlocksGetpid` -- Fork child. Child calls seccomp(STRICT). Child calls getpid() -- the dispatch hook returns ENOSYS (not a process kill in MVP). Child checks that getpid() returned an error (WouldBlock/ENOSYS). If error received, child exits 0. If getpid succeeded unexpectedly, child exits 1. Parent waits and verifies child exited 0.
    4. `testSeccompFilterAllowAll` -- Fork child. Child sets no_new_privs via prctl. Child installs BPF filter that returns SECCOMP_RET_ALLOW for all syscalls. Child calls getpid() (should succeed). Child exits 0.
    5. `testSeccompFilterBlockGetpid` -- Fork child. Child sets no_new_privs. Child installs BPF filter: if syscall == SYS_GETPID, return SECCOMP_RET_ERRNO | EPERM(1), else SECCOMP_RET_ALLOW. Child calls getpid() -- should fail with EPERM. Child exits 0 if getpid returned error.
    6. `testSeccompRequiresNoNewPrivs` -- Fork child. Child does NOT set no_new_privs. Child tries seccomp(FILTER, ...) -- should fail with EACCES or EPERM. Child exits 0 on expected error.
    7. `testSeccompStrictCannotBeUndone` -- Fork child. Child calls seccomp(STRICT). Child tries seccomp(FILTER, ...) -- should fail. Child exits 0 on expected error.
    8. `testSeccompFilterErrno` -- Fork child. Child sets no_new_privs. Installs filter: if syscall == SYS_GETPID, return SECCOMP_RET_ERRNO | 22 (EINVAL). Calls getpid(). Verifies it gets EINVAL error. Exits 0.
    9. `testSeccompInheritedOnFork` -- Fork child A. A sets no_new_privs + installs filter blocking SYS_GETPID with ERRNO|1. A forks child B. B calls getpid() -- should fail. A waits for B and checks result. A exits 0 if B got the error.
    10. `testPrctlNoNewPrivs` -- Call prctl(PR_SET_NO_NEW_PRIVS, 1). Call prctl(PR_GET_NO_NEW_PRIVS). Verify returns 1. (Can run in main process since no_new_privs is harmless.)

    IMPORTANT syscall number references for BPF filters:
    - Use `syscall.uapi.syscalls.SYS_GETPID` to get the correct syscall number for the current architecture
    - The seccomp_data.nr offset is 0 (first field, i32)
    - BPF filter to check syscall number: `BPF_LD | BPF_W | BPF_ABS` with k=0 (offset of nr field)
    - BPF filter to compare: `BPF_JMP | BPF_JEQ | BPF_K` with k=syscall_number, jt=0 (next insn), jf=1 (skip one)
    - BPF filter to return: `BPF_RET | BPF_K` with k=return_value

    Example BPF filter (block SYS_GETPID with EPERM):
    ```zig
    var filter = [_]SockFilterInsn{
        .{ .code = BPF_LD | BPF_W | BPF_ABS, .jt = 0, .jf = 0, .k = 0 },  // A = seccomp_data.nr
        .{ .code = BPF_JMP | BPF_JEQ | BPF_K, .jt = 0, .jf = 1, .k = @intCast(syscalls.SYS_GETPID) }, // if A == GETPID goto next, else skip
        .{ .code = BPF_RET | BPF_K, .jt = 0, .jf = 0, .k = SECCOMP_RET_ERRNO | 1 },  // return ERRNO(EPERM)
        .{ .code = BPF_RET | BPF_K, .jt = 0, .jf = 0, .k = SECCOMP_RET_ALLOW },       // return ALLOW
    };
    ```
