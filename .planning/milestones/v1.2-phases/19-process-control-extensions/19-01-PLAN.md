Define the `CloneArgs` struct matching Linux's `struct clone_args`:
```zig
const CloneArgs = extern struct {
    flags: u64,          // Clone flags
    pidfd: u64,          // Pointer to store PID FD (unused, validate 0)
    child_tid: u64,      // Pointer to store child TID
    parent_tid: u64,     // Pointer to store parent TID
    exit_signal: u64,    // Signal to deliver on exit
    stack: u64,          // Stack pointer for child
    stack_size: u64,     // Stack size (informational)
    tls: u64,            // TLS pointer
    set_tid: u64,        // Pointer to set_tid array (unused, validate 0)
    set_tid_size: u64,   // Size of set_tid array (unused, validate 0)
    cgroup: u64,         // Cgroup FD (unused, validate 0)
};
```

Implementation:
- Validate `size >= @sizeOf(CloneArgs)` (minimum 88 bytes). Return EINVAL if too small.
- Read CloneArgs from userspace using UserPtr.readValue(CloneArgs). Return EFAULT on failure.
- Validate unsupported fields are zero: pidfd, set_tid, set_tid_size, cgroup. Return EINVAL if non-zero.
- If `flags & CLONE_THREAD != 0`: delegate to the thread-creation path (reuse the core logic from sys_clone's CLONE_THREAD branch, but read stack/tls/parent_tid/child_tid from the struct instead of registers).
- If `flags == 0` and `exit_signal == SIGCHLD` (17): treat as fork. Call sys_fork(frame).
- If `flags & CLONE_VM == 0` and exit_signal is SIGCHLD: also treat as fork (standard child process creation).
- For the CLONE_THREAD path, extract flags, stack, tls, parent_tid, child_tid from CloneArgs and use the same logic as existing sys_clone CLONE_THREAD handling. The key difference is args come from the struct, not from register positions.
- Construct synthetic flag bitmask including CLONE_PARENT_SETTID/CLONE_CHILD_SETTID/CLONE_CHILD_CLEARTID if the corresponding struct pointer fields are non-zero (Linux semantics: clone3 infers these flags from non-null pointers, but the flags field can also set them explicitly).
- Important: `sys_clone3` must take `frame: *hal.syscall.SyscallFrame` as first arg (needed for copyThreadState/setForkChildReturn), then `clone_args_ptr: usize, size: usize`.

```zig
const P_PID: usize = 1;

const WCONTINUED: usize = 8;
const WNOWAIT: usize = 0x01000000;
```

Define `SigInfo` struct for the siginfo_t output (must match Linux ABI, 128 bytes):
```zig
const SigInfo = extern struct {
    si_signo: i32,    // Signal number (SIGCHLD = 17)
    si_errno: i32,    // Errno value
    si_code: i32,     // Signal code (CLD_EXITED=1, CLD_KILLED=2, CLD_STOPPED=5, CLD_CONTINUED=6)
    _pad0: i32 = 0,   // Padding
    // Union fields for SIGCHLD:
    si_pid: i32,      // Child PID
    si_uid: i32,      // Child's real UID
    si_status: i32,   // Exit status or signal number
    _pad: [128 - 28]u8 = [_]u8{0} ** (128 - 28), // Pad to 128 bytes total
};
```
CRITICAL: Verify `@sizeOf(SigInfo) == 128` with comptime assert.

Implementation:
- Validate options: must have at least one of WEXITED, WSTOPPED, WCONTINUED set. Return EINVAL otherwise.
- Validate idtype: must be P_ALL, P_PID, or P_PGID. Return EINVAL otherwise.
- If infop is 0, return EFAULT (Linux requires non-null infop for waitid).
- Reuse the same zombie-searching loop pattern from sys_wait4:
  - Disable interrupts, acquire process_tree_lock write lock.
  - Iterate current_proc.first_child, matching by:
    - P_ALL: match any child
    - P_PID: match child.pid == id
    - P_PGID: if id == 0, match child.pgid == current_proc.pgid; else match child.pgid == id
  - For zombie children (state == .Zombie):
    - If WEXITED is set in options:
      - If WNOWAIT is NOT set: remove zombie from list (same as wait4 reaping)
      - If WNOWAIT is set: do NOT remove (peek only)
      - Fill SigInfo: si_signo=17(SIGCHLD), si_code=CLD_EXITED(1), si_pid=child.pid, si_uid=child.uid, si_status=(exit_status >> 8) & 0xFF
      - Write SigInfo to infop using UserPtr.writeValue
      - If reaped (not WNOWAIT): propagate CPU times, destroy process (same as wait4)
      - Return 0 (success -- waitid returns 0 on success, unlike wait4 which returns PID)
  - If no zombie found and WNOHANG is set (options & 1 != 0): zero-fill infop (si_pid=0, si_signo=0), return 0
  - If no matching children at all: return ECHILD
  - Otherwise block (same pattern as wait4: set wait4_waiting flag, call sched.block())
- `_ = rusage_ptr;` -- ignore rusage (not implemented, same as wait4)

Note on WSTOPPED/WCONTINUED: The existing kernel does support SIGSTOP/SIGCONT for process state changes, but the wait infrastructure does not track stopped/continued transitions in a waitable manner. For this phase, accept the flags but only act on WEXITED (zombie reaping). If WSTOPPED or WCONTINUED are the ONLY flags (without WEXITED), the call will block indefinitely or return WNOHANG with zero -- this matches current kernel capabilities. Document this limitation.

**Step 1: Add userspace wrappers in process.zig**

```zig
// =============================================================================
// Modern Process Creation (clone3)
// =============================================================================

/// Clone arguments structure (matches Linux struct clone_args)
pub const CloneArgs = extern struct {
    flags: u64 = 0,
    pidfd: u64 = 0,
    child_tid: u64 = 0,
    parent_tid: u64 = 0,
    exit_signal: u64 = 0,
    stack: u64 = 0,
    stack_size: u64 = 0,
    tls: u64 = 0,
    set_tid: u64 = 0,
    set_tid_size: u64 = 0,
    cgroup: u64 = 0,
};

/// SIGCHLD signal number
pub const SIGCHLD: u64 = 17;

/// Create a child process using clone3
/// Returns: child PID to parent, 0 to child
pub fn clone3(args: *const CloneArgs) SyscallError!i32 {
    const ret = primitive.syscall2(
        syscalls.SYS_CLONE3,
        @intFromPtr(args),
        @sizeOf(CloneArgs),
    );
    if (primitive.isError(ret)) return primitive.errorFromReturn(ret);
    return @truncate(@as(isize, @bitCast(ret)));
}

// =============================================================================
// waitid
// =============================================================================

/// waitid idtype values
pub const P_ALL: u32 = 0;
pub const P_PID: u32 = 1;
pub const P_PGID: u32 = 2;

/// waitid option flags
pub const WEXITED: u32 = 4;
pub const WSTOPPED: u32 = 2;
pub const WCONTINUED: u32 = 8;
pub const WNOWAIT: u32 = 0x01000000;

/// siginfo_t structure for waitid (128 bytes, Linux ABI)
pub const SigInfo = extern struct {
    si_signo: i32 = 0,
    si_errno: i32 = 0,
    si_code: i32 = 0,
    _pad0: i32 = 0,
    si_pid: i32 = 0,
    si_uid: i32 = 0,
    si_status: i32 = 0,
    _pad: [128 - 28]u8 = [_]u8{0} ** (128 - 28),
};

/// CLD_* codes for si_code
pub const CLD_EXITED: i32 = 1;
pub const CLD_KILLED: i32 = 2;
pub const CLD_DUMPED: i32 = 3;
pub const CLD_TRAPPED: i32 = 4;
pub const CLD_STOPPED: i32 = 5;
pub const CLD_CONTINUED: i32 = 6;

/// Wait for child process state changes (modern interface)
/// Returns 0 on success
pub fn waitid(idtype: u32, id: u32, info: *SigInfo, options: u32) SyscallError!void {
    const ret = primitive.syscall5(
        syscalls.SYS_WAITID,
        idtype,
        id,
        @intFromPtr(info),
        options,
        0, // rusage not implemented
    );
    if (primitive.isError(ret)) return primitive.errorFromReturn(ret);
}
```
