<objective>
Implement fsync, fdatasync, sync, and syncfs syscalls with userspace wrappers and integration tests.

Purpose: Enable explicit file synchronization semantics. Since this kernel has no write-back cache (SFS writes go directly to the block device via writeSector), these are validation-only syscalls that verify the FD is valid and return success. This matches Linux behavior for filesystems with synchronous I/O.

Output: 4 kernel syscall handlers, 4 userspace wrappers, 8+ integration tests passing on both architectures.

  <action>
Add 4 syscall handlers to `src/kernel/sys/syscall/fs/fs_handlers.zig`:

1. **sys_fsync(fd_num: usize)** -- Flush file data and metadata to storage.
   - Validate fd_num: cast to u32 via `std.math.cast(u32, fd_num)`, return EBADF if null.
   - Look up FD in global table via `base.getGlobalFdTable().get(fd_u32)`, return EBADF if null.
   - Since this kernel has no write-back buffer cache (SFS writes synchronously via writeSector), the data is already on disk. Return 0.
   - Note: Do NOT check if the FD is writable. Linux fsync succeeds on read-only FDs too (it just has nothing to flush).

2. **sys_fdatasync(fd_num: usize)** -- Flush file data only (skip non-essential metadata).
   - Same implementation as sys_fsync. In a kernel without a buffer cache, this is identical. Return 0 after FD validation.

3. **sys_sync()** -- Flush all filesystem buffers globally.
   - Takes no arguments. Since there is no buffer cache, this is a pure no-op. Return 0.
   - No FD validation needed. Linux sync() always succeeds.

4. **sys_syncfs(fd_num: usize)** -- Flush buffers for the filesystem containing the given FD.
   - Validate fd_num: cast to u32, look up in global FD table. Return EBADF if invalid.
   - Return 0 (no buffer cache to flush).

All handlers must follow the pattern: `pub fn sys_X(...) base.SyscallError!usize`. The dispatch table auto-discovers them from the fs_handlers module by matching SYS_FSYNC -> sys_fsync etc.

Import requirements: These handlers only need `std`, `base` (from "base.zig"), which are already imported in fs_handlers.zig. No new imports needed.

Add userspace wrappers to `src/user/lib/syscall/io.zig`:

1. `pub fn fsync(fd: i32) SyscallError!void` -- calls syscall1(SYS_FSYNC, fd_as_usize), checks error.
2. `pub fn fdatasync(fd: i32) SyscallError!void` -- calls syscall1(SYS_FDATASYNC, fd_as_usize).
3. `pub fn sync_() void` -- calls syscall0(SYS_SYNC). Name with trailing underscore to avoid conflict with Zig keyword. Cannot fail per POSIX.
4. `pub fn syncfs(fd: i32) SyscallError!void` -- calls syscall1(SYS_SYNCFS, fd_as_usize).

Follow the existing pattern from io.zig (e.g., `ftruncate`, `close`). Use `@bitCast(@as(isize, fd))` for the fd argument as done elsewhere.

Re-export all 4 from `src/user/lib/syscall/root.zig`:
```
pub const fsync = io.fsync;
pub const fdatasync = io.fdatasync;
pub const sync_ = io.sync_;
pub const syncfs = io.syncfs;
```
