  <name>Task 1: Fix sys_tee repeated-peek loop bug</name>
  <files>
    src/kernel/sys/syscall/io/splice.zig
    src/kernel/fs/pipe.zig
  </files>
  <action>
**Root cause:** `sys_tee` (splice.zig lines 354-380) has a `while (total_copied < len)` loop that calls `peekPipeBuffer` on each iteration. Since `peekPipeBuffer` does NOT advance `read_pos`, every iteration peeks the same data from the source pipe and writes duplicate copies to the destination pipe. If the source has 13 bytes and `len=128`, the loop copies 13 bytes repeatedly (13, 26, 39, ...) until the dest pipe fills or `total_copied >= 128`. The test expects `teed == 13` but gets a larger value, causing TestFailed.

**Fix approach -- choose ONE of these two strategies:**

**Strategy A (simpler, recommended): Remove the loop from sys_tee entirely.**

Replace the while loop (lines 352-380) with a single peek+write:

```zig
// Single peek from source pipe (without consuming)
const peek_len = @min(len, tee_buf_size);
const bytes_peeked = pipe_mod.peekPipeBuffer(in_handle, kbuf[0..peek_len]);
if (bytes_peeked == 0) {
    heap.allocator().free(kbuf);
    return 0; // No data available
}

// Write to dest pipe
const bytes_written = pipe_mod.writeToPipeBuffer(out_handle, kbuf[0..bytes_peeked]);
heap.allocator().free(kbuf);
if (bytes_written == 0) {
    return error.EAGAIN;
}

return bytes_written;
```

With this approach, remove the `defer heap.allocator().free(kbuf)` and manage the free manually, OR keep the defer and just replace the loop body. The key point: ONE call to `peekPipeBuffer`, ONE call to `writeToPipeBuffer`, return the result.

**Strategy B (if large tees matter): Add offset parameter to peekPipeBuffer.**

Modify `peekPipeBuffer` in pipe.zig to accept an offset parameter:
```zig
pub fn peekPipeBuffer(handle: *PipeHandle, buf: []u8, offset: usize) usize {
```
Then peek starting at `(read_pos + offset) % PIPE_BUF_SIZE` and limit to `data_len - offset`. Update the sys_tee loop to track a `peek_offset` variable that advances by `bytes_peeked` each iteration.

Strategy A is recommended because tee in practice is bounded by pipe buffer size (64KB) which fits in a single chunk. Strategy B is over-engineering for this kernel.

**After the fix, verify the logic:**
- Source pipe has N bytes
- tee(pipe_a_read, pipe_b_write, M, 0) where M >= N
- peekPipeBuffer returns N bytes (all available data)
- writeToPipeBuffer writes N bytes to dest
- Returns N
- Source pipe still has N bytes at read_pos (unchanged)
- Reading from source pipe after tee returns the original data

**Problem:** Three copy_file_range tests (testCopyFileRangeBasic, testCopyFileRangeWithOffsets, testCopyFileRangeInvalidFlags) all timeout because they create SFS files and hit the known SFS close deadlock. The SFS close deadlock is a pre-existing issue (Phase 11 scope) -- not something to fix here. Additionally, testSpliceZeroLength never runs because it's registered after the timeout tests.

**Fix 1: Rewrite testCopyFileRangeBasic to avoid SFS close deadlock.**

The test currently creates two SFS files (open/write/close/reopen pattern). Rewrite to:
- Use an InitRD file (/shell.elf) as the read source (no SFS needed for input)
- Create ONE SFS file as destination, keep it open (never close it -- fd leak is OK in tests per MEMORY.md)
- Use O_RDWR for the destination so you can write then lseek(0) and read back
- Do NOT call close() on SFS file descriptors

```zig
pub fn testCopyFileRangeBasic() !void {
    const dst_path = "/mnt/zcio_cfr.txt";

    // Source: InitRD file (read-only, no SFS needed)
    const src_fd = try syscall.open("/shell.elf", syscall.O_RDONLY, 0);
    // Don't defer close -- we'll close InitRD fd explicitly (safe, not SFS)

    // Destination: SFS file, keep open throughout
    const dst_fd = syscall.open(dst_path, syscall.O_CREAT | syscall.O_RDWR, 0o644) catch |err| {
        syscall.close(src_fd) catch {};
        if (err == error.ReadOnlyFileSystem) return error.SkipTest;
        return err;
    };
    // Do NOT close dst_fd (SFS deadlock avoidance)
    // Do NOT defer unlink (requires close first on SFS)

    // Copy 20 bytes from shell.elf to SFS file
    const copied = syscall.copy_file_range(src_fd, null, dst_fd, null, 20, 0) catch |err| {
        syscall.close(src_fd) catch {};
        return err;
    };
    syscall.close(src_fd) catch {};  // Safe: InitRD close doesn't deadlock
    if (copied != 20) return error.TestFailed;

    // Verify by seeking back to 0 and reading (avoids close/reopen)
    _ = try syscall.lseek(dst_fd, 0, 0); // SEEK_SET
    var buf: [32]u8 = undefined;
    const read_len = try syscall.read(dst_fd, &buf, 20);
    if (read_len != 20) return error.TestFailed;

    // Verify content matches direct read from source
    const src_fd2 = try syscall.open("/shell.elf", syscall.O_RDONLY, 0);
    var direct_buf: [32]u8 = undefined;
    const direct_len = try syscall.read(src_fd2, &direct_buf, 20);
    syscall.close(src_fd2) catch {};

    if (direct_len != 20 or !std.mem.eql(u8, buf[0..20], direct_buf[0..20])) {
        return error.TestFailed;
    }
    // Leave dst_fd open intentionally (SFS close deadlock workaround)
}
```

**Fix 2: Rewrite testCopyFileRangeWithOffsets similarly.**

- Source: Write "ABCDEFGHIJ" to a single SFS file opened O_RDWR
- Use the SAME fd for both source and destination (copy_file_range allows same-file copies per Linux semantics)
- Copy 4 bytes from offset 3 to offset 5 within the same file
- Read back and verify bytes 5-8 are "DEFG"
- Never close the SFS fd

Alternatively, if same-fd copy is too tricky with the current implementation, use two SFS files but never close either. The key constraint: do NOT call close() on any SFS fd.

```zig
pub fn testCopyFileRangeWithOffsets() !void {
    const path = "/mnt/zcio_cfo.txt";

    // Create file with known content, keep open as O_RDWR
    const fd = syscall.open(path, syscall.O_CREAT | syscall.O_RDWR, 0o644) catch |err| {
        if (err == error.ReadOnlyFileSystem) return error.SkipTest;
        return err;
    };

    // Write "ABCDEFGHIJ" (10 bytes)
    _ = try syscall.write(fd, "ABCDEFGHIJ", 10);

    // Copy 4 bytes from offset 3 to offset 5 (same file)
    var off_in: u64 = 3;
    var off_out: u64 = 5;
    const copied = try syscall.copy_file_range(fd, &off_in, fd, &off_out, 4, 0);
    if (copied != 4) return error.TestFailed;

    // Verify offsets updated
    if (off_in != 7 or off_out != 9) return error.TestFailed;

    // Seek to 0 and read full content
    _ = try syscall.lseek(fd, 0, 0);
    var buf: [10]u8 = undefined;
    const read_len = try syscall.read(fd, &buf, 10);
    if (read_len != 10) return error.TestFailed;

    // After copy: "ABCDEDEFGJ" -- bytes 5-8 should be "DEFG"
    if (!std.mem.eql(u8, buf[5..9], "DEFG")) {
        return error.TestFailed;
    }
    // Leave fd open (SFS deadlock workaround)
}
```
