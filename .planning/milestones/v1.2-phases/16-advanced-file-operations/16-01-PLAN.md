<objective>
Implement fallocate and renameat2 syscalls for advanced file operations.

Purpose: Provide POSIX-compatible file space pre-allocation and atomic rename-with-flags functionality. fallocate allows applications to reserve disk space upfront (preventing ENOSPC during later writes). renameat2 with RENAME_NOREPLACE enables safe "create if not exists" patterns, and RENAME_EXCHANGE enables atomic file swaps (useful for safe config updates).

Output: 2 kernel syscalls, VFS rename2 method, SFS rename2 implementation, 2 userspace wrappers, ~10 integration tests, all passing on both x86_64 and aarch64.
</objective>

    **Part A: sys_fallocate (285 on x86_64, 47 on aarch64)**

    Add to `src/kernel/sys/syscall/fs/fs_handlers.zig` after the fsync/syncfs block:

    ```
    pub fn sys_fallocate(fd_num: usize, mode: usize, offset_arg: usize, len_arg: usize) SyscallError!usize
    ```

    Implementation:
    1. Cast fd_num to u32 via `std.math.cast`, look up in global FD table, return EBADF if invalid
    2. Check fd is writable (`file_desc.isWritable()`), return EBADF if not
    3. Interpret offset_arg and len_arg as i64 (via `@bitCast`). Return EINVAL if either is negative or len is 0
    4. Define mode flag constants locally:
       - `FALLOC_FL_KEEP_SIZE: usize = 0x01`
       - `FALLOC_FL_PUNCH_HOLE: usize = 0x02`
    5. If mode contains FALLOC_FL_PUNCH_HOLE: return EOPNOTSUPP (SFS has contiguous blocks, cannot punch holes). Note: Linux requires FALLOC_FL_KEEP_SIZE with PUNCH_HOLE, but we reject PUNCH_HOLE regardless.
    6. If mode contains any bits other than FALLOC_FL_KEEP_SIZE: return EOPNOTSUPP (unsupported mode)
    7. Calculate required_size = offset + len using `std.math.add(u64, ...)`, return EFBIG on overflow
    8. If mode is 0 (default fallocate, no flags): extend file if needed. Get current file size from `file_desc.size`. If `required_size > current_size`, call `file_desc.ops.truncate` (same as ftruncate) to extend to required_size. If truncate op is null, return EOPNOTSUPP. Convert truncate errors: AccessDenied->EACCES, IOError->EIO.
    9. If mode is FALLOC_FL_KEEP_SIZE: validate the FD but do NOT change the file size. This is effectively a space reservation hint. Since SFS allocates blocks contiguously and there's no buffer cache, just validate and return 0. The blocks will be allocated on-demand when the application writes.
    10. Return 0 on success.

    SYS_FALLOCATE is already defined in `src/uapi/syscalls/root.zig` (285 x86_64, 47 aarch64). The comptime dispatch table will auto-register this.

    **Part B: sys_renameat2 (316 on x86_64, 276 on aarch64)**

    First, add a `rename2` method to the VFS and SFS:

    In `src/fs/vfs.zig`:
    1. Add to `FileSystem` struct a new optional function pointer:
       ```
       rename2: ?*const fn (ctx: ?*anyopaque, old_path: []const u8, new_path: []const u8, flags: u32) Error!void = null,
       ```
    2. Add `Vfs.rename2(old_path, new_path, flags)` public method. Same mount-point resolution logic as existing `Vfs.rename`, but:
       - If the filesystem has `rename2`, call it with flags
       - If the filesystem only has `rename` and flags == 0, call `rename` (fallback)
       - If the filesystem only has `rename` and flags != 0, return NotSupported

    In `src/fs/sfs/ops.zig`:
    1. Add `pub fn sfsRename2(ctx, old_path, new_path, flags: u32) vfs.Error!void`
    2. Define constants: `RENAME_NOREPLACE: u32 = 1`, `RENAME_EXCHANGE: u32 = 2`
    3. Implementation shares the same directory scan as sfsRename (extract old_name/new_name, scan directory for both):
       - **flags == 0**: Delegate to existing `sfsRename`
       - **RENAME_NOREPLACE**: After scanning, if new_name exists, return AlreadyExists. Otherwise, proceed with standard rename (just change the name in the old entry)
       - **RENAME_EXCHANGE**: After scanning, if EITHER entry is missing, return NotFound. Under alloc_lock: re-read both blocks, swap the names (write old_name into new entry's name field, new_name into old entry's name field), write both blocks back
       - Any other flags: return NotSupported
    4. Register `sfsRename2` in the SFS FileSystem struct (the `.rename2 = sfsRename2` field in the init)

    Then in `src/kernel/sys/syscall/fs/fs_handlers.zig`:

    Add `renameKernel2(raw_old, raw_new, flags: u32)` internal helper. Same logic as `renameKernel` but:
    - If flags contains RENAME_NOREPLACE: check `Vfs.statPath(new_path)`. If it returns non-null, return EEXIST immediately (before calling VFS). This provides a fast-path check at the syscall level.
    - Calls `Vfs.rename2(old_path, new_path, flags)` instead of `Vfs.rename`
    - Map errors: AlreadyExists->EEXIST (in addition to existing mappings)

    Add `pub fn sys_renameat2(olddirfd, oldpath_ptr, newdirfd, newpath_ptr, flags)`:
    - Same path resolution logic as sys_renameat (copy strings, resolve relative to dirfd)
    - Cast flags from usize to u32
    - Validate: if flags contains both RENAME_NOREPLACE and RENAME_EXCHANGE, return EINVAL (mutually exclusive per Linux)
    - Call `renameKernel2(resolved_old, resolved_new, flags_u32)`

    SYS_RENAMEAT2 is already defined in `src/uapi/syscalls/root.zig` (316 x86_64, 276 aarch64).

    **Part C: Userspace wrappers**

    In `src/user/lib/syscall/io.zig`, add after the sync section:

    ```zig
    // =============================================================================
    // Advanced File Operations (fallocate, renameat2)
    // =============================================================================

    pub const FALLOC_FL_KEEP_SIZE: u32 = 0x01;
    pub const FALLOC_FL_PUNCH_HOLE: u32 = 0x02;

    pub const RENAME_NOREPLACE: u32 = 1;
    pub const RENAME_EXCHANGE: u32 = 2;

    /// Pre-allocate or manipulate file space
    pub fn fallocate(fd: i32, mode: u32, offset: i64, len: i64) SyscallError!void {
        const ret = primitive.syscall4(syscalls.SYS_FALLOCATE, @bitCast(@as(isize, fd)), @as(usize, mode), @bitCast(offset), @bitCast(len));
        if (primitive.isError(ret)) return primitive.errorFromReturn(ret);
    }

    /// Rename file with flags (RENAME_NOREPLACE, RENAME_EXCHANGE)
    pub fn renameat2(olddirfd: i32, oldpath: [*:0]const u8, newdirfd: i32, newpath: [*:0]const u8, flags: u32) SyscallError!void {
        const ret = primitive.syscall5(syscalls.SYS_RENAMEAT2, @bitCast(@as(isize, olddirfd)), @intFromPtr(oldpath), @bitCast(@as(isize, newdirfd)), @intFromPtr(newpath), @as(usize, flags));
        if (primitive.isError(ret)) return primitive.errorFromReturn(ret);
    }
    ```

    In `src/user/lib/syscall/root.zig`, add re-exports:
    ```zig
    pub const fallocate = io.fallocate;
    pub const renameat2 = io.renameat2;
    pub const FALLOC_FL_KEEP_SIZE = io.FALLOC_FL_KEEP_SIZE;
    pub const FALLOC_FL_PUNCH_HOLE = io.FALLOC_FL_PUNCH_HOLE;
    pub const RENAME_NOREPLACE = io.RENAME_NOREPLACE;
    pub const RENAME_EXCHANGE = io.RENAME_EXCHANGE;
    ```
