## Tech Debt

### Non-Critical (deferred to future milestones)

1. **signalfd polling approach** (Phase 13)
   - signalfd uses 10ms WaitQueue polling instead of direct signal delivery wakeup
   - Requires global watcher registry for direct integration
   - Current approach is vastly better than yield-loop (true sleep vs CPU spinning)

2. **sendfile buffer optimization vs true zero-copy** (Phase 14)
   - Uses 64KB kernel buffer copy, not page-level zero-copy
   - True zero-copy requires VFS page cache abstraction and FileOps.splice_pages
   - Current approach provides 16x improvement over prior 4KB buffer

3. **aarch64 test suite timeout** (pre-existing)
   - Test suite sometimes times out on aarch64 in later tests
   - Not caused by any v1.1 changes
   - Infrastructure issue to address in future milestone
