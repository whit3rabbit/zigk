**Gap 1 fix -- SIGPIPE EPIPE assertion (lines ~1116-1126):**

In `testSigpipeMsgNosignal`, the variable `got_epipe` is computed and then discarded with `_ = got_epipe`. Replace the discard with an actual assertion. The test must fail if write() did NOT return BrokenPipe.

Replace lines 1116-1126 (from `const got_epipe =` through `_ = got_epipe;`) with:
```zig
    // write_result is null if BrokenPipe was returned (block above breaks with null),
    // or Some(0) if BrokenPipe was caught and converted. Either way, not-null+nonzero means
    // write succeeded, which is wrong for a broken pipe.
    const got_epipe = (write_result == null or write_result == @as(?usize, 0));
    if (!got_epipe) return error.TestFailed; // MUST get EPIPE on broken pipe write
```

Remove the old `_ = got_epipe;` line and the surrounding comment about "unexpected but not fatal". The assertion must be strict: writing to a broken pipe MUST return EPIPE when SIG_IGN is installed.

**Gap 2 fix -- SWS avoidance small writes (lines ~938-966):**

In `testSwsAvoidance`, replace the single 10-byte write with 10 individual 1-byte writes. This exercises the small-write path through the TCP stack. Even though we cannot observe segment coalescing from userspace, sending multiple tiny writes is a more authentic exercise of the SWS avoidance and Nagle algorithm code paths.

Replace the body of `testSwsAvoidance` (keeping the function signature and the TCP pair setup/defer lines) with:
```zig
    // Send 10 individual 1-byte writes from client to exercise the small-write
    // path through Nagle/SWS avoidance. In a single-threaded test runner we cannot
    // concurrently observe coalescing, but this at least pushes data through the
    // "tiny write" code path in the TCP send logic.
    const digits = "0123456789";
    var write_count: usize = 0;
    for (digits) |byte| {
        const single = [_]u8{byte};
        _ = syscall.write(pair.client_fd, &single, 1) catch |err| {
            if (err == error.NotImplemented or err == error.NetworkDown) return error.SkipTest;
            break; // Accept partial delivery
        };
        write_count += 1;
    }
    if (write_count == 0) return error.TestFailed; // Must send at least one byte

    // Use MSG_WAITALL to receive exactly write_count bytes.
    var recv_buf: [10]u8 = undefined;
    const n = syscall.recvfromFlags(pair.accepted_fd, recv_buf[0..write_count], syscall.MSG_WAITALL, null) catch |err| {
        if (err == error.NotImplemented or err == error.NetworkDown) return error.SkipTest;
        return error.TestFailed;
    };

    if (n != write_count) return error.TestFailed;
    if (!std.mem.eql(u8, recv_buf[0..n], digits[0..write_count])) return error.TestFailed;
```

**Gap 3 fix -- MSG_WAITALL multi-segment (lines ~1189-1216):**

In `testMsgWaitallMultiSegment`, replace the single 8-byte write with two separate 4-byte writes ("ABCD" then "EFGH") with a nanosleep(1ms) between them to encourage separate TCP segments. Then MSG_WAITALL requests 8 bytes to accumulate both.

Replace the body of `testMsgWaitallMultiSegment` (keeping function signature, TCP pair setup, and defer lines) with:
```zig
    // Send 8 bytes in TWO separate writes to encourage separate TCP segments.
    // The nanosleep between writes gives the TCP stack a chance to send the first
    // segment before the second write arrives.
    const part1 = "ABCD";
    const sent1 = syscall.write(pair.client_fd, part1.ptr, part1.len) catch |err| {
        if (err == error.NotImplemented or err == error.NetworkDown) return error.SkipTest;
        return error.TestFailed;
    };
    if (sent1 != part1.len) return error.TestFailed;

    // Small delay to encourage separate TCP segments (1ms)
    syscall.sleep_ms(1) catch {};

    const part2 = "EFGH";
    const sent2 = syscall.write(pair.client_fd, part2.ptr, part2.len) catch |err| {
        if (err == error.NotImplemented or err == error.NetworkDown) return error.SkipTest;
        return error.TestFailed;
    };
    if (sent2 != part2.len) return error.TestFailed;

    // Server calls MSG_WAITALL requesting exactly 8 bytes.
    // MSG_WAITALL should accumulate data from both writes (possibly separate segments).
    var buf: [8]u8 = undefined;
    const n = syscall.recvfromFlags(pair.accepted_fd, buf[0..8], syscall.MSG_WAITALL, null) catch |err| {
        if (err == error.NotImplemented or err == error.NetworkDown) return error.SkipTest;
        return error.TestFailed;
    };

    if (n != 8) return error.TestFailed;
    if (!std.mem.eql(u8, buf[0..8], "ABCDEFGH")) return error.TestFailed;
```

**Gap 4 fix -- Raw socket with traffic (lines ~970-997):**

In `testRawSocketBlockingRecv`, after the initial MSG_DONTWAIT empty-queue check, add a section that generates UDP loopback traffic and then does a second MSG_DONTWAIT recv on the raw socket to see if it captured anything. Raw sockets with IPPROTO_ICMP only see ICMP packets (not UDP), so we do not assert that data arrives -- we just attempt it. The important thing is the code path does not crash or hang. Add after the existing WouldBlock check:

```zig
    // Additionally, generate some loopback IP traffic (UDP) and try again.
    // Raw ICMP sockets may not see UDP packets, but this exercises the raw socket
    // recv path after network activity. We do not assert data arrival since the
    // protocol filter may exclude UDP packets from ICMP raw sockets.
    const udp_fd = syscall.socket(syscall.AF_INET, syscall.SOCK_DGRAM, 0) catch {
        return; // UDP unavailable, raw socket empty-queue test already passed
    };
    defer _ = syscall.close(udp_fd) catch {};

    const localhost = syscall.parseIp("127.0.0.1") orelse return;
    var udp_addr = syscall.SockAddrIn.init(localhost, 9212);
    syscall.bind(udp_fd, &udp_addr) catch {
        return; // Bind failed, raw socket empty-queue test already passed
    };

    // Send a UDP packet to self to generate loopback IP traffic
    const ping_msg = "rawtest";
    _ = syscall.sendto(udp_fd, ping_msg, &udp_addr) catch {
        return; // Send failed, raw socket empty-queue test already passed
    };

    // Small delay for loopback packet processing
    syscall.sleep_ms(1) catch {};

    // Try non-blocking recv again on raw socket after traffic generation.
    // We accept any outcome: data (raw socket saw something), WouldBlock (protocol
    // filter excluded the UDP packet), or other error. No assertion on content.
    const result2 = syscall.recvfromFlags(raw_fd, buf[0..256], syscall.MSG_DONTWAIT, null);
    _ = result2; // Any outcome acceptable -- we are testing the code path, not data capture
```

IMPORTANT: The existing function body for `testRawSocketBlockingRecv` has a control flow where `result` is checked with if/else and the function may return early on TestFailed. The new UDP traffic code must go AFTER the existing result check block ends (after line 997's closing brace), before the function's final closing brace.
