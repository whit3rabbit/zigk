- Raw socket blocking recv -> scheduler block/wake pattern complete

### Integration Issues (2)

**1. SO_RCVBUF/SO_SNDBUF pre-connect propagation gap**
- When `setsockopt(SO_RCVBUF)` is called before `connect()`, the Socket stores the value but `Tcb.init()` creates a new TCB with `rcv_buf_size=0`, ignoring the Socket-level setting.
- Impact: Minor. Most applications call setsockopt after connect. The requirement text ("accepted via setsockopt, value stored and applied") is satisfied for the post-connect case.
- Fix: `Tcb.init()` should accept parent Socket reference and copy buffer sizes.

**2. TCP_CORK flush locking**
- `options.zig` calls `transmitPendingData()` under `sock.lock` but without `tcb.mutex`. If `processTimers()` runs concurrently, there is a race on `snd_nxt` and `retrans_timer`.
- Impact: Minor. Single-threaded kernel context makes this unlikely to trigger, but it violates the locking contract.
- Fix: Acquire `tcb.mutex` before calling `transmitPendingData()` in the uncork path.
